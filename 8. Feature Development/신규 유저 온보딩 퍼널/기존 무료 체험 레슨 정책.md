
## PODO 무료 체험 레슨 & 온보딩 정책 종합

### 1. 체험 레슨 결제 타입

PODO에는 **두 가지** 체험 결제 방식이 있어요:

|PaymentType|설명|금액|
|---|---|---|
|`TRIAL`|체험 결제 (빌링키 발급)|500원 (첫 체험권 기준)|
|`TRIAL_FREE`|무료 체험|0원|

**`TRIAL_FREE` 조건**: 첫 체험권(500원 티켓)에서만 사용 가능. BFF 서버에서 `subPrice !== 500`이면 거부됨.

---

### 2. 체험 커리큘럼 타입 결정 (핵심 정책)

`SubscriptionEntity.applyTrialLogic`에서 `trialClasses` 배열과 구독 구매 여부를 조합해요:

|조건|curriculumType|의미|
|---|---|---|
|체험 수업 2회 이상|`TRIAL`|이미 체험 경험 있음|
|체험 1회 + 해당 언어 체험 완료|`TRIAL`|동일 언어 재체험|
|체험 1회 + **다른 언어** 체험|`TRIAL_2`|새 언어 첫 체험|
|체험 0회 + 다른 언어 구독 구매 있음|`TRIAL_2`|기존 유료 사용자의 새 언어 체험|
|체험 0회 + **첫 구매**|`TRIAL_1`|완전 신규 사용자|

- `TRIAL` 타입이면 → 기존 체험 수업 레벨과 동일한 레슨만 필터링
- `TRIAL_1` = 완전 신규, `TRIAL_2` = 다른 언어 체험

---

### 3. 체험 레벨 (trialLevel)

체험 신청 시 레벨을 선택해요:

```
'B' | 'C1' | 'C2' | 'D'
```

이 값은 결제 리다이렉트 파라미터와 웹훅 호출 시 `trial_level`로 전달됩니다.

---

### 4. 온보딩 플로우 (체험 결제 후)

**trial-tutorial 도메인**이 담당하며, 표시 조건:

> `trialPaymentYn === 'Y' && trialClassCompYn === 'N'` (체험 결제 완료 + 수업 미완료)

**플로우:**

1. 체험 결제 완료
2. **튜토리얼 화면** 표시 (첫 수업 안내, 앱 사용법, 체험 레슨 준비 가이드)
3. 튜토리얼 완료 → **체험 레슨 예약 화면**으로 이동

**API:**

- `GET /api/v1/user/podo/getTrialStepInfo` — 현재 체험 단계 조회
- `POST /api/v1/user/podo/insertStep` — 단계 업데이트 (stepCode)

---

### 5. 첫 수업 부스터 (first-lesson-booster)

체험 예약 후 첫 수업 전 **예습 독려 퍼널**:

- 예습을 건너뛰려 하면 독려 메시지 표시
- **최대 건너뛰기 횟수(`max_skip_count`)** 초과 시 건너뛰기 불가

---

### 6. 홈 화면 상태 우선순위 (체험 관련)

home-greeting에서 사용자 상태에 따라 화면이 결정돼요:

|우선순위|조건|화면|
|---|---|---|
|1|수강권 없음|`NoTicketState`|
|2~5|예약된 수업 있음|입장/예습 관련 화면|
|5|수강권 있음 + 예약 없음|`RecommendRegularLesson`|
|**6**|**체험 미진행**|**`RecommendTrialLesson`**|

배너 조건 (레거시 Vue → React 포팅):

```typescript
if (user.payment_yn !== 'Y' && user.trial_payment_yn === 'N') {
  return <TrialBanner />  // 체험도, 결제도 안 한 사용자에게 체험 배너
}
```

### 7. 체험 수업 데이터 구조 (GT_CLASS)

|필드|값|의미|
|---|---|---|
|`CITY`|`PODO_TRIAL`|체험 수업 구분자|
|`TUTOR_PRICE_PER_CLASS`|`0`|체험 무료 튜터 배정|
|`INVOICE_STATUS`|`CREATED` → `RESERVED` → `COMPLETED`|수업 상태 흐름|

체험 수업에는 예습(`IS_PRESTUDY='Y'`) 수업도 별도 레코드로 생성됩니다.

---

### 8. 체험 → 유료 전환 시 잠금 해제

다음 조건이 **모두** 충족되면 만료된 체험 레슨이 다시 표시돼요:

- 동일 언어 정기 구독 구매 완료
- 체험 수업 완료 (`classStatus === '완료'`)
- 인보이스 상태 `CREATED`
- `isPurchased === false`

반대로, **레슨 그룹이 비워지는** 조건:

- 구독 구매 + 인보이스 `CANCEL` → 빈 배열
- 구독 구매 + 수업 `미수업` → 빈 배열

---

`★ Insight ─────────────────────────────────────`

- PODO의 체험 시스템은 **커리큘럼 타입 3단계(`TRIAL_1`, `TRIAL_2`, `TRIAL`)** 로 신규/재방문/다언어 사용자를 구분하는 게 핵심이에요. 단순히 "무료 체험 1회"가 아니라, 사용자의 이전 체험 이력과 구독 이력을 조합해서 맞춤형 체험을 제공하는 구조입니다.
- `TRIAL_FREE`(0원)와 `TRIAL`(500원)의 분기는 BFF 서버 레벨에서 검증하고, 백엔드 웹훅을 직접 호출하는 방식으로 처리돼요. 포트원 PG 연동 없이 서버 간 직접 통신하는 특수한 결제 흐름입니다. `─────────────────────────────────────────────────`
