# Brief: 신규 유저 첫 체험 완료 퍼널

> 이 문서는 [[PRD - 신규 유저 온보딩 퍼널]] v1.0의 미팅 사전 읽기용 요약이다.
> 전체 PRD를 읽지 않아도 배경·현황·방향을 이해하고 미팅에 참여할 수 있도록 구성했다.

**대상 유저**: 앱을 처음 다운로드하고 가입한 유저 (체험 레슨 전)
**담당**: Squad C — 온보딩/리텐션 | PO: 이준서

---

## 1. 핵심 지표 정의 및 현황

### North Star Metric

> **체험 레슨 완료율** = 체험 레슨 정상 완료 유저 / 가입 유저

예약률이 아니라 **실제 수업 완료**까지를 핵심 지표로 잡았다. 예약만 해놓고 수업에 안 오면 서비스 가치를 체험하지 못하고, 결제 전환으로 이어지지 않기 때문이다.

### 현재 퍼널 수치

```
가입 ──→ 체험 예약 ──→ 체험 수업 참석 ──→ 체험 완료
          (측정 필요)     17.4%             ~17%
                        (예약 대비)       (가입 대비 추정)
```

| 지표 | 현재 | 목표 |
|---|---|---|
| **체험 레슨 완료율** (North Star) | ~17% (가입 대비, 추정) | **40% 이상** |
| 무료 체험 예약률 | 기준선 측정 필요 | +20%p 개선 |
| 예약 → 수업 참석률 | 17.4% (= 노쇼 82.6%) | **50% 이상** |
| 체험 → 정규 결제 전환율 | 측정 필요 | +10%p 개선 |

> **주의**: 현재 82.6% 노쇼율에는 React 전환 시 발생한 **"수업 입장하기" CTA 버그**가 포함되어 있다. 수업 시간에 접속했지만 레슨방에 입장할 수 없어 노쇼로 집계된 케이스가 상당수다. 이 버그는 현재 수정 중이며, **버그 수정 후 순수 UX 기인 노쇼율을 재측정해야 한다.**

## 2. 지표에 영향을 주는 요소

### 예약률을 떨어뜨리는 요소

| 요소 | 설명 |
|---|---|
| **가치 전달 부재** | 로그인 직후 서비스가 뭔지 설명 없이 바로 결제/예약 페이지로 이동시킴 |
| **레벨 매칭 실패** | 유저가 자기 레벨을 5단계에서 직접 고르는 방식. 진단 없이 정확한 매칭 불가 |
| **개인화 없음** | 학습 목표를 묻지 않아 "나를 위한 서비스"라는 인식이 형성되지 않음 |
| **이탈 후 재진입 경로 없음** | 예약 페이지에서 이탈하면 다시 예약으로 돌아오는 자연스러운 동선이 없음 |

### 참석률을 떨어뜨리는 요소

| 요소 | 설명 |
|---|---|
| **CTA 버그 (주요 원인)** | 2025년 하반기 React 전환 과정에서 "수업 입장하기" 버튼이 작동하지 않는 버그. 수정 중 |
| **대기 시간 공백** | 예약과 수업 사이에 유저를 붙잡아두는 장치가 없음 (교재 미리보기, AI학습 등) |
| **1회 한정 체험** | 무료 체험이 1회뿐이라 노쇼하면 기회 소멸 — 그런데 이걸 예약 시점에 충분히 인지시키지 않음 |
| **레슨방 입장 불안** | 입장 직전에 긴장감을 완화해주는 안내가 없음 |

### 결제 전환을 떨어뜨리는 요소

| 요소 | 설명 |
|---|---|
| **진단 리포트 도달 경로 미흡** | 체험 완료 후 AI 진단 리포트가 카카오톡 알림톡으로만 전달됨. 앱 내에서 바로 확인하는 경로 없음 |
| **결제 유도 시점 부재** | 체험 직후(동기 최고조 시점)에 결제로 연결하는 CTA가 없음 |

## 3. 서비스 플로우 전반 딥다이브 및 진단

### AS-IS: 현재 신규 유저 경험

```
카카오/Apple 로그인
  → (아무 설명 없이) /subscribes/trial 페이지로 자동 이동
  → 언어 선택 (EN/JP)
  → 레벨 자가 선택 (5단계)
  → 결제 (30,000원 또는 trial_free ON 시 0원)
  → 예약 완료
  → (대기 시간 — 아무 장치 없음)
  → 수업 시간 → 입장 CTA 버그로 입장 불가 / 또는 그냥 안 옴
  → 체험 완료 시 → 카카오톡으로 리포트 발송 (앱 밖)
  → (결제 유도 없음)
```

**핵심 진단**: 퍼널의 모든 단계에서 유저를 놓치고 있다.
- **진입**: 서비스 가치를 모르는 상태에서 결제/예약 요구
- **대기**: 예약 후 수업까지 아무 경험 없음
- **입장**: CTA 버그로 물리적 입장 불가 (수정 중)
- **완료 후**: 진단 리포트가 앱 밖(카카오톡)에 있고, 결제 유도 없음

### TO-BE: 제안하는 신규 유저 경험

퍼널을 3개 Sub-PRD로 나누어 단계적으로 개선한다.

```
[Sub 1: 온보딩 → 예약]
  로그인 → 앱 소개(3화면) → 언어 선택 → 목표 설정 → 레벨 추천(5문항)
  → 맞춤 결과 안내 → 무료 체험 예약 → 예약 완료

[Sub 2: 예약 → 체험 완료]
  대기 홈(교재·AI학습 CTA) → 리마인드 → 레슨방 입장 → 수업 → 인앱 진단 리포트

[Sub 3: 체험 완료 → 결제 유도]
  인앱 리포트 하단 결제 CTA → 수강권 구매 페이지
```

### Sub 1 — 온보딩 플로우 변경 요약

| AS-IS | TO-BE | 변경 이유 |
|---|---|---|
| (없음) | 앱 소개 2~3화면 | 서비스가 뭔지 먼저 알려줌 |
| 언어 선택만 | 환영 + 언어 선택 | 첫인상 개선 |
| (없음) | 학습 목표 설정 | 개인화 데이터 수집 |
| 레벨 자가 선택 5단계 | 5문항 퀴즈 기반 레벨 추천 | 레벨 매칭 정확도 향상 |
| (없음) | 추천 결과 + 가치 안내 | 체험 동기 강화 ("AI 진단 리포트 예고") |
| 14일 캘린더 전체 노출 | 추천 시간 4개 우선 노출 | 선택 피로 감소, 전환율 향상 |

전체 소요 시간: **약 2분 15초** (라이트한 플로우)

### Sub 2 — 예약 → 체험 완료 변경 요약

| 항목 | 내용 |
|---|---|
| 대기 홈 화면 | 레슨 상태 카드 + AI학습/교재 미리보기 CTA |
| 레슨방 입장 | 10분 전 입장 가능 + 긴장감 완화 워딩 |
| 노쇼 처리 | 학생 노쇼 → 재예약 유도, 튜터 노쇼 → 체험권 복구 |
| 진단 리포트 | 카카오톡 알림톡 → **인앱 리포트**로 변경 |

### Sub 3 — 체험 완료 → 결제 유도 (v1 MVP)

인앱 진단 리포트 하단에 **결제 CTA 1개 추가**하는 것이 v1 전체 스코프.
홈 화면 전체 재구성(혜택 카드, 프로모션 배너 등)은 v2에서 진행.

## 4. 트렌드/벤치마크 리서치

### 온보딩 설계 벤치마크

| 서비스 | 온보딩 패턴 | 포도 적용 |
|---|---|---|
| **듀오링고** | 스플래시 → 캐릭터 인사 → "7 quick questions" 예고 → 언어/레벨 → 소셜 프루프 → 목표 → 루틴/일일목표 → 알림 → 성과 예고 → 유무료 선택 → 레벨 배치 | 인지적 준비(질문 예고), 가치 선행, 레벨 추천, 성과 예고 |
| **Preply** | 스플래시(2화면) → 언어 선택 → 가입 → 목표 → 토픽 추천 → 문화/억양 → 요일 → 시간대 → 예산 → 맞춤 튜터 리스트 | 목표 기반 추천, 일정 선호 수집, "All set!" 완료감 |
| **캄(Calm)** | 목적 질문 → 경험 수준 → 맞춤 추천 → 무료 체험 | 목표 설정 → 레벨 추천 → 맞춤 결과 흐름 |
| **스피킹맥스/튜터링** | 레벨 테스트 → AI 분석 → 커리큘럼 추천 | 온보딩 레벨 추천 + 첫 수업 후 AI 정밀 분석 2단계 구조 |

### 듀오링고 온보딩 상세 분석

언어학습 앱의 온보딩 표준으로 꼽히는 레퍼런스. 가입 전에 가치를 먼저 체험시키는 구조가 핵심이다.

**플로우 (총 ~14단계):**
```
스플래시 → 캐릭터 인사 ("Hi there! I'm Duo!")
→ 온보딩 예고 ("Just 7 quick questions before we start your first lesson!")
→ 언어 선택
→ 레벨 자가 선택 (5단계: "I'm new" ~ "I can discuss most topics in detail")
→ 소셜 프루프 ("7 million people currently learning French!")
→ 학습 목표 (복수 선택: Just for fun, Boost career, Travel 등)
→ 대화 준비 예고 ("Let's prepare you for conversations!")
→ 학습 루틴 설정 ("Let's set up a learning routine!")
→ 일일 목표 (5/10/15/20 min — "Regular" 10min 기본 선택)
→ 주간 목표 예고 ("That's 50 words in your first week!")
→ 알림 권한 + 위젯 추가
→ 3개월 달성 성과 예고 ("Here's what you can achieve in 3 months!")
→ 유료/무료 선택 ("Super Duolingo" vs "Learn for free")
→ 시작 레벨 ("Start from scratch" vs "Find my level" — Recommended)
→ 레벨 배치 결과 ("Since you know a few words, let's start at Score 10!")
```

**포도에 시사하는 점:**

| 듀오링고 패턴 | 포도 적용 여부 | 판단 |
|---|---|---|
| 캐릭터 기반 인사 + 온보딩 예고 | **적용** | Intro 3화면에서 캐릭터 일러스트 + "간단한 질문 몇 개" 예고 구조 차용 |
| "7 quick questions" 사전 고지 | **적용** | 인지적 준비(Cognitive Priming) — 질문이 올 거라는 기대 형성으로 이탈률 감소 |
| 소셜 프루프 ("7 million people") | **미적용 (검토 가능)** | 포도 규모에서 숫자 소셜 프루프는 역효과 가능. 리뷰/후기 형태가 더 적합할 수 있음 |
| 학습 목표 복수 선택 | **적용** | Step 2 학습 목표 설정과 동일 |
| 일일/주간 목표 설정 | **미적용** | 포도는 수업 기반 서비스라 일일 학습 목표보다 "체험 예약"이 더 적합한 다음 단계 |
| 유료/무료 선택을 온보딩 후반에 배치 | **참고** | 가치를 충분히 보여준 뒤 결제를 묻는 순서. 포도도 Step 4 가치 안내 후 Step 5 예약으로 동일 |
| "Find my level" 레벨 테스트 추천 | **적용** | Step 3 레벨 추천 5문항과 동일한 패턴 |
| 3개월 후 성과 예고 | **검토 가능** | Step 4 결과 화면에서 "첫 수업 후 AI 진단 리포트" 예고와 유사한 동기부여 장치 |

### Preply 온보딩 상세 분석

Preply는 1:1 언어 튜터링 플랫폼으로, 포도와 서비스 구조가 가장 유사한 벤치마크다.

**플로우 (총 ~10단계):**
```
스플래시 2화면 ("Find the right tutor" / "Practice makes possible")
→ 언어 선택 → 회원가입 → 알림 권한
→ 목표 설정 (복수 선택, Skip 가능)
→ 토픽 추천 (목표 기반 추천 + "Show all"로 전체 노출)
→ 문화/억양 선호 (국가 플래그 토글)
→ 수업 가능 요일 (Mon~Sun 선택, Skip 가능)
→ 선호 시간대 (Morning/Afternoon/Evening/Night → 세부 시간대)
→ 예산 범위 (슬라이더)
→ "All set! Ready to meet them?"
→ 맞춤 튜터 리스트 (필터 적용 상태)
```

**포도에 시사하는 점:**

| Preply 패턴 | 포도 적용 여부 | 판단 |
|---|---|---|
| 목표 → 토픽 자동 추천 | **적용** | Step 2 목표 → Step 4 맞춤 결과로 연결하는 구조와 동일 |
| 대부분 단계에 "Skip this question" 제공 | **미적용** | 포도는 ~2분 라이트 플로우이므로 Skip 없이 진행 (Step 5 예약만 "나중에" 제공) |
| 일정·시간대 선호 수집 | **적용** | Step 5에서 추천 시간 4개 우선 노출로 유사하게 구현 |
| 예산 질문 | **미적용** | 포도는 무료 체험이므로 불필요 |
| 온보딩 끝에 매칭 결과 노출 | **적용** | Step 4 맞춤 결과 + Step 5 예약으로 연결 |
| 단계별 일러스트/비주얼 | **디자인 참고** | 각 단계마다 상단에 일러스트를 배치해 시각적 흥미 유지 |

### 핵심 인사이트

1. **가치 선행, 결제 후행**: 성공적인 서비스들은 서비스가 뭔지 먼저 보여준 뒤 결제/예약을 요청함
2. **인지적 준비(Cognitive Priming)**: "지금부터 질문할 거야"라고 미리 알려주면 갑작스러운 질문보다 이탈률이 낮음
3. **목표 → 추천 연결**: Preply, 듀오링고 모두 목표를 먼저 물은 뒤 그에 맞는 추천을 제공. 포도의 Step 2(목표) → Step 4(맞춤 결과) 흐름과 동일한 패턴
4. **추천 시간 패턴**: 전체 캘린더 대신 추천 슬롯 소수를 먼저 보여주는 것이 선택 피로 감소에 효과적 (Preply도 요일·시간대를 미리 수집해 필터링)
5. **2단계 레벨링**: 온보딩에서 대략적 추천 → 실제 체험 후 AI 정밀 분석. 온보딩 부담을 줄이면서 정확도 확보
6. **단계별 비주얼**: Preply는 모든 질문 화면 상단에 일러스트를 배치해 텍스트 피로를 줄임. 디자인팀 참고 포인트

## 5. 개선 방향 및 우선순위

### 실행 우선순위

```
[긴급] CTA 버그 수정 (현재 진행 중)

[2순위] Sub 2: 예약 → 체험 완료 (W5~W10, Sub 1과 일부 병렬)
  → 참석률 개선. 대기 경험 + 노쇼 방지 + 인앱 진단 리포트

[3순위] Sub 3: 체험 완료 → 결제 유도 (W8~W10)
  → 전환율 개선. v1은 인앱 리포트 내 결제 CTA만 스코프
```

### 실험 방식

- **A/B 테스트 2그룹**: Control(기존 플로우) vs Treatment(온보딩 플로우)
- **의사결정 기준**: 체험 완료율 +10%p 이상이면 전체 롤아웃
- **최소 표본**: 각 그룹 200명, 최소 2주 실험
