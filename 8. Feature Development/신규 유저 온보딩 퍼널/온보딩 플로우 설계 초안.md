# 신규 유저 온보딩 플로우 설계 초안

> 이 문서는 [[PRD — 신규 유저 온보딩 퍼널]] (v0.4)의 상세 화면 설계·트래킹 이벤트 정의 문서다.

## 설계 원칙

1. **가치 선행, 결제 후행** — 서비스의 가치를 먼저 체감시킨 후 전환 유도
2. **개인화된 첫인상** — 시작 레벨 추천과 목표 설정으로 "나를 위한 서비스"라는 인식 형성
3. **추천, 제한 아님** — 레벨 추천은 출발점일 뿐. 유저는 모든 레벨/주제를 자유 선택 가능
4. **최소 마찰** — 전체 온보딩 ~2분 이내 완료. Step 1~4 건너뛰기 없음 (충분히 라이트하므로)
5. **정밀 레벨링은 AI에 위임** — 온보딩 레벨 추천은 대략적. 정확한 분석은 첫 수업 후 AI 리포트가 담당
6. **측정 가능** — 모든 단계에 이벤트 트래킹, 단계별 이탈률 측정
7. **점진적 롤아웃** — `onboarding_v1` 피처 플래그 기반 A/B 테스트로 검증 후 확대

---

## 현재 플로우 (AS-IS) — 코드 검증 완료

```
카카오/Apple 로그인
  → isTrialPopUpOpenCondition 체크
  → /subscribes/trial 자동 리다이렉트
  → 체험레슨 신청 (언어 선택 + 레벨 자가선택 + 결제)
```

**게이트 조건:**
```
isTrialPopUpOpenCondition = trialClassCompYn === 'N' && !(paymentYn === 'Y' || trialPaymentYn === 'Y')
```

**문제점:**
- 서비스가 뭔지 모르는 상태에서 결제/예약 페이지로 바로 보냄
- 레벨을 "자가 선택"하게 해서 적정 레벨 매칭 실패
- 학습 목적/목표를 모르니 개인화 불가
- 결과: 1회차 이탈 82.6%

---

## 제안 플로우 (TO-BE)

```
카카오/Apple 로그인
  → [Intro] 앱 소개 — 2~3화면 스와이프           (10~15초)
  → [Step 1] 환영 + 학습 언어 선택              (15초)
  → [Step 2] 학습 목표 설정                      (20초)
  → [Step 3] 시작 레벨 추천 — 5문항            (60초)
  → [Step 4] 추천 결과 + 서비스 가치 안내         (20초)
  → [Step 5] 무료 체험 레슨 예약                 (15초)
  → 예약 완료 화면 (+ 후속 경험 CTA 슬롯)
  → 홈 진입
```

**2회차 이후**: `onboarding_completed = true` → 바이패스 → 홈 직행

### 건너뛰기 정책

> **Intro + Step 1~4: 건너뛰기 없음.** Intro 포함 전체 ~2분 15초의 라이트한 플로우이므로 건너뛰기 옵션을 제공하지 않는다.
> **Step 5만 "나중에 할게요" 제공.** 체험 예약은 유저 의사 존중. 건너뛴 유저에게는 홈 배너로 재유도.

## 단계별 상세 설계

### Intro: 앱 소개 (2~3화면)

**목적**: 브랜드 첫인상 형성, 서비스 핵심 가치 인지, 온보딩 진행 예고

**화면 구성 (스와이프 또는 자동 전환):**

| 화면 | 내용 | 역할 |
|---|---|---|
| Intro 1 | 포도 로고 + 캐릭터 일러스트 + "포도에 오신 걸 환영해요" | 브랜드 첫인상 — "이게 뭔 앱이지?" 해소 |
| Intro 2 | 핵심 가치 한 줄 + 아이콘/일러스트 ("1:1 전화영어, AI가 실력을 분석해줘요") | 서비스 이해 — 포도가 뭘 해주는 앱인지 |
| Intro 3 | "간단한 질문 몇 개로 맞춤 수업을 준비할게요" + "시작하기" CTA 버튼 | 온보딩 예고 — 질문이 나올 거라는 기대 형성 + 진입 유도 |

**인터랙션:**
- 하단 페이지 인디케이터 (도트 3개)
- 좌우 스와이프로 화면 전환
- Intro 3에서 "시작하기" CTA 탭 → Step 1으로 진행
- 자동 전환 없음 (유저가 직접 스와이프/탭)

**설계 의도:**
- 듀오링고 레퍼런스 — 스플래시 → 캐릭터 인사 → 온보딩 예고의 3단계 패턴 차용
- **인지적 준비(cognitive priming)**: "지금부터 질문할 거야"라고 미리 알려주면 갑작스러운 질문 진입보다 이탈률이 낮아짐
- 앱을 처음 다운로드한 유저에게 "이게 뭔 서비스인지" 알려주는 최소한의 맥락 제공
- 3화면 × ~5초 = 10~15초. 전체 온보딩 시간에 큰 부담 없음

**트래킹 이벤트:**
- `onboarding_intro_view` — Intro 첫 화면 진입
- `onboarding_intro_swipe` — 스와이프 (page: 1/2/3)
- `onboarding_intro_cta_click` — "시작하기" CTA 클릭
- `onboarding_intro_complete` — Intro 완료 (→ Step 1 진입)

---

### Step 1: 환영 + 학습 언어 선택

**목적**: 첫인상 형성, 개인화 시작

**화면 구성:**
- 포도 브랜드 일러스트/로고
- "반가워요! 어떤 언어를 배울까요?"
- 언어 카드 2개: 🇺🇸 영어 / 🇯🇵 일본어
- 카드 선택 시 바로 다음 단계로 전환 (별도 버튼 불필요)

**설계 의도:**
- 질문을 하나만 해서 부담 최소화
- "로그인했더니 뭘 고르라네?" 가 아니라 "나한테 맞춰주려는 거구나" 인식
- 기존 `/subscribes/trial`에도 언어 선택이 있으므로, 이 데이터를 재활용

**트래킹 이벤트:**
- `onboarding_step1_view` — 화면 진입
- `onboarding_step1_select_language` — 언어 선택 (language: en/ja)
- `onboarding_step1_complete` — 단계 완료

---

### Step 2: 학습 목표 설정

**목적**: 동기 부여, 개인화 데이터 확보, 커밋먼트 효과

**화면 구성:**
- "영어 배우는 이유가 궁금해요!"
- 목표 카드 (복수 선택 가능, 최대 2개):
  - 💼 비즈니스/업무 회화
  - ✈️ 여행/일상 회화
  - 📚 시험 준비 (토익, 토플 등)
  - 🎯 자기계발/취미
- "다음" 버튼

**선택에 따른 후속 영향:**
- Step 4 결과 화면에서 목표에 맞는 커리큘럼 미리보기 제공
- 추후 튜터 매칭/레슨 추천에 활용 가능 (백엔드 확장)

**설계 의도:**
- 목표를 "선택"하게 하면 자기 결정 효과(self-determination) 발생
- 사용자가 목표를 설정하면 이탈 비용이 심리적으로 높아짐
- 4개 이하 선택지로 선택 피로도 방지

**트래킹 이벤트:**
- `onboarding_step2_view`
- `onboarding_step2_select_goal` — 목표 선택 (goals: [])
- `onboarding_step2_complete`

---

### Step 3: 시작 레벨 추천

**목적**: 추천 시작 레벨 제시. "이 서비스는 내 실력을 파악해준다"는 인식. **잠금이 아닌 추천.**

**화면 구성:**
- "간단한 질문으로 레벨을 맞춰볼게요"
- 진행바 (5문항 기준)
- 기존 퀴즈 인프라 재활용 문항:
  - 활용 가능 문제 타입: `MULTI_CHOICE_WORD`, `BLANK_FILLING`, `MULTI_CHOICE_SENTENCE`, `SENTENCE_MAKING`
  - 문항 수: 5문항 (교육팀과 협의하여 확정)
  - API: `/api/v1/diagnosis/getQuestions` (GET), `/api/v1/diagnosis/checkAnswer` (POST)
- 문항당 제한시간 없음 (but 평균 소요시간 트래킹)

**레벨 추천 로직:**
- 5문항 답변 결과 → 추천 시작 레벨 매핑
- **추천 결과는 잠금이 아님** — 유저는 이후 모든 레벨/주제를 자유롭게 선택 가능
- 추천 레벨은 서버에 저장 → 첫 수업 기본값으로 제안
- 정밀 레벨 분석은 첫 수업 후 AI 리포트(FLUENCY/VOCAB/GRAMMAR/COMPLEXITY 점수 + A~F 등급)에 위임

**설계 의도:**
- "자가 선택"이 아닌 "추천"으로 바꾸면 개인화된 느낌 상승
- 5문항이면 60초 내 완료 가능 → 이탈 최소화
- 5문항: 신뢰감 확보와 이탈 방지의 균형점. 60초 내 완료 가능
- 추천일 뿐 잠금이 아니므로, 유저 자유도를 제한하지 않음

**트래킹 이벤트:**
- `onboarding_step3_view`
- `onboarding_step3_answer` — 각 문항 답변 (question_id, question_type, answer, is_correct, time_spent_ms)
- `onboarding_step3_complete` — 추천 완료 (recommended_level)

---

### Step 4: 추천 결과 + 서비스 가치 안내

**목적**: "나를 위한 서비스"라는 확신, 전환 직전 동기 극대화

**3단 메시지 구성:**

| # | 메시지 | 목적 |
|---|---|---|
| ① | "[이름]님은 **[레벨]**로 시작해보세요" | 개인화된 느낌 |
| ② | "물론, 다른 레벨도 자유롭게 들을 수 있어요" | 선택 폭 제한 불안 해소 |
| ③ | "첫 수업 후 AI가 실력을 분석해줘요" | 체험 수업을 들어야 할 동기 + AI 리포트 가치 예고 |

**화면 구성:**
- 결과 카드:
  - "[이름]님은 **중급(STEP3)**으로 시작해보세요"
  - "물론, 다른 레벨도 자유롭게 들을 수 있어요"
- 서비스 가치 안내:
  - 📊 "첫 수업 후 AI가 실력을 분석해줘요"
  - 🎯 "레벨에 맞는 1:1 전화 수업을 받아보세요"
- "무료 체험 시작하기" CTA 버튼

**설계 의도:**
- 이름 + 추천 레벨 + 목표를 조합한 개인화 메시지 → "나를 알고 있다" 느낌
- 메시지 ②로 "레벨에 갇히는 거 아닐까?" 불안 선제 해소
- 메시지 ③으로 체험 수업 동기 부여 — "수업 들으면 AI가 정밀 분석해줌"
- CTA가 "결제"가 아니라 "무료 체험 시작" → 심리적 장벽 최소화

**트래킹 이벤트:**
- `onboarding_step4_view`
- `onboarding_step4_scroll_depth` — 가치 안내 스크롤/스와이프 깊이
- `onboarding_step4_cta_click` — CTA 클릭
- `onboarding_step4_complete`

---

### Step 5: 무료 체험 레슨 예약

**목적**: 전환(체험 레슨 예약), 온보딩의 마지막 단계

**화면 구성:**
- "레슨 일정을 선택해주세요."
- 수평 날짜 스크롤 (오늘~14일, 실제 앱 `trial-subscribes/view.tsx` UI 재활용)
- 시간 그리드 (15:30~23:30, 30분 간격, 3열 배치) — 예약 마감/2시간 전 비활성화
- 주말: 빈 상태 (포도 캐릭터 + "해당 기간에 유효한 레슨권이 없습니다")
- 레벨은 Step 3 추천 결과로 기본 설정
- "선택한 날짜에 예약" CTA 버튼
- **예약 확인 바텀시트**: CTA 클릭 시 날짜/시간/수업/비용 확인 모달 → "예약 확정" 버튼
- 하단: "나중에 할게요" 텍스트 링크 (→ 홈으로 이동, 홈 배너에서 재유도)

**`trial_free` 플래그 연동:**
- 플래그 ON: 무료 체험 → 결제 수단 입력 없이 바로 예약
- 플래그 OFF: 체험 유료 → 결제 수단 선택 후 예약 (기존 `/subscribes/trial` 결제 UI 재활용)

**설계 의도:**
- 기존 앱 예약 UI를 그대로 재활용 → 개발 비용 절감 + 일관된 UX
- 14일 범위로 충분한 선택지 제공 — "추천 시간대"는 유저 스케줄을 알 수 없으므로 제거
- 확인 바텀시트로 오예약 방지 (날짜/시간/수업 정보 2차 확인)
- "나중에 할게요"를 제공해서 강제성 배제 (but 작은 텍스트로 CTA보다 덜 눈에 띄게)

**트래킹 이벤트:**
- `onboarding_step5_view`
- `onboarding_step5_reserve_click` — 예약 버튼 클릭 (바텀시트 열림)
- `onboarding_step5_confirm` — 바텀시트에서 "예약 확정" 클릭
- `onboarding_step5_skip` — "나중에 할게요" 클릭
- `onboarding_complete` — 온보딩 전체 완료 (reserved: true/false)

---

### 예약 완료 화면

**목적**: 예약 확인 + 후속 경험 연결 준비

**화면 구성:**

| 항목 | 내용 |
|---|---|
| **핵심 메시지** | "체험 레슨이 예약됐어요!" |
| **예약 정보** | 날짜/시간/튜터 정보 |
| **CTA 슬롯** | 후속 PRD에서 정의 (예: "수업 전에 예습해볼까요?", "AI 학습 미리 체험" 등) |
| **홈 진입 버튼** | "홈으로 가기" |

> **CTA 슬롯 안내**: 체험 예약과 실제 수업 사이 시간 갭을 채울 경험(예습, AI 학습 등)은 별도 PRD에서 설계한다. 이 화면에는 해당 CTA를 삽입할 자리만 마련해둔다.

**트래킹 이벤트:**
- `onboarding_reservation_complete_view`
- `onboarding_reservation_cta_click` — CTA 슬롯 클릭 (cta_type)
- `onboarding_reservation_home_click` — "홈으로 가기" 클릭

## 온보딩 완료 후 홈 진입

**예약 완료한 유저:**
- 홈 화면에 기존 `TrialTutorial` 5단계 가이드 노출 (기존 로직 유지)
- 추가: "첫 수업까지 D-N" 카운트다운 배너

**"나중에 할게요" 유저:**
- 홈 화면 상단에 "체험 레슨 예약하기" 배너 지속 노출 (기존 `TrialBanner` 컴포넌트 활용)

## 바이패스 로직

```
로그인 시:
  IF onboarding_v1 플래그 OFF → 기존 플로우 (/subscribes/trial)
  IF onboarding_completed = true → 홈 직행
  IF onboarding_completed = false AND onboarding_step = 0 → Intro부터 시작
  IF onboarding_step > 0 (중간 이탈 후 재진입) → 마지막 완료 단계 다음부터 재개 (Intro는 재진입 시 건너뜀)
```

**저장 위치:**
- 서버: `onboarding_status` 테이블 (user_id, current_step, completed_at, language, goals, recommended_level, skipped_trial)
- 클라이언트: `localStorage`에 캐싱 (기존 `podo_auto_redirected` 패턴 활용)

## A/B 테스트 설계

### 피처 플래그

```
onboarding_v1    // 온보딩 플로우 전체 ON/OFF (단일 플래그)
```

### 실험 그룹

| 그룹 | 플래그 상태 | 경험 | 측정 목표 |
|---|---|---|---|
| Control | `onboarding_v1=OFF` | 기존 플로우 (로그인 → `/subscribes/trial`) | 기준선 |
| Treatment | `onboarding_v1=ON` | 풀 온보딩 (Step 1~5, 시작 레벨 추천 포함) | 체험 예약률, 1회차 잔존율 |

> **왜 2그룹인가?** 유저가 이미 모든 레벨/주제를 자유 선택할 수 있고, 첫 수업 후 AI 리포트로 정밀 레벨링이 되므로, 레벨 추천 생략(자가선택 유지) 그룹을 별도로 둘 실익이 없다.

### 핵심 지표

**Primary:**
- 온보딩 완료율 (Step 5 도달 / Step 1 진입)
- 체험 레슨 예약률 (예약 / 온보딩 완료)
- 1회차 잔존율 (체험 수업 실제 참석 / 예약)

**Secondary:**
- 단계별 이탈률 (각 Step 이탈 / 진입)
- 온보딩 소요 시간 (Step 1 진입 → Step 5 완료)
- 추천 레벨 수용률 (추천 레벨로 첫 수업 선택 / 레벨 추천 완료)
- 체험→정규 전환율 (정규 결제 / 체험 완료)
