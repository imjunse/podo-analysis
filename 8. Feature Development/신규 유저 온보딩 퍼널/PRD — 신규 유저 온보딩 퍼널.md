# PRD: 신규 유저 온보딩 퍼널

## 1. 개요

### 한 줄 요약
기존 체험 예약 팝업(`/subscribes/trial`)을 **개인화된 온보딩 플로우**로 대체하여, 신규 유저가 서비스 가치를 이해한 뒤 자연스럽게 무료 체험 레슨을 예약하게 만든다.

### 배경

현재 신규 유저의 첫 경험은 **체험 레슨 결제/예약 페이지**(`/subscribes/trial`)로의 자동 리다이렉트다. 이 페이지에서 언어 선택 → 레벨 자가 선택 → 결제(또는 `trial_free` 플래그 ON 시 0원 결제) → 예약을 한 번에 처리한다.

문제는 이 페이지가 **서비스가 무엇인지, 왜 나한테 필요한지** 전달하지 않고 바로 예약/결제를 요구한다는 점이다. 사실상 이 페이지가 유일한 "온보딩"이지만, 개인화도 가치 전달도 없다.

### 대상 유저
> **체험 전 신규 유저** — 앱을 처음 다운로드하고 가입한 유저. 체험 레슨도 아직 하지 않은 상태.

이 PRD는 체험 전 신규 유저만 다룬다. 정규 결제 유저의 결제→예약 플로우(Squad A 담당)와는 별도 경로다.

### 현재 상태 (AS-IS) — 코드 검증 완료

**진입 경로:**

| 진입점 | 파일 | 로직 |
|---|---|---|
| 카카오/Apple 로그인 직후 | `login.handler.ts` (line 125) | `isTrialPopUpOpenCondition` 체크 → `redirection=trial` 파라미터 추가 |
| OAuth 리다이렉트 | `redirect/page.tsx` (line 46-48) | `userInfo.isTrialPopUpOpenCondition` → `redirection=trial` 세팅 |
| 홈 진입 시 | `home-redirection.tsx` (line 86-105) | `redirection=trial` 파라미터 → `/subscribes/trial`로 강제 이동 |
| 첫 홈 방문 시 | `home-redirection.tsx` (line 138-144) | `podo_auto_redirected` localStorage 없으면 자동 리다이렉트 |

**게이트 조건:**
```
isTrialPopUpOpenCondition = trialClassCompYn === 'N' && !(paymentYn === 'Y' || trialPaymentYn === 'Y')
```
→ 체험 수업 미완료 + 결제 이력 없음 + 체험 결제 이력 없음

**현재 `/subscribes/trial` 페이지 구성:**

| 순서 | 요소 | 설명 |
|---|---|---|
| 1 | 언어 선택 | EN/JP 카드 |
| 2 | 레벨 자가 선택 | 5단계 (아이콘+텍스트) |
| 3 | 결제 금액 | 30,000원 - 할인 (또는 `trial_free` ON 시 0원) |
| 4 | 결제수단 선택 | 카드/카카오페이 (`trial_free` ON 시 숨김) |
| 5 | 약관 동의 | 체크박스 |
| 6 | "체험레슨 예약하기" CTA | 결제+예약 처리 |

**핵심 기술 요소:**
- `trial_free` 피처 플래그: 이미 구현 완료. ON 시 결제 없이 0원 체험 가능
- `isFirstTrialSubscribe`: `selectedSubscribe.subPrice === 500`으로 판별
- 피처 플래그 시스템: `@podo/flags/react`의 `useFlagsCompat` 사용

### 해결하려는 문제

| 문제 | 근거 데이터 |
|---|---|
| 서비스 가치 전달 없이 바로 예약/결제 유도 | 로그인 → `/subscribes/trial` 자동 리다이렉트 |
| 레벨 매칭 부정확 | 자가 선택 5단계 — 진단 없이 유저가 직접 고름 |
| 개인화 불가 | 학습 목표 데이터 수집 없음 |
| 결제/예약 이탈 후 재진입 경로 부재 | 이탈 시 홈으로 돌아가지만 다시 예약으로 유도하는 장치 없음 |
| 1회차 이탈 82.6% | 체험 레슨 예약 후 실제 수업 미참석 |

## 2. 목표 & 성공 지표

### 목표
> 기존 `/subscribes/trial` 체험 예약 팝업을 **개인화된 온보딩 플로우**로 대체하여, 신규 유저가 서비스 가치를 인지하고 자연스럽게 **무료 체험 레슨을 예약**하게 만든다.

### 성공 지표 (KPI)

| 지표 | 현재 (AS-IS) | 목표 | 측정 방법 |
|---|---|---|---|
| **온보딩 완료율** | N/A (현재 `/subscribes/trial` 단일 페이지) | 70% 이상 | Step5 도달 / Step1 진입 |
| **무료 체험 레슨 예약률** | `/subscribes/trial` 진입 대비 예약 완료율 (기준선 측정 필요) | +20%p 개선 | 예약 완료 / 온보딩 Step1 진입 |
| **1회차 잔존율** | 17.4% (= 100% - 82.6% 이탈) | 30% 이상 | 체험 수업 참석 / 가입 |
| **추천 레벨 수용률** | N/A | 60% 이상 | 추천 레벨로 첫 수업 선택 / 레벨 추천 완료 |
| **체험→정규 전환율** | 측정 필요 | +10%p 개선 | 정규 결제 / 체험 완료 |

### Non-Goal (이번에 하지 않는 것)
- 정규 구독 결제 유도 (이 PRD의 전환 목표는 **무료 체험 예약**까지)
- 정규 구독 온보딩 (체험 → 정규 전환 플로우는 Squad A 담당)
- 기존 유저 대상 온보딩 소급 적용
- 튜터 앱 측 온보딩 변경
- 앱푸시/넛지 시퀀스 (별도 과제 "첫 7일 넛지 시퀀스"로 분리)
- 체험 예약 후 대기 시간 경험 설계 (별도 PRD로 분리 — 예습 CTA, AI 학습 등)

## 3. 유저 스토리

### 대상 유저 정의

```
이 PRD의 대상: 체험 전 신규 유저
  → 앱 첫 다운로드 → 카카오/Apple 로그인 → [온보딩] → 무료 체험 예약

별도 PRD (Squad A):  정규 결제 유저
  → 결제 완료 → 예약 원탭 연결 → 홈 진입
```

### 핵심 시나리오

**US-1: 신규 유저의 첫 로그인**
> 처음 가입한 유저로서, 로그인 후 나에게 맞는 학습 경험이 어떤 건지 알고 싶다. 그래야 이 서비스가 나한테 맞는지 판단할 수 있다.

수용 기준:
- 최초 로그인 시 온보딩 플로우가 시작된다 (기존 `/subscribes/trial` 대신)
- 학습 언어, 목표, 레벨을 설정할 수 있다
- 설정 결과에 따른 개인화 안내를 받는다
- 온보딩 마지막에 **무료 체험 레슨을 예약**할 수 있다

**US-2: 온보딩 중간 이탈 후 재진입**
> 온보딩 중 앱을 닫은 유저로서, 다시 앱을 열면 이어서 진행하고 싶다. 처음부터 다시 하기 싫다.

수용 기준:
- 진행 상태가 서버에 저장된다
- 재진입 시 마지막 완료 단계 다음부터 재개된다

**US-3: 2회차 이후 로그인**
> 온보딩을 완료한 유저로서, 다시 로그인하면 바로 홈 화면에 들어가고 싶다.

수용 기준:
- `onboarding_completed = true`이면 온보딩을 건너뛴다
- 홈 화면으로 직행한다

**US-4: 체험 예약 건너뛴 유저**
> 온보딩은 했지만 체험 레슨을 예약하지 않은 유저로서, 홈에서 쉽게 예약할 수 있었으면 좋겠다.

수용 기준:
- 홈 화면 상단에 "무료 체험 레슨 예약하기" 배너가 노출된다 (기존 `TrialBanner` 컴포넌트 활용)
- 배너 클릭 시 Step 5(체험 예약 화면)로 이동한다

## 5. 온보딩 플로우 상세

> 상세 화면 설계, 단계별 구성, 트래킹 이벤트 정의는 [[온보딩 플로우 설계 초안]] 참조.

### AS-IS → TO-BE 매핑

```
AS-IS (/subscribes/trial)              TO-BE (온보딩 플로우)
─────────────────────────              ────────────────────────
(없음 — 바로 예약 페이지)        →      [Intro] 앱 소개 2~3화면           ★ 신규
언어 선택 (EN/JP 카드)          →      [Step 1] 환영 + 학습 언어 선택
                                       [Step 2] 학습 목표 설정           ★ 신규
레벨 자가 선택 (5단계)          →      [Step 3] 시작 레벨 추천 (5문항)  ★ 개선
                                       [Step 4] 맞춤 결과 + 가치 안내     ★ 신규
결제 + 예약 CTA                →      [Step 5] 무료 체험 레슨 예약       ★ 간소화
                                              + 예약 완료 후 CTA 슬롯    ★ 신규
```

### 플로우 요약

```
카카오/Apple 로그인
  → [Intro] 앱 소개 — 2~3화면 스와이프           (10~15초)
  → [Step 1] 환영 + 학습 언어 선택              (15초)
  → [Step 2] 학습 목표 설정                      (20초)
  → [Step 3] 시작 레벨 추천 — 5문항            (60초)
  → [Step 4] 추천 결과 + 서비스 가치 안내         (20초)
  → [Step 5] 무료 체험 레슨 예약                 (15초)
  → 예약 완료 화면 (+ 후속 경험 CTA 슬롯)
  → 홈 진입
```

### 단계별 목적 & 핵심 판단

| Step | 목적 | 핵심 판단 | AS-IS 대응 |
|---|---|---|---|
| Intro. 앱 소개 | 브랜드 첫인상, 서비스 가치 인지, 온보딩 예고 | 2~3화면 스와이프. 마지막 화면에서 "시작하기" CTA | **신규** — 현재 없음 |
| 1. 환영 + 언어 선택 | 첫인상, 개인화 시작 | 카드 탭 시 자동 진행 (버튼 없음) | 기존 언어 선택 카드 대체 |
| 2. 학습 목표 설정 | 동기부여, 커밋먼트 | 복수 선택 가능 (최대 2개) | **신규** — 현재 없음 |
| 3. 시작 레벨 추천 | 추천 시작 레벨 제시 | 기존 퀴즈 인프라 재활용 5문항. **잠금이 아닌 추천** — 유저는 모든 레벨/주제 자유 선택 가능 | 기존 5단계 자가 선택 대체 |
| 4. 추천 결과 + 가치 안내 | 전환 동기 극대화 | ① 추천 레벨 + ② 자유 선택 안내 + ③ "첫 수업 후 AI가 정밀 분석" 예고 | **신규** — 현재 없음 |
| 5. 무료 체험 예약 | 체험 레슨 예약 | 기존 예약 UI 재활용 (날짜 14일 스크롤 + 시간 그리드 15:30~23:30). 확인 바텀시트로 2차 확인. `trial_free` ON: 결제 없이 바로 예약. OFF: 결제 포함 | 기존 예약 화면 UI 재활용 |

### Step 3: 시작 레벨 추천 — 설계 원칙

> **레벨 추천은 잠금이 아니라 출발점이다.**

| 원칙 | 설명 |
|---|---|
| **추천, 제한 아님** | 진단 결과는 "추천 시작 레벨"이며, 유저는 이후 어떤 레벨/주제든 자유롭게 수업 선택 가능 |
| **기존 인프라 재활용** | `/api/v1/diagnosis/getQuestions`, `checkAnswer` API의 문제 타입(MULTI_CHOICE_WORD, BLANK_FILLING 등) 재활용 |
| **정밀 레벨링은 첫 수업 후** | 온보딩 진단은 대략적 추천. 정확한 레벨 분석은 첫 체험 수업 후 AI 리포트(FLUENCY/VOCAB/GRAMMAR/COMPLEXITY)에 위임 |
| **문항 수** | 5문항 확정. 신뢰감 확보와 이탈 방지의 균형점 (60초 내 완료 가능) |

```
온보딩 레벨 추천 (Step 3)  →  "시작 레벨 추천" (기본값 세팅)
                                ↓
실제 수업 선택              →  유저가 원하는 주제/난이도 자유 선택 가능
                                ↓
첫 수업 후 AI 리포트        →  FLUENCY/VOCAB/GRAMMAR/COMPLEXITY 기반 정밀 레벨 보정
```

### Step 4: 추천 결과 + 가치 안내 — 메시지 구성

Step 4에서 전달할 3가지 메시지:

| # | 메시지 | 목적 |
|---|---|---|
| ① | "[이름]님은 **[레벨]**로 시작해보세요" | 개인화된 느낌 |
| ② | "물론, 다른 레벨도 자유롭게 들을 수 있어요" | 선택 폭 제한 불안 해소 |
| ③ | "첫 수업 후 AI가 실력을 분석해줘요" | 체험 수업을 들어야 할 동기 + AI 리포트 가치 예고 |

### Step 5 완료 후: 예약 완료 화면

| 항목 | 내용 |
|---|---|
| **화면** | 예약 완료 확인 (날짜/시간/튜터 정보) |
| **핵심 메시지** | "체험 레슨이 예약됐어요!" |
| **CTA 슬롯** | 후속 PRD에서 정의 (예: "수업 전에 예습해볼까요?", "AI 학습 미리 체험" 등) |
| **홈 진입 버튼** | "홈으로 가기" |

> **CTA 슬롯 안내**: 체험 예약과 실제 수업 사이 시간 갭을 채울 경험(예습, AI 학습 등)은 별도 PRD에서 설계한다. 이 화면에는 해당 CTA를 삽입할 자리만 마련해둔다.

### 건너뛰기 정책

> **Intro + Step 1~4: 건너뛰기 없음.** Intro 포함 전체 소요 시간 ~2분 15초의 라이트한 플로우이므로 건너뛰기 옵션을 제공하지 않는다.
> **Step 5만 "나중에 할게요" 제공.** 체험 예약은 유저 의사 존중. 건너뛴 유저에게는 홈 배너로 재유도.

### 바이패스 & 엣지케이스

| 케이스 | 처리 |
|---|---|
| 온보딩 완료 유저 재로그인 | 홈 직행 |
| 온보딩 중간 이탈 후 재진입 | 마지막 완료 단계 다음부터 재개 |
| `onboarding_v1` 플래그 OFF | 기존 플로우 유지 (`/subscribes/trial` 리다이렉트) |
| `trial_free` 플래그 ON + 온보딩 | Step 5에서 결제 없이 바로 예약 |
| `trial_free` 플래그 OFF + 온보딩 | Step 5에서 결제 수단 선택 UI 표시 |
| 이미 체험 결제 완료한 유저 | `isTrialPopUpOpenCondition = false` → 온보딩 미노출, 홈 직행 |
| Smart Talk만 구독 중인 유저 | `hasOnlyCharacterChatSubscriptionTicket` 체크 → 별도 처리 검토 |

## 7. A/B 테스트 계획

### 실험 설계

| 항목 | 내용 |
|---|---|
| 실험 단위 | 유저 (신규 가입자 — 체험 전 신규만 대상) |
| 할당 | 피처 플래그 기반 랜덤 배정 |
| 최소 표본 크기 | 각 그룹 200명 이상 (유의수준 5%, 검정력 80% 기준) |
| 실험 기간 | 최소 2주 (신규 가입 볼륨에 따라 조정) |

| 그룹 | 플래그 상태 | 경험 |
|---|---|---|
| Control | `onboarding_v1=OFF` | 기존 플로우 (로그인 → `/subscribes/trial`) |
| Treatment | `onboarding_v1=ON` | 풀 온보딩 (Step 1~5, 시작 레벨 추천 포함) |

> **왜 2그룹인가?** 유저가 이미 모든 레벨/주제를 자유 선택할 수 있고, 첫 수업 후 AI 리포트로 정밀 레벨링이 되므로, 레벨 추천 생략(자가선택 유지) 그룹을 별도로 둘 실익이 없다.

### 의사결정 기준

- Treatment가 Control 대비 **체험 예약률 +15%p 이상** → 전체 롤아웃
- 유의미하지 않음 → 온보딩 소요시간/단계별 이탈 분석 후 개선
