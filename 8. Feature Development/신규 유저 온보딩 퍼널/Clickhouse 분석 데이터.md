### Q8 노쇼율 시계열 분리 결론

- 2월에 `meet_connected` 이벤트가 **0건** → 3월부터 집계 시작
  - meet_connected 이벤트 트래킹이 3월에 추가됐거나
  - CTA 버그로 인해 2월에는 수업 입장 자체가 불가했을 가능성
- **ClickHouse에 React 전환 전 데이터 없음** (최초 데이터: 2025-11-04)
- **→ 순수 UX 기인 노쇼율은 백엔드 DB(레슨 기록)에서 별도 산출 필요**

## 데이터 한계 및 추가 분석 필요 사항

1. **React 전환 전 데이터 부재**: ClickHouse에 2025-11-04 이후 데이터만 존재. 순수 UX 노쇼율 산출을 위해 백엔드 DB(레슨 기록) 분석 필요
2. **meet_connected 이벤트 누락 (2월)**: 2월 데이터에 수업 연결 이벤트가 0건 — 트래킹 추가 시점 또는 CTA 버그 확인 필요
3. **체험 vs 정규 구분 한계**: booking_complete, meet_connected 등은 체험+정규 합산. 정확한 체험 전용 퍼널은 /subscribes/trial → /lessons/trial 경로 유저 필터링으로 산출
4. **시간 갭별 노쇼율**: 예약-수업 시간 갭이 길수록 노쇼가 높은지 검증하려면 예약 후 미참석 유저 데이터 필요 (현재 이벤트로는 "안 온 유저" 식별 어려움)
