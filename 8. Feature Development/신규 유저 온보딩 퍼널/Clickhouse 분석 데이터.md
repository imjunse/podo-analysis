## 데이터 한계 및 추가 분석 필요 사항

1. **meet_connected 이벤트 부재**: 2/23~3/4 기간에는 meet_connected 이벤트가 0건. 해당 이벤트 트래킹은 **3/10부터 시작**됨 → MySQL GT_CLASS 기반 체험완료(470명, 41.5%)를 사용
2. **ClickHouse 트래킹 시작일**: 실질적 트래킹은 2/23부터 (2/10~2/22는 테스트 수준 데이터). 1월 가입 유저의 행동 데이터는 존재하지 않음
3. **체험 vs 정규 구분**: booking_complete, btn-join-class, subscribes 페이지 등은 체험+정규 유저 합산. 체험 전용 행동은 `/subscribes/trial`, `/lessons/trial` 경로로 필터링
4. **체험 페이지 방문 > 가입 수**: 가입 전에 체험 페이지를 방문하거나 여러 번 재방문하는 유저 때문에 유니크 유저 수가 가입 수보다 많을 수 있음
5. **시간 갭 데이터**: booking_complete → btn-join-class 기준이며, 체험+정규 합산. 체험 전용 시간 갭은 별도 필터링 필요
