## 1. 핵심 발견: SUBSCRIBE → LUMP_SUM 대전환

### 월별 유료 결제 유형 분포 (SUBSCRIBE vs LUMP_SUM)

| 월 | SUBSCRIBE | LUMP_SUM | **SUB 비율** | 변화 |
|----|-----------|----------|-------------|------|
| 8월 | 590 | 369 | **61.5%** | baseline |
| 9월 | 530 | 429 | **55.3%** | -6.2%p |
| **10월** | **400** | **490** | **44.9%** | **역전 시작** |
| **11월** | **69** | **619** | **10.0%** | **SUB 급감 (83% 감소)** |
| 12월 | 243 | 638 | **27.6%** | 부분 회복 |
| 1월 | 254 | 782 | **24.5%** | LUMP_SUM 주도 정착 |
| 2월* | 85 | 601 | **12.4%** | 진행중 |

> **Insight: 구독(SUBSCRIBE) → 일시불(LUMP_SUM) 대전환**
> - 8월: 구독 61.5% > 일시불 38.5% → **11월: 구독 10.0% < 일시불 90.0%**
> - 이것이 이전 분석에서 발견한 "구독 매출 51% 급감"의 **직접적 원인**
> - SUBSCRIBE = 자동갱신(recurring revenue) / LUMP_SUM = 일시불(no auto-renewal)
> - **자동갱신 유저가 사라지면 → Leaky Bucket은 구조적으로 악화될 수밖에 없음**

## 3. 상품 구조 변경 타임라인

### Phase 1: 안정기 (8-9월)
- 상품명: "일본어 **스탠다드** 무제한", "영어 **스탠다드** 무제한"
- 제한 상품: "일본어 **스탠다드 월 4회**" (LESSON_COUNT_PER_MONTH=4)
- SUBSCRIBE가 주력 (61.5%)
- 상품 다양성: 월 35+ 상품 (5명 이상 가입)

### Phase 2: 전환기 (10-11월) — **위기의 핵심**
- **10월**: 가격 +2,000원 인상 + SUB_TYPE 버전 V2로 전환
  - SUB 비율 역전: SUBSCRIBE < LUMP_SUM
  - 상품 다양성 유지 (39개 상품)
- **11월**: SUBSCRIBE 거의 전멸 (69건, 전월 대비 -83%)
  - LUMP_SUM 619건으로 급증
  - **SUB 상품이 신규 등록에서 사실상 소멸** → 일시불만 판매
  - 상품 다양성 급감 (24개 → Top 5 집중)
  - PROMOTION_TYPE 시스템 변경 (11월 6일경 추정, 컬럼 추가)

### Phase 3: 재편기 (12-1월)
- **12월**: 이벤트 상품 대량 투입
  - "더블팩 스탠다드 무제한 **(Event)**" — 70명 가입 (월 Top 1)
  - "페이백 영어 스탠다드 무제한", "페이백 더블팩 스탠다드 무제한"
  - 새 브랜드명 등장: "일본어 **무제한 12개월**", "영어 **무제한 12개월**"
  - **라이트** 상품 첫 등장: "영어 라이트 12개월" (월8회, 59,000원), "일본어 라이트 3개월" (월8회, 126,000원)
- **1월**: 신체계 정착
  - 상품명 통일: "일본어 **무제한** 3개월" (스탠다드 제거)
  - "**라이트**" = 월 8회 (기존 "월 4회"에서 업그레이드)

## 4. 상품 변경이 리텐션에 미친 영향 — 인과관계 분석

### 직접 인과 (Confirmed)

| 변경 | 리텐션 영향 | 증거 |
|------|-----------|------|
| SUBSCRIBE → LUMP_SUM 전환 | **자동갱신 소멸 → 수동 이탈 급증** | 이탈 유형 중 Finished(자연만료) 65-73% 차지 |
| 11월 SUB 69건 (전월 400 대비 -83%) | **Net Flow 최악 -191** | 유입 773 (최저) + 이탈 964 |
| LUMP_SUM = no auto-renewal | **구독 매출 51% 급감** (매출 분석) | Recurring revenue 기반 붕괴 |

### 간접 인과 (Probable)

| 변경 | 리텐션 영향 | 추정 근거 |
|------|-----------|----------|
| +2,000원 가격 인상 (10월) | 신규 가입 감소 (1,019→973) | 가격 민감도 + 전환율 하락 |
| 상품 라인업 공백 (11월) | 유입 급감 773 (최저) | 판매 가능 상품 부족 |
| "스탠다드 → 무제한/라이트" 리브랜딩 | 기존 유저 혼란 가능 | 갱신 시 상품 매칭 실패 가능성 |
| 12월 이벤트 상품 투입 | Net Flow 개선 -191→-91 | 이벤트 효과 but 일시적 |

### 구조적 문제 (Structural)

```
SUBSCRIBE(자동갱신) 중심 → LUMP_SUM(일시불) 중심
         ↓
만료 시 자동으로 결제됨 → 만료 시 직접 재구매해야 함
         ↓
이탈 = 적극적 해지 필요 → 이탈 = 아무것도 안 하면 자동
         ↓
기본값: 유지 → 기본값: 이탈
         ↓
Leaky Bucket 관리 가능 → Leaky Bucket 구조적 악화
```

## 6. 시사점 및 제언

### 즉시 검증 필요
1. **LUMP_SUM 만료 유저 재구매율**: 자동갱신 없이 만료된 유저가 얼마나 다시 사는지?
2. **SUBSCRIBE 유저 잔존 현황**: 기존 SUB 유저가 아직 얼마나 남아있고, 갱신되고 있는지?
3. **11월 SUB 69건의 정체**: 신규 판매 중단인지, 시스템 변경인지 확인 필요

### 전략적 고려
- LUMP_SUM 전환이 **의도적 결정**이었다면 → 만료 전 리텐션 캠페인이 필수 (push, email, 할인 오퍼)
- 현재 "기본값 = 이탈" 구조 → **만료 7일/3일/1일 전 재구매 유도** 자동화 시급
- 라이트(월8회) 상품은 이탈 위험군의 다운그레이드 옵션으로 활용 가능

## 사용 SQL 쿼리

### Q1: 월별 PAYMENT_TYPE 분포
```sql
SELECT
  DATE_FORMAT(gsm.START_DATE, '%Y-%m') AS month,
  gs.PAYMENT_TYPE,
  COUNT(*) AS cnt
FROM GT_SUBSCRIBE_MAPP gsm
JOIN GT_SUBSCRIBE gs ON gsm.SUBSCRIBE_ID = gs.ID
WHERE gsm.START_DATE >= '2025-08-01'
  AND gsm.START_DATE < '2026-03-01'
GROUP BY DATE_FORMAT(gsm.START_DATE, '%Y-%m'), gs.PAYMENT_TYPE
ORDER BY month, gs.PAYMENT_TYPE
```

### Q2: 월별 상품별 가입자 수 (Top 상품)
```sql
SELECT
  DATE_FORMAT(gsm.START_DATE, '%Y-%m') AS month,
  gs.SUB_NAME, gs.SUB_TYPE, gs.PAYMENT_TYPE,
  gs.SUB_PRICE, gs.LESSON_COUNT_PER_MONTH, gs.LESSON_TIME,
  COUNT(*) AS new_subs
FROM GT_SUBSCRIBE_MAPP gsm
JOIN GT_SUBSCRIBE gs ON gsm.SUBSCRIBE_ID = gs.ID
WHERE gsm.START_DATE >= '2025-08-01'
  AND gsm.START_DATE < '2026-02-01'
  AND gs.PAYMENT_TYPE IN ('SUBSCRIBE', 'LUMP_SUM')
GROUP BY month, gs.SUB_NAME, gs.SUB_TYPE, gs.PAYMENT_TYPE,
         gs.SUB_PRICE, gs.LESSON_COUNT_PER_MONTH, gs.LESSON_TIME
HAVING COUNT(*) >= 5
ORDER BY month, new_subs DESC
```
