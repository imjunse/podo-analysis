### 데이터가 말하는 3대 병목

| 순위 | 병목 | 데이터 근거 | 매출 영향 |
|:---:|------|------------|----------|
| 1 | **정기전환율 급락** (40%→13%) | 리텐션 분석 - 25년 하반기 이후 3배 하락 | 정기결제액 51% 감소 (Leaky Bucket) |
| 2 | **결제 퍼널 이탈** (69% 이탈) | ClickHouse - 결제제출→확인 30.8% 전환 | 신규결제 전환율 직접 영향 |
| 3 | **1~2회차 이탈** (82~96%) | 정기결제 이탈 분석 - 1~2회차가 전체 이탈의 57% | M3 잔존 45%, M6 잔존 15% |

### 해지 사유 Top 3

- **시간 부족** 29% → 리텐션 오퍼 / 미니레슨
- **동기 저하** 14% → 온보딩 / 학습 목표 설정
- **다른 서비스 고려** 12% → 콘텐츠 차별화 / AI 기능

## 데이터 기반 스쿼드별 로드맵

### Squad A: **전환 퍼널** — 박선영

> **미션**: "돈 내려는 사람이 못 내고 나간다" 해결 **핵심 지표**: 결제 퍼널 전환율 30.8% → 50% / 정기전환율 13% → 25%

**데이터 배경**: 결제제출→확인 69% 이탈, 구독→티켓 42% 이탈, 정기전환율 3배 급락

|순서|과제|구체 내용|근거 데이터|난이도|
|:-:|---|---|---|:-:|
|1|**결제 완료→예약 원탭 연결**|결제 성공 화면에 "바로 첫 수업 예약" CTA 추가. Ticket 생성 API 응답의 ticketId로 `/booking` 직행|결제 퍼널 69% 이탈 + 1회차 이탈 82.6%|낮|
|2|**결제수단 등록 인라인화**|`/my-podo/payment-methods/register` 페이지 이동 제거 → 결제 플로우 내 바텀시트로 처리|정기구독 시 별도 페이지 이동 = 이탈 포인트. `page_subscribe_ticket_v2_enabled` 플래그 활용|중|
|3|**수강권 선택 간소화**|6종 수강권(tickets, tickets-v2, smart-talk, payback, trial 등)을 사용자 상태 기반 1~2개 추천으로 축소|구독→티켓 단계 42% 이탈. 선택지 과다가 원인|중|
|4|**동일 언어 중복 구매 허용**|PRD 기반 구현. 고객 주도 활성화 방식 + 멀티트랙 보너스 규칙 수정|PRD 작성 완료. CS 인입 발생 중|높|
|5|**정기전환율 급락 원인 대응**|25-10~11 상품/가격 변경 이력 조사 → 원인에 따라 상품 구조 또는 가격 정책 수정|정기전환율 29.4%(25-10)→7.0%(25-11). 같은 시기 무언가 변경됨|높|
|6|**체험→결제 퍼널 최적화**|체험예약→완료 전환율 72%→55%로 하락 중. 체험 리마인더, 노쇼 방지, 체험 후 즉시 결제 CTA|26-02 체험완료→결제 50.4%로 역대 최저|중|

### Squad B: **학습 경험 / DAU** — 송하림

> **미션**: "수업을 듣는 사람이 계속 듣게 만든다" **핵심 지표**: 예약 변경률 49% → 30% / AI 기능 사용률 15% → 30% / 주간 수업 횟수

**데이터 배경**: 예약 변경 49%, 과다요청 12.5%, AI 기능 15% 사용, 레슨 리뷰 6% 도달

|순서|과제|구체 내용|근거 데이터|난이도|
|:-:|---|---|---|:-:|
|1|**예약 변경 UX 개선**|예약 변경이 쉬워야 헤비유저가 이탈하지 않음. 변경 플로우 간소화 + 과다요청 경고 대신 빠른 변경 UI|변경 1,652명(49%), 과다요청 419명이 **1인당 43회** 변경. 열성 유저를 시스템이 막는 구조|중|
|2|**12시간 예약 규칙 완화/UX 개선**|피크타임(21~22시) 유저가 다음날 오전 수업 예약 구조적 불가 → 규칙 완화 or "내일 아침 수업" 퀵 예약 도입|피크 21~22시(12만 이벤트) + 해지사유 "12시간 전 예약 힘들어요" 6.3%|중|
|3|**AI 리포트/리뷰 도달 경로 개선**|수업 완료→AI 리포트 생성 자동 안내. 레슨 리뷰까지 가는 퍼널 재설계. 홈에서 "미확인 리포트" 노출|AI 리포트 15% 사용, 레슨 리뷰 6% 도달. 기능은 있으나 도달 경로가 없음|중|
|4|**예습 참여율 향상**|예습 완료 시 포인트/리워드, 예습→수업 연결 강화, 예습 리마인더 푸시|btn-prestudy 2,159명(37%) 클릭하지만 완료 데이터 부족. 예습이 수업 품질에 직결|중|
|5|**학습 진도 시각화**|주간/월간 학습 리포트, 연속 수업 스트릭, 레벨 진행도 대시보드|해지사유 "실력이 늘지 않는 것 같아요" 3.9% + 동기저하 14%. 성장 체감이 없음|중|
|6|**콘텐츠 확장 + 레슨 다양화**|중급 토픽 확장, AI Mini Lessons 도입으로 정규 수업 외 학습 경로 제공|해지사유 "모든 레슨 완료, 수업할 자료 없음" 5.2%. 장기 유저 콘텐츠 소진|높|

### Squad C: **온보딩 / 리텐션** — 이준서

> **미션**: "들어온 사람이 가치를 느끼기 전에 나간다" 해결 **핵심 지표**: 온보딩 완료율 (신규 도입) / M3 잔존율 45% → 60% / WelcomeBack 재결제율

**데이터 배경**: 온보딩 플래그 0개, 1회차 이탈 82.6%, M3 잔존 45%, WelcomeBack 단일 오퍼

|순서|과제|구체 내용|근거 데이터|난이도|
|:-:|---|---|---|:-:|
|1|**에러 메시지 한글화**|"Something went wrong" → "문제가 발생했어요". 하드코딩 영어 문자열 전수 교체|한국 사용자 대상 앱. i18n 미도입, 하드코딩 확인|낮|
|2|**신규 유저 온보딩 퍼널**|최초 로그인→레벨테스트→레벨 선택→서비스 가치 안내→첫 수업 부스터→홈 진입. 2회차부터 바이패스|온보딩 플래그 0개. 로그인 후 바로 홈 진입. 1회차 이탈 82.6%|중|
|3|**첫 7일 넛지 시퀀스**|결제 후 1일차: 예약 리마인더, 3일차: 예습 안내, 7일차: 첫 AI 리포트 안내. 앱푸시+인앱 메시지|결제→첫수업 시간 갭이 이탈 원인. 골든타임(체험완료→결제 0.8일)은 빨라졌지만 이후 이탈|중|
|4|**해지 사유 기반 맞춤 리텐션 오퍼**|`le_subscribe_mapp_history` 해지 사유 데이터 활용. 시간부족→일시정지 안내, 가격→할인, 동기저하→1:1 상담|WelcomeBack 단일 오퍼(WELCOMEBACK_30). 해지사유 827건 데이터 존재하나 미활용|중|
|5|**M3 이탈 절벽 대응**|3개월차 유저에게 선제적 리텐션 개입. 수업 빈도 감소 감지→맞춤 오퍼 트리거|M1 85%→M2 75%→**M3 45% 급락**. 3개월이 결정적 이탈 시점|중|
|6|**홈 화면 통합**|`/home`(Basic) + `/home/ai`(Smart Talk) 통합. 구독 상태 기반 동적 섹션 렌더링|두 개 홈 분리로 인한 혼란 + AI Home 강제 리디렉트. React 마이그레이션(86%)과 동기화|높|

## 스쿼드 간 시너지 포인트

|시점|연동 내용|
|---|---|
|**3M**|Squad A(결제→예약 연결) + Squad B(예약 UX) — 예약 화면 진입점 통일|
|**4M**|Squad C(온보딩) → Squad B(AI 리포트 도달) — 온보딩에서 AI 기능 첫 경험 설계|
|**4M**|Squad A(수강권 간소화) → Squad C(온보딩) — 온보딩에서 맞춤 수강권 추천 연동|
|**5M**|Squad B(예습+포인트) + Squad A(포인트 설계) — 포인트 시스템 공동 설계|
|**5M**|Squad C(맞춤 리텐션 오퍼) → Squad A(정기전환율 대응) — 이탈 방지가 곧 정기전환 개선|
|**6M**|Squad C(홈 통합) + Squad B(학습 진도 시각화) — 통합 홈에 학습 대시보드 포함|
