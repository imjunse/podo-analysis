# Squad A: 전환 퍼널 — 박선영

> **미션**: "돈 내려는 사람이 못 내고 나간다" 해결
> **핵심 KPI**: 결제 퍼널 전환율 30.8% → 50% / 정기전환율 13% → 25%

---

## 데이터 배경

- 결제제출→확인 **69% 이탈**
- 구독→티켓 단계 **42% 이탈**
- 정기전환율 **40% → 13%** 급락 (3배)
- 정기결제액 2.15억 → 1.06억 = **51% 감소**

---

## 과제 상세

### 1. 결제 완료→예약 원탭 연결

> 난이도: 낮 (퀵윈) | 시기: 3M 전반

**현상**: 결제 완료 후 예약까지의 연결이 없음. 결제 성공 화면에서 유저가 알아서 예약을 찾아가야 함.

**해결**:
- 결제 성공 화면에 "바로 첫 수업 예약" CTA 추가
- Ticket 생성 API 응답의 ticketId로 `/booking` 직행
- 신규 결제 유저에게는 예약 화면 자동 진입 (옵트아웃 가능)

**근거 데이터**:
- 결제 퍼널 69% 이탈
- 1회차 이탈 82.6%
- 결제→첫수업 시간 갭이 이탈 원인

---

### 2. 결제수단 등록 인라인화

> 난이도: 중 | 시기: 3M~4M

**현상**: 결제수단 등록이 `/my-podo/payment-methods/register` 별도 페이지로 이동. 페이지 전환 시 이탈 발생.

**해결**:
- 별도 페이지 이동 제거
- 결제 플로우 내 바텀시트로 결제수단 등록 처리
- `page_subscribe_ticket_v2_enabled` 피처 플래그 활용

**근거 데이터**:
- 정기구독 시 별도 페이지 이동 = 이탈 포인트
- 결제 퍼널에서 불필요한 네비게이션 단계

---

### 3. 수강권 선택 간소화

> 난이도: 중 | 시기: 4M

**현상**: 6종 수강권(tickets, tickets-v2, smart-talk, payback, trial 등)이 나열. 선택지 과다로 구독→티켓 단계에서 42% 이탈.

**해결**:
- 사용자 상태(신규/기존, 레벨, 학습 빈도) 기반 1~2개 추천으로 축소
- "나에게 맞는 수강권" 추천 로직 도입
- 나머지는 "전체 수강권 보기"로 접어둠

**근거 데이터**:
- 구독→티켓 단계 42% 이탈
- 선택지 과다 = 결정 마비 (Choice Overload)

---

### 4. 동일 언어 중복 구매 허용

> 난이도: 높 | 시기: 4M~5M

**현상**: 동일 언어 수강권을 이미 보유한 유저가 추가 구매 불가. CS 인입 발생.

**해결**:
- PRD 기반 구현 (이미 작성 완료)
- 고객 주도 활성화 방식
- 멀티트랙 보너스 규칙 수정

**근거 데이터**:
- PRD 작성 완료
- CS 인입 발생 중

---

### 5. 정기전환율 급락 원인 대응

> 난이도: 높 | 시기: 5M~6M

**현상**: 25-10~11 시기에 정기전환율이 29.4% → 7.0%으로 급락. 같은 시기 상품/가격 변경 이력이 있을 것으로 추정.

**해결**:
- 25-10~11 상품/가격 변경 이력 조사 (상품-가격 변경 이력 분석 참조)
- 원인에 따라 상품 구조 또는 가격 정책 수정
- A/B 테스트로 효과 검증

**근거 데이터**:
- 정기전환율 29.4%(25-10)→7.0%(25-11) — 같은 시기 무언가 변경됨
- 정기결제액 51% 감소의 근본 원인

---

### 6. 체험→결제 퍼널 최적화

> 난이도: 중 | 시기: 6M

**현상**: 체험예약→완료 전환율이 72% → 55%로 하락 추세. 체험완료→결제도 50.4%로 역대 최저.

**해결**:
- 체험 리마인더 강화 (앱푸시 + SMS)
- 노쇼 방지 메커니즘
- 체험 완료 후 즉시 결제 CTA (시간제한 혜택)

**근거 데이터**:
- 26-02 체험완료→결제 50.4% (역대 최저)
- 체험예약→완료 55.1%로 하락 중
