# Squad C: 온보딩 / 리텐션 — 이준서

> **미션**: "들어온 사람이 가치를 느끼기 전에 나간다" 해결
> **핵심 KPI**: 온보딩 완료율 (신규 도입) / M3 잔존율 45% → 60% / WelcomeBack 재결제율

---

## 데이터 배경

- 온보딩 플래그 **0개** (온보딩 플로우 자체가 없음)
- 1회차 이탈 **82.6%**, 1~2회차가 전체 이탈의 **57%**
- M1 85% → M2 75% → **M3 45% 급락** (3개월 절벽)
- WelcomeBack 단일 오퍼 (WELCOMEBACK_30만 존재)
- 해지사유 827건 데이터 존재하나 미활용

---

## 과제 상세

### 1. 에러 메시지 한글화

> 난이도: 낮 (퀵윈) | 시기: 3M 전반

**현상**: "Something went wrong" 등 영어 에러 메시지가 한국 사용자에게 노출. i18n 미도입, 하드코딩 확인.

**해결**:
- 하드코딩 영어 문자열 전수 조사 및 교체
- "Something went wrong" → "문제가 발생했어요. 다시 시도해주세요."
- 에러 메시지 표준화 (에러 코드별 한글 메시지 매핑)

**근거 데이터**:
- 한국 사용자 대상 앱
- 하드코딩 영어 문자열 확인됨
- 난이도 매우 낮아 퀵윈으로 적합

---

### 2. 신규 유저 온보딩 퍼널

> 난이도: 중 | 시기: 3M~4M | PRD: [[PRD — 신규 유저 온보딩 퍼널]] (v0.4)

**현상**: 온보딩 플로우가 존재하지 않음. 최초 로그인 시 `/subscribes/trial`(체험 예약 팝업)로 자동 리다이렉트되어, 서비스 가치 전달 없이 바로 결제/예약을 요구. 레벨은 자가 선택(5단계)으로 매칭 부정확. 결과적으로 1회차 이탈 82.6%.

**해결 — 5단계 온보딩 플로우**:
```
카카오/Apple 로그인
  → [Step 1] 환영 + 학습 언어 선택              (15초)
  → [Step 2] 학습 목표 설정                      (20초)
  → [Step 3] 시작 레벨 추천 — 3~5문항            (60초)
  → [Step 4] 추천 결과 + 서비스 가치 안내         (20초)
  → [Step 5] 무료 체험 레슨 예약                 (15초)
  → 예약 완료 화면 (+ 후속 경험 CTA 슬롯)
  → 홈 진입
```
- 기존 `/subscribes/trial` 체험 예약 팝업을 **개인화된 온보딩 플로우**로 대체
- Step 3: 기존 퀴즈 인프라 재활용한 **시작 레벨 추천** (잠금이 아닌 추천 — 유저는 모든 레벨/주제 자유 선택 가능)
- Step 5: `trial_free` 플래그 ON 시 결제 없이 바로 예약 / OFF 시 결제 포함
- **전환 목표는 무료 체험 예약까지** (정규 결제 유도는 Squad A 담당)
- 건너뛰기 정책: Step 1~4 없음 (전체 ~2분으로 라이트), Step 5만 "나중에 할게요" 제공
- 2회차부터 바이패스 (`onboarding_completed = true`)
- 중간 이탈 시 서버에 상태 저장 → 재진입 시 이어서 진행
- 예약 완료 화면에 **후속 CTA 슬롯** (대기 시간 경험은 별도 PRD로 분리)

**A/B 테스트**:
- `onboarding_v1` 단일 피처 플래그로 전체 ON/OFF 제어
- 2그룹: Control(기존 플로우) vs Treatment(풀 온보딩)
- 의사결정 기준: Treatment가 체험 예약률 +15%p 이상 → 전체 롤아웃

**성공 지표 (KPI)**:
| 지표 | 현재 | 목표 |
|---|---|---|
| 온보딩 완료율 | N/A | 70%+ |
| 무료 체험 예약률 | 기준선 측정 필요 | +20%p |
| 1회차 잔존율 | 17.4% | 30%+ |
| 추천 레벨 수용률 | N/A | 60%+ |
| 체험→정규 전환율 | 측정 필요 | +10%p |

**근거 데이터**:
- 온보딩 관련 플래그 0개 — 플로우 자체가 없음
- 1회차 이탈 82.6%
- 현재 `/subscribes/trial` 자동 리다이렉트가 서비스 가치 전달 없이 결제/예약 요구 (코드 검증 완료)
- 레벨 자가 선택(5단계)으로 적정 레벨 매칭 실패
- 이탈 후 재진입 경로 부재

### 3. 첫 7일 넛지 시퀀스

> 난이도: 중 | 시기: 4M

**현상**: 결제 후 첫 수업까지의 시간 갭에서 이탈 발생. 골든타임(체험완료→결제 0.8일)은 빨라졌지만, 결제 후 학습 습관이 형성되기 전에 이탈.

**해결**:
- Day 1: 예약 리마인더 ("첫 수업을 예약해보세요")
- Day 3: 예습 안내 ("수업 전에 이것만 봐보세요")
- Day 5: 학습 진도 알림 ("벌써 N시간 학습했어요!")
- Day 7: 첫 AI 리포트 안내 ("수업 리포트가 도착했어요")
- 앱푸시 + 인앱 메시지 조합

**근거 데이터**:
- 결제→첫수업 시간 갭이 이탈 원인
- 첫 7일이 습관 형성의 골든타임

---

### 4. 해지 사유 기반 맞춤 리텐션 오퍼

> 난이도: 중 | 시기: 4M~5M

**현상**: WelcomeBack 오퍼가 `WELCOMEBACK_30` 하나뿐. 해지 사유가 다양한데(시간부족 29%, 가격 10%, 동기저하 14%) 모두에게 같은 오퍼를 제공.

**해결**:
- `le_subscribe_mapp_history` 해지 사유 데이터 활용
- 시간부족 → 일시정지 안내 + 미니레슨 추천
- 가격 → 할인 오퍼
- 동기저하 → 1:1 상담 + 학습 목표 재설정
- 다른 서비스 → 차별화 포인트 강조

**근거 데이터**:
- WelcomeBack 단일 오퍼 (WELCOMEBACK_30)
- 해지사유 827건 데이터 존재하나 미활용
- 사유별 맞춤 대응 시 재결제율 상승 기대

---

### 5. M3 이탈 절벽 대응

> 난이도: 중 | 시기: 5M

**현상**: M1 85% → M2 75% → M3 45%로 급격한 이탈. 3개월차가 결정적 이탈 시점. 학습 습관이 형성되지 않으면 이 시점에서 이탈.

**해결**:
- 3개월차 유저 선제적 리텐션 개입
- 수업 빈도 감소 감지 → 자동 트리거
- "3개월 학습 리포트" 제공 (성장 체감)
- 장기 유저 전환 인센티브 (6개월 플랜 할인 등)
- 튜터 매칭 만족도 체크 → 불만 시 튜터 변경 안내

**근거 데이터**:
- M1 85% → M2 75% → M3 45% (급락)
- 3개월이 결정적 이탈 시점
- 이 시점 유저를 잡으면 M6까지 잔존율 크게 개선

---

### 6. 홈 화면 통합

> 난이도: 높 (장기) | 시기: 6M

**현상**: `/home`(Basic)과 `/home/ai`(Smart Talk)가 분리. 구독 상태에 따라 강제 리디렉트. 유저 혼란 유발. React 마이그레이션(86%)과 동기화 필요.

**해결**:
- 두 개 홈을 하나로 통합
- 구독 상태 기반 동적 섹션 렌더링
- 학습 대시보드(Squad B)와 연계
- Nuxt→React 마이그레이션 완료 후 진행 (현재 86%)

**근거 데이터**:
- 두 개 홈 분리로 인한 혼란
- AI Home 강제 리디렉트 문제
- React 마이그레이션과 동기화하면 효율적
