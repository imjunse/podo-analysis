
## Podo 사용자 플로우 개선 분석

### 1. 홈 화면 분리로 인한 혼란 (우선순위: 높음)

**AS-IS**: Basic Home(`/home`)과 AI Home(`/home/ai`)이 완전히 분리되어 있고, 전환 시 **전체 페이지 리로드** 발생

**문제점**:

- 사용자가 두 개의 홈이 있다는 것을 인지하기 어려움
- 모드 전환 시 리로드로 UX 끊김
- AI 전용 수강권만 있으면 AI Home으로 강제 리디렉션 → 정규 수업 탐색 접근 차단

**TO-BE**: `podo-workflow-improved.md`에 이미 제안된 **통합 홈** 방안 — 수강권 상태에 따라 섹션을 동적으로 렌더링

**기대 효과**: 수업 탐색 접근성 향상, 전환 이탈 감소

---

### 2. 결제까지의 단계가 너무 많음 (우선순위: 높음)

**AS-IS**: 홈 → 수업 탐색(`/subscribes`) → 수강권 목록 → 수강권 V2 **또는** 스마트톡 **또는** 페이백 → 결제 페이지 → 결제 완료 (최소 4~5단계)

**문제점**:

- 수강권 종류가 너무 많음 (tickets, tickets-v2, smart-talk, payback, trial, trial/smart-talk)
- 사용자가 어떤 수강권을 선택해야 할지 판단하기 어려움
- 결제 페이지 자체도 7개 섹션으로 복잡

**TO-BE**: 수강권 추천 시스템 도입 — 사용자 학습 패턴 기반으로 1~2개 추천 후 원탭 결제

**기대 효과**: 결제 전환율 향상, 결제 이탈 감소

---

### 3. 결제 완료 후 예약까지 자동 연결 부재 (우선순위: 높음)

**AS-IS**: 결제 완료 후 "예습하러 가기" / "홈으로" 버튼만 제공. 예약은 별도 탭에서 직접 해야 함

**문제점**:

- 결제 → 예약이 핵심 여정인데 자동 연결이 없음
- 신규 사용자는 결제 후 "다음에 뭘 해야 하지?" 상태에 놓임
- 첫 수업까지의 시간 갭이 이탈 원인

**TO-BE**: 결제 완료 → "바로 첫 수업 예약하기" CTA를 primary로 배치, 예약 화면으로 자동 연결

**기대 효과**: 첫 수업 도달률 대폭 향상

---

### 4. 온보딩 플로우 부재 (우선순위: 높음)

**AS-IS**: 로그인 후 바로 홈 화면 진입. 레벨 테스트, 첫 수업 부스터는 홈 내부 다이얼로그로만 존재

**문제점**:

- 신규 사용자가 앱의 가치를 체험하기 전에 복잡한 홈 화면에 노출
- 레벨 테스트가 선택사항이라 스킵되기 쉬움
- `podo-workflow-improved.md`에서도 이 문제를 인식하고 온보딩 분리를 제안

**TO-BE**: 신규 유저 전용 온보딩 (레벨 테스트 → 맞춤 수강권 추천 → 체험 수업 바로 시작)

**기대 효과**: 신규 사용자 활성화율 향상

---

### 5. 해지 플로우의 만류 혜택이 정적 (우선순위: 중간)

**AS-IS**: 해지 사유에 따라 고정된 4가지 혜택 제안 (할인, 일시정지, 컨설팅, 무료연장)

**문제점**:

- 사용자별 맞춤 혜택이 아님
- 사용 패턴(수업 횟수, 만족도, 잔여 티켓)을 반영하지 않음
- PRD에 있는 리텐션 프로모션과 연계되지 않음

**TO-BE**: 사용자 데이터 기반 동적 만류 혜택 — 수업 빈도 낮은 유저에겐 일시정지, 가격 민감 유저에겐 할인

**기대 효과**: 해지 만류 성공률 향상

---

### 6. 수강권 중복 구매 불가 (우선순위: 중간)

**AS-IS**: 동일 언어 수강권 중복 보유 불가 → CS 인입 발생

**문제점**: PRD에 이미 상세히 기술됨. 이벤트/지원금 활용 시점에 구매 불가

**TO-BE**: PRD대로 중복 구매 허용 + 기존 회원 전용 결제 페이지 추가

**기대 효과**: 리텐션 향상, CS 부담 감소

---

### 7. 에러 메시지가 영어 (우선순위: 낮음)

**AS-IS**: 홈 에러 화면이 "Something went wrong" / "An error occurred..." 영어 메시지

**문제점**: 한국 사용자 대상 앱인데 에러 메시지가 영어

**TO-BE**: 한국어 에러 메시지 + 구체적 안내

**기대 효과**: 사용자 혼란 감소

---
