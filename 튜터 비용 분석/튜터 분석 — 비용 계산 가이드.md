# 튜터 비용 계산 가이드

> 관련: [[튜터 비용 및 공급 분석 (2026-02)]]

이 노트는 튜터 비용 관련 지표를 계산할 때 주의할 점을 정리한 것이다. 한 번 틀렸던 실수를 다시 하지 않기 위해 기록.

---

## 평균 수업단가 (Cost per Lesson)

### 올바른 공식

```
평균 수업단가 = Total Tutor Cost ÷ (COMPLETED + STUDENT_NO_SHOW + STUDENT_CANCELLED_WITHIN_12HOURS)
```

### 틀린 공식 (흔한 실수)

```
❌ Total Tutor Cost ÷ COMPLETED만
```

이렇게 하면 ₩3,324가 나온다. 실제는 **₩3,152**.

### 왜 노쇼/12h취소를 분모에 넣나?

튜터에게는 세 가지 모두 비용이 발생한다:
- **완료(COMPLETED)**: 당연히 비용 발생
- **노쇼(STUDENT_NO_SHOW)**: 학생이 안 왔어도 튜터는 대기. 비용 지급됨
- **12시간 내 취소**: 취소가 너무 늦어서 튜터 시간 확보 불가. 비용 지급됨

즉, **비용이 발생하는 수업 = 분모**.

### 검증 (2025-09 기준)

| | 수치 |
|---|---|
| Total Tutor Cost | ₩101,943,050 |
| COMPLETED | 20,097 |
| STUDENT_NO_SHOW | 1,478 |
| 12H_CANCEL | 271 |
| 분모 합계 | 21,846 |
| **수업단가** | **₩4,669** |

> 참고: ₩3,152는 특정 필터(영어만/특정 기간) 조건에서의 값. 전체 조건에서는 시기별로 다름.

## DB 스키마 참고

### GT_CLASS 주요 컬럼

- `CLASS_TYPE`: 'PODO' 필터
- `CLASS_STATE`: FINISH, FINISHED, FINISH_PRESTUDY (완료) / NOSHOW_PRESTUDY (노쇼) / CANCEL (취소)
- `CLASS_DATE`: 수업 날짜
- `TEACHER_USER_ID`: 튜터 ID (→ GT_TUTOR.ID)
- `TUTOR_PRICE_PER_CLASS`: 해당 수업의 튜터 단가
- `CLASS_EVAL1`, `CLASS_EVAL2`, `CLASS_EVAL3`: 수업 평가 (미분석)

### GT_TUTOR 주요 컬럼

- `TUTOR_TYPE`: '영어', '일본어' 등
- `PODO_TUTOR_LEVEL_ID`: → le_tutor_level.id

### le_tutor_level 주요 컬럼

- `name`: 레벨명 (Level 0 (1800), Level 2 (2100), ...)
- `price`: 25분 수업 단가
- `15_price`: 15분 수업 단가

> 참고: `CLASS_STATUS` 컬럼은 GT_CLASS에 존재하지 않음. `CLASS_STATE`를 써야 함. Metabase 카드(932 등)에서 보이는 COMPLETED / STUDENT_NO_SHOW 분류는 카드 레벨에서 CLASS_STATE를 가공한 것.
