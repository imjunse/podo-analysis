# 방안 4: 개인화 이메일 자동화

> 상위 문서: [[문제 분석 및 해결 방안]]
> 난이도: 중 | 소요: ~2주 | 기대 효과: 중 | 우선순위: Tier 2 - 4순위
> 포지셔닝: Level Up 커리큘럼 출시(4~5월)까지의 **브릿지 솔루션**

---

## 1. 핵심 가설

> "수동 개인화 이메일이 50% 성공률을 보였다면, 자동화해도 30-40% 수준은 유지할 수 있다. 풀부킹 발생 즉시 트리거하면 수동보다 타이밍이 빨라 성공률이 더 높을 수도 있다."

**수동 테스트 결과 (기존 데이터):**
- 성공률: ~50% (이메일 보낸 튜터 중 스케줄을 연 비율)
- 효과: 1-2시간 추가 오픈 → 해당 시간대 풀부킹 해소

---

## 2. 자동화 시스템 설계

### 2-1. 트리거 조건

```
매일 특정 시점에 다음 조건 체크:

IF (특정 시간대의 바이링구얼 튜터 가용 슬롯 == 0)
AND (해당 시간대 직전/직후에 스케줄이 끝나는 튜터가 존재)
AND (해당 튜터에게 이번 주 아직 이메일을 보내지 않음)
THEN → 이메일 발송
```

### 2-2. 트리거 타이밍 옵션

| 옵션 | 시점 | 장점 | 단점 |
|------|------|------|------|
| A. 고정 시간 (17시) | 매일 17:00 | 구현 간단, 예측 가능 | 17시에 풀부킹 아니면 불필요 발송 |
| B. 실시간 트리거 | 풀부킹 발생 즉시 | 타이밍 최적, 정확도 높음 | 개발 공수 높음 |
| C. 하이브리드 | D-1 21시 (사전 예측) + 당일 실시간 | 양쪽 장점 | 복잡도 높음 |

**권장: 옵션 A (MVP) → 옵션 B로 진화**

### 2-3. 튜터 선정 로직

풀부킹 시간대 기준, 이메일 대상 튜터 2명 선정:

```
1. 해당 시간대 직전에 수업이 끝나는 바이링구얼 튜터 조회
   (예: 21시 풀부킹 → 20:25에 수업 끝나는 튜터)

2. 필터:
   - 이번 주 이메일 발송 이력 없음 (주 1회 제한)
   - 최근 4주 이내 유사 이메일에 응답(스케줄 오픈)한 이력 있으면 가점
   - 평균 응답률 기준 상위 순

3. 상위 2명 선정

4. 예외:
   - 튜터가 명시적으로 opt-out한 경우 제외
   - 연속 3회 미응답 튜터는 4주간 제외 (이메일 피로 방지)
```

---

## 3. 이메일 콘텐츠 설계

### 3-1. 핵심 원칙

수동 테스트에서 50% 성공률을 만든 핵심 요소:
- **담당자가 직접 보낸 것처럼** 느껴지는 자연스러운 톤
- **구체적 상황 설명** ("오늘 21시에 학생이 기다리고 있다")
- **부담 없는 요청** ("1시간만 더")

자동화에서도 이 요소를 유지해야 함.

### 3-2. 이메일 템플릿 (일본어)

**템플릿 A: 기본형**

```
件名: [튜터이름]先生、今晩お時間ありますか？

[튜터이름]先生、

いつもレッスンありがとうございます。

今日の[시간]時台に、レッスンを受けたい生徒さんがいるのですが、
現在空きがない状態です。

もし可能でしたら、あと1時間だけスケジュールを開けていただけないでしょうか？

お忙しいところ恐れ入りますが、ご検討いただけると嬉しいです。

[담당자 이름]
Podo チューターチーム
```

**템플릿 B: 관계 강화형 (이전에 응답한 적 있는 튜터)**

```
件名: [튜터이름]先生にお願いがあります 🙏

[튜터이름]先生、

前回スケジュールを追加してくださって、本当に助かりました！
生徒さんもとても喜んでいました。

実は今日も[시간]時にレッスンの空きがなくて困っています。
[N]人の生徒さんが待っている状況です。

もしお時間があれば、ご協力いただけると大変助かります。

[담당자 이름]
```

**템플릿 C: 성과 공유형**

```
件名: [튜터이름]先生の生徒さんからのメッセージ

[튜터이름]先生、

先生のレッスンを受けた[학생이름]さんが
「[실제 피드백 발췌]」とフィードバックを残してくれました。

今日の[시간]時にもレッスンを受けたい生徒さんがいるのですが、
現在空きがありません。

1時間だけスケジュールを追加していただくことは可能でしょうか？

[담당자 이름]
```

### 3-3. 템플릿 로테이션

같은 튜터에게 매번 같은 톤의 이메일이 가면 패턴 파악 → 효과 소멸.

- 3개 템플릿을 라운드 로빈으로 로테이션
- 튜터별로 마지막 사용 템플릿 기록 → 다음엔 다른 템플릿
- 2-3개월마다 새 템플릿 추가/교체

### 3-4. 개인화 변수

| 변수 | 소스 | 예시 |
|------|------|------|
| 튜터 이름 | 튜터 DB | 田中 |
| 시간대 | 풀부킹 데이터 | 21시 |
| 대기 학생 수 | 웨이팅리스트 | 3명 |
| 담당자 이름 | 고정 or 랜덤 | Andrew, Scott |
| 학생 피드백 | 수업 리뷰 DB | "발음이 좋아졌어요" |
| 이전 협력 언급 | 이메일 이력 | "전에 도와주셔서 감사" |

## 4. 효과 소멸 대응 전략

### 4-1. 예상 수명: 2-3개월

- 튜터들이 패턴을 인식하면 반응률 하락
- 특히 "매주 같은 요일, 같은 시간에 이메일이 온다"면 빠르게 파악

### 4-2. 수명 연장 전략

| 전략 | 효과 | 구현 복잡도 |
|------|------|-----------|
| 템플릿 로테이션 | 중 | 낮음 |
| 발송 요일/시간 랜덤화 | 중 | 낮음 |
| 실제 담당자 이름 로테이션 | 낮 | 매우 낮음 |
| 이메일 외 채널 (LINE, 카카오) 교대 사용 | 높 | 중 |
| 진짜 수동 이메일과 자동 이메일 혼합 | 높 | 낮음 |

### 4-3. Level Up 출시와의 연계

- 이 방안의 유효 기간(2-3개월)은 Level Up 출시 예상(4-5월)과 겹침
- **명시적 브릿지 솔루션으로 포지셔닝:**
  - 3월 말 시작 → 5월 Level Up 출시 → 비바이링구얼 튜터 확대로 자연 전환
  - 효과 소멸 시점과 근본 해결책 출시 시점이 맞물려 전환이 자연스러움

## 5. 윤리적/문화적 고려

### 5-1. 자동화인데 수동처럼 보내는 것은 괜찮은가?

**고려할 점:**
- 튜터와의 신뢰 관계. 자동 이메일임이 밝혀지면 신뢰 저하 가능
- 그러나 내용 자체는 사실 (실제로 학생이 기다리고, 빈자리가 필요함)
- "자동화된 커뮤니케이션"이 아닌 "시스템이 도와주는 커뮤니케이션"으로 프레이밍

**권장:**
- 이메일 하단에 작은 텍스트로 "이 메시지는 예약 상황에 따라 자동으로 발송됩니다"를 넣을지 팀 내 논의 필요
- 주 1회 제한을 엄수하여 스팸 느낌 방지

### 5-2. 일본 문화 맥락

- 일본에서 직장(본업) 외 부업에 대한 "요청"은 부담으로 느낄 수 있음
- **반드시 "お忙しいところ恐れ入りますが"(바쁘신데 죄송합니다) 류의 경어 사용**
- 요청이 아닌 "ご検討いただけると嬉しいです"(검토해주시면 기쁘겠습니다) 수준의 정중한 표현
- 절대 "~してください"(~해주세요)처럼 직접적 요청 금지

## 6. 구현 범위

### Phase 1 (MVP, ~1주)
- [ ] 풀부킹 시간대 감지 크론잡 (매일 17시)
- [ ] 대상 튜터 선정 로직 (직전 시간 수업 종료 + 주 1회 제한)
- [ ] 이메일 템플릿 3종 (일본어 네이티브 검수 필수)
- [ ] 발송 + 로깅

### Phase 2 (~2주)
- [ ] 실시간 트리거 (풀부킹 발생 즉시)
- [ ] 웨이팅리스트 데이터 연동 ("N명 대기 중" 변수)
- [ ] 응답률 대시보드
- [ ] A/B 테스트: 템플릿별 응답률 비교

## 7. 성공/실패 판단 기준

### 2주 시점

| 지표 | 목표 | 실패 기준 |
|------|------|----------|
| 이메일 응답률 (스케줄 오픈) | 30%+ | 15% 미만 |
| 주간 추가 수업 수 | +50회 이상 | +10회 미만 |
| 응답률 하락 추세 | 안정 | 주당 5pp 이상 하락 |

### 6주 시점 (효과 소멸 점검)

| 지표 | 지속 기준 | 종료 기준 |
|------|----------|----------|
| 응답률 | 20% 이상 유지 | 10% 미만 하락 |
| 튜터 opt-out 비율 | 10% 미만 | 20% 이상 |

## 8. 리스크 및 완화

| 리스크 | 확률 | 영향 | 완화 방안 |
|--------|------|------|----------|
| 패턴 파악 → 효과 소멸 | 높 | 중 | 템플릿 로테이션 + Level Up 출시까지 브릿지 |
| 튜터 반감 (스팸 느낌) | 중 | 높 | 주 1회 엄수, 3회 미응답 시 4주 중단 |
| 자동 이메일 발각 → 신뢰 저하 | 중 | 높 | 톤 품질 관리, 수동 이메일 혼합 |
| 일본어 표현 실수 | 낮 | 높 | 네이티브 검수 필수, 정기적 톤 리뷰 |
