## 2. 검토 의견

### 2.1 ✅ 잘 된 점

- **비용 계산 정확성**: 마일스톤별 단가 계산이 정확함
- **시나리오 다양성**: 3가지 옵션 × 증가율 변형으로 의사결정 범위 제공
- **2월 실제 데이터 기반**: 495명 튜터의 실제 수업 수 분포로 시뮬레이션

### 2.2 ❌ 핵심 문제: 인센티브가 상위 튜터에게 편중

2월 데이터의 튜터 분포를 보면:

| 월 수업 | 튜터 수 | 누적 비율 | 인센티브 효과 |
|---------|---------|----------|-------------|
| 0-30 | ~215명 | **43%** | Milestone 0-1 → 0~0.5% (거의 없음) |
| 30-60 | ~106명 | 65% | Milestone 1-3 → 0.5~1.5% (미미) |
| 60-100 | ~57명 | 76% | Milestone 3-5 → 1.5~2.5% |
| 100-200 | ~43명 | 85% | Milestone 5-13 → 2.5~6.5% |
| 200+ | ~25명 | 90%+ | Milestone 13+ → 6.5%+ **(혜택 집중)** |

> **하위 43% 튜터(월 30회 미만)는 인센티브를 거의 받지 못한다.** 이 구간이 바로 활성화 타겟인 저활동 튜터.

### 2.3 ❌ 상위 튜터에게 불필요한 보너스

| 상위 튜터 | 월 수업 | Option 2 인상률 | 수당 증가/수업 |
|----------|---------|---------------|-------------|
| 켄고 | 352.5 | ~18% | +₩607 |
| 세리 | 344.5 | ~18% | +₩607 |
| 소노코 | 249.5 | ~15% | +₩506 |
| 이시하라 | 206 | ~13% | +₩439 |
| 아이코 | 191.5 | ~13% | +₩439 |

> 이미 풀타임급으로 일하는 튜터에게 보너스를 주는 구조. 이들이 추가 수업을 더 열 가능성은 낮음 (이미 최대치).

### 2.4 ❌ 효과 추정 부재

모델에서 누락된 핵심 항목:

| 누락된 항목 | 왜 필요한지 |
|-----------|-----------|
| **예상 추가 슬롯/월** | 인센티브로 슬롯이 몇 개 느는지 |
| **슬롯당 ROI** | 추가 슬롯 × 수업 매출 - 추가 비용 |
| **풀부킹 감소 예상치** | 현재 770회/일 → 예상 |
| **이탈 방지 매출** | 풀부킹 완화로 지키는 구독 매출 |
| **부업 시간 제약 반영** | 인센티브를 줘도 물리적으로 시간이 없는 튜터 비율 |

### 2.5 ⚠️ 10%/25% Increase 시나리오의 의미 불명확

"10% Increase" / "25% Increase" 열의 의미가:
- 튜터 수가 10-25% 증가?
- 각 튜터의 수업 수가 10-25% 증가?
- 고마일스톤 도달 튜터가 10-25% 증가?

모델 내 설명이 없어 해석이 어려움. 명확한 정의 필요.

## 3. 수정 제안

### 3.1 인센티브 구조 재설계: "증가분" 기반

```
현재: 절대 수업 수 기반 → 고수업 = 고인센티브 (상위 보상)
          ↓
제안: 전월 대비 증가분 기반 → 성장 = 고인센티브 (성장 보상)
```

#### 제안 구조

| 전월 대비 증가 | 증가분 수당 배율 |
|--------------|---------------|
| +5회 이상 | 1.2배 (추가분만) |
| +10회 이상 | 1.5배 (추가분만) |
| +20회 이상 | 2.0배 (추가분만) |

#### 장점

- 월 10회 → 20회 늘린 **저활동 튜터도 보상**받음
- 이미 300회 하는 튜터에게 불필요한 보너스 안 줌
- **실제 "추가 슬롯"에만 비용이 투입**됨

### 3.2 비용 효율 비교

Option 1 수준 예산(₩200만원/월)으로 "증가분 인센티브" 적용 시:

| 항목 | 수치 |
|------|------|
| 추가 슬롯 1개당 비용 | ₩3,374 × 1.5 = ₩5,061 |
| ₩200만원 ÷ ₩5,061 | **약 395슬롯/월 추가 가능** |
| 일평균 추가 | **+약 13슬롯/일** |

> 같은 ₩200만원 예산이라도 절대량 인센티브보다 **증가분 인센티브**가 저활동 튜터를 움직이는 데 훨씬 효과적.

### 3.3 하이브리드 모델 제안

두 가지를 결합:

```
1단계: 기본 마일스톤 인센티브 (현행 Option 1 유지)
   → 상위 튜터 리텐션 역할

2단계: 증가분 인센티브 추가
   → 저활동 튜터 활성화 역할

총 예산: Option 1 수준 (₩200만원/월) 내에서 배분
   → 마일스톤 70% + 증가분 30%
```
