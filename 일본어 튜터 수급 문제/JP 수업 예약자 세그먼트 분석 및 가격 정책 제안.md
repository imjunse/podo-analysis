## 1. 분석 배경

일본어 수업 풀 부킹으로 유저들이 수업을 예약하지 못하는 현상이 지속되고 있다. 튜터 수급이 불안정한 상황에서, 수요 측 분석을 통해 **누가 슬롯을 소비하고 있는지** 파악하고, 가격 정책으로 수요를 조절할 수 있는지 검토한다.

### 분석 목표

1. JP 수업 예약자의 빈도별 세그먼트 분류 (저/중/고빈도)
2. 행동 기반 세그먼트(이벤트 DB)와 교차 분석
3. 슬롯 소비 집중도 파악
4. 가격 정책 제안

## 7. 예약 리드타임 분석

| 리드타임 | 전체 | 고빈도 | 중빈도 | 저빈도 |
|---------|------|--------|--------|--------|
| 당일 예약 | 58.9% | 59.5% | 59.6% | 45.8% |
| 1일 전 | 38.9% | 39.1% | 36.1% | 40.2% |
| 2일+ 전 | 2.2% | 1.4% | 4.3% | **14.0%** |

> 고빈도 유저는 98.6%가 당일~전날에 예약 (현행 1건 사전예약 제한의 영향). 저빈도 유저는 14%가 2일+ 전에 예약 — 사전 계획형.

### 사전 예약 3건 확대 시 영향

사전 예약을 1건 → 3건으로 확대하면:
- 고빈도 유저 1,920명이 미래 3일치를 한꺼번에 선점 가능
- 잠재 선점 슬롯: 1,920 → 5,760개 (3배)
- 일 슬롯 ~1,053개 대비 **5.5일치를 한 번에 잠글 수 있음**
- **오버부킹 악화 가능성 높음** → 사전 예약 확대는 비추천

## 9. 문제의 구조적 원인

```
┌───────────────────────────────────────────────────────────┐
│  공급 = 고정 (튜터 ~520명, 일 ~1,053슬롯, 감소 추세)       │
│  수요 = UNLIMIT 유저에게 사실상 무제한                      │
│  가격 = 수요를 조절하지 못하는 flat pricing                 │
│  결과 = 고빈도 35%가 86% 슬롯 소비, 나머지 65%가 피해       │
└───────────────────────────────────────────────────────────┘
```

무제한 수강권이 "진짜 무제한"인 것이 근본 원인. 공급이 제한된 상황에서 가격이 수요를 조절하지 못하고 있다.

## 10. 해결 방안: 가격 정책 재설계

### 10.1 방안 비교

| 방안 | 설명 | 기대 효과 | 리스크 |
|------|------|----------|--------|
| A. 단순 가격 인상 | UNLIMIT 월정액 인상 | 일부 유저 이탈로 수요 감소 | 저빈도 유저가 먼저 이탈 (역선택) |
| B. 주간 수업 상한 | UNLIMIT에 주 5회 캡 | 초고빈도 259명 제한, ~8,000+ 슬롯 해방 | "무제한인데 제한?" 불만 |
| **C. 단계별 요금제** | Basic/Pro/Ultimate | 수요를 가격으로 자연 분산 | **가장 깔끔한 해법** |
| D. 피크타임 추가 요금 | 20-24시 프리미엄 | 밤 44% 수요 분산 | UX 복잡 |
| E. 대기열 시스템 | 풀 부킹 시 대기열 | 슬롯 낭비 방지 | 근본 해결 아님 |

### 10.2 추천안: C. 단계별 요금제

```
현재:   UNLIMIT = 무제한 (flat pricing)
              ↓
변경:   Standard  = 주 3회  (현재가 유지 or 소폭 인하)
        Premium   = 주 5회  (현재가)
        Ultimate  = 주 7회  (현재가 × 1.5~2배)
```

#### 예상 영향

| 대상 | 현재 | 변경 후 | 영향 |
|------|------|---------|------|
| 초고빈도 259명 | 주 6-7회 | Ultimate 결제 or 주 5회로 줄임 | 수익 증가 or 슬롯 해방 |
| 고빈도B 615명 | 주 3-5회 | Premium 유지 | 변화 없음 |
| 고빈도A 763명 | 주 2-3회 | Standard로 충분 | 가격 부담 감소 가능 |
| 중/저빈도 1,100명 | 주 1-2회 | Standard로 충분 | 가격 부담 감소 가능 |

#### 기대 효과

1. **초고빈도 259명이 Ultimate로 전환 시**: 수익 증가 (가격 × 1.5~2배)
2. **초고빈도 중 일부가 Premium(주5회)으로 전환 시**: 약 8,000~18,000 슬롯 해방
3. **저빈도 유저**: Standard로 오히려 가격 부담 감소 → 이탈 방지
4. 튜터 수급 늘리지 않아도 기존 공급으로 더 많은 유저 수용 가능

### 10.3 보조 방안: 피크타임 수요 분산

단계별 요금제와 병행하여:

- **밤 20-24시** 수업은 프리미엄 슬롯으로 분류
- Standard 유저는 주 3회 중 1회만 프라임타임 가능
- Premium 이상은 프라임타임 제한 없음
- 이를 통해 44%에 집중된 밤 수요를 오전/오후로 분산

## 부록: 쿼리 참조

### 빈도 그룹 분류 기준

| 그룹 | 기준 (3개월간 예약 횟수) |
|------|----------------------|
| 저빈도 | 1-4회 |
| 중빈도 | 5-12회 |
| 고빈도 | 13회+ |

### 행동 세그먼트 분류 로직

```sql
CASE
  WHEN meet_count > 0 AND booking_count > 0
       AND report_video_count > 0 AND total_events >= 30
    THEN 'POWER_LEARNER'
  WHEN meet_count > 0 AND (booking_count > 0 OR total_events >= 10)
    THEN 'ACTIVE_LEARNER'
  WHEN payment_page_count > 0
    THEN 'PURCHASE_INTENT'
  WHEN subscribe_page_count > 0 AND meet_count = 0 AND booking_count = 0
    THEN 'WINDOW_SHOPPER'
  WHEN subscribe_page_count > 0 AND meet_count = 0
    THEN 'TRIAL_EXPLORER'
  ELSE 'ENGAGED_BROWSER'
END
```

### 데이터 참고 사항

- `GT_CLASS.CLASS_DATE`에 미래 날짜(2120년) 데이터 존재 → `CLASS_DATE <= today()` 필터 적용
- `CANCEL_PRESTUDY` 상태는 취소된 수업으로 분석에서 제외
- `user_segment_dict`의 segment는 수강권 타입(UNLIMIT/COUNT/TRIAL/NOTHING)을 의미
