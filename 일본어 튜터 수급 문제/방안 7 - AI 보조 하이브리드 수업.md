# 방안 7: AI 보조 하이브리드 수업

> 상위 문서: [[문제 분석 및 해결 방안]]
> 난이도: 중 | 소요: 4-8주 | 기대 효과: 중-높 | 우선순위: Tier 3 - 기존 AI 콘텐츠 리사이클링 가능성 확인 필요

---

## 1. 핵심 가설

> "100% AI 수업은 환불률 17.9%로 실패했지만, AI 연습 + 튜터 피드백의 하이브리드 모델은 수업 품질을 유지하면서 튜터 1인당 커버 가능한 학생 수를 2-3배로 늘릴 수 있다."

**근거:**
- 100% AI 실패 원인: 학생이 원하는 건 "사람과의 대화" → AI만으로는 불충분
- 하지만 학생이 원하는 "사람 부분"은 **피드백과 격려**이지 전체 수업 시간이 아닐 수 있음
- 15분 AI 연습 + 10분 튜터 피드백 = 25분 수업인데, 튜터는 10분만 사용

---

## 2. 수업 모델 설계

### 2-1. 모델 A: "AI 연습 + 튜터 피드백" (25분)

```
[Phase 1: AI 연습 - 15분] (튜터 불필요)
├─ AI가 발음 교정 드릴 진행
├─ AI가 단어/문법 퀴즈 진행
├─ AI가 간단한 대화 연습 (패턴 드릴)
└─ AI가 학생의 약점 리포트 생성

[Phase 2: 튜터 피드백 - 10분]
├─ 튜터가 AI 리포트 확인 (30초)
├─ 학생의 약점 부분 집중 교정 (5분)
├─ 자연스러운 회화 연습 (3분)
└─ 다음 학습 방향 안내 (1.5분)
```

**튜터 효율:** 25분 슬롯에서 실제 튜터 시간 10분 → 1시간에 6명 커버 가능 (현재 2-3명)

### 2-2. 모델 B: "사전학습 AI + 풀 튜터 수업" (15+15분)

```
[사전학습: AI - 15분] (수업 전 별도 진행)
├─ 오늘 수업 단어/문법 미리 학습
├─ 발음 연습
├─ 간단한 퀴즈로 준비도 확인
└─ 튜터에게 학생 준비 상태 리포트 전달

[수업: 튜터 - 15분]
├─ 사전학습 기반으로 바로 실전 회화
├─ AI에서 어려웠던 부분 집중
├─ 자연스러운 대화 연습
└─ 피드백 + 다음 학습 안내
```

**이 모델은 Level Up 커리큘럼(15분 수업 + 사전학습)과 정확히 일치.**

### 2-3. 모델 C: "릴레이 수업" (5+15+5분)

```
[도입: 튜터 - 5분]
├─ 인사, 오늘 학습 주제 안내
└─ 학생 컨디션 체크

[메인: AI 연습 - 15분]
├─ 튜터가 설정한 주제로 AI 드릴
├─ 학생 혼자 AI와 연습
└─ 실시간 학습 데이터 수집

[마무리: 튜터 - 5분]
├─ AI 연습 결과 리뷰
├─ 핵심 교정 포인트
└─ 격려 + 다음 수업 안내
```

## 3. 모델 비교

| 항목 | 모델 A (AI+피드백) | 모델 B (사전학습+수업) | 모델 C (릴레이) |
|------|------------------|-------------------|--------------|
| 튜터 시간 | 10분 | 15분 | 10분 |
| 학생 체감 | "AI 수업 + 짧은 피드백" | "준비된 수업" | "튜터가 함께하는 느낌" |
| 튜터 효율 | 6명/시간 | 4명/시간 | 6명/시간 |
| 기존 AI 활용도 | 높음 | 높음 | 높음 |
| Level Up 시너지 | 중 | **높음** | 중 |
| 학생 수용도 (예상) | 중 | 높 | 중-높 |
| 비바이링구얼 호환 | 높음 (10분 피드백) | 중 (15분 수업) | 높음 (10분 실시간) |

**권장: 모델 B를 메인으로, Level Up 커리큘럼과 통합 설계**

모델 B가 Level Up(15분 수업 + 사전학습)과 구조적으로 동일하므로, 별도 개발이 아닌 **Level Up의 AI 사전학습 모듈로 자연스럽게 포지셔닝** 가능.

## 4. 기존 AI 콘텐츠 리사이클링

### 4-1. 현재 Podo의 AI 인터랙티브 콘텐츠 (확인 필요)

문서에 "인터랙티브 AI 수업이 존재"한다고 언급. 구체적으로:

| AI 콘텐츠 유형 | 하이브리드 활용 가능성 | 활용 방식 |
|---------------|-------------------|---------|
| 발음 교정 드릴 | 높음 | Phase 1 (AI 연습) 그대로 활용 |
| 단어/문법 퀴즈 | 높음 | 사전학습 자료로 재포지셔닝 |
| AI 대화 연습 | 중 | 패턴 드릴로 축소하여 활용 |
| 리스닝 연습 | 중 | 사전학습 보충 자료 |

### 4-2. 추가 개발 필요 항목

| 항목 | 신규 개발 필요 여부 | 복잡도 |
|------|-----------------|--------|
| AI → 튜터 리포트 생성 | 신규 | 중 |
| AI 연습 결과 요약 대시보드 (튜터용) | 신규 | 중 |
| AI 연습 ↔ 튜터 수업 전환 UX | 신규 | 낮음 |
| 기존 AI 콘텐츠 레벨별 분류 | 정리 작업 | 낮음 |

### 4-3. 100% AI 수업 실패에서 배운 것

**환불률 17.9%의 원인 분석:**

| 원인 | 하이브리드에서의 대응 |
|------|-------------------|
| "사람이 없다" 느낌 | 튜터 피드백 세션으로 보완 |
| AI의 교정이 기계적 | AI는 드릴/연습, 교정은 튜터가 |
| 대화가 부자연스러움 | 자연 대화는 튜터 시간에 |
| 동기부여 부족 | 튜터의 격려 + 진도 관리 |
| 기술적 문제 (음성 인식 등) | AI 부분에서 발생해도 튜터가 보완 |

## 5. 비바이링구얼 튜터 호환성

이 방안의 가장 큰 장점 중 하나: **비바이링구얼 튜터로도 가능**.

### 5-1. 왜 가능한가

현재 문제: 비바이링구얼 튜터가 25분 초급 수업을 한국어 없이 진행 불가 (2-4번 파트)

하이브리드 모델에서:
- AI가 문법 설명, 단어 학습, 패턴 드릴을 한국어로 처리 (사전학습)
- 튜터는 10-15분만 사용 → 발음 교정 + 간단한 피드백 위주
- 복잡한 문법 설명이 필요 없음 (AI가 이미 처리)
- **튜터의 역할이 "가르치기"에서 "교정/피드백/격려"로 축소**

### 5-2. 비바이링구얼 튜터 수업 가능 범위

| 활동 | 한국어 필요성 | 비바이링구얼 가능 여부 |
|------|-------------|-------------------|
| 발음 교정 | 불필요 | 가능 |
| 간단한 패턴 회화 | 낮음 | 가능 (AI 사전학습 후) |
| 문법 설명 | 높음 | **AI가 처리** |
| 자유 회화 (초급) | 높음 | 제한적 (AI 프롬프트로 보조) |
| 피드백/격려 | 낮음 | 가능 (정형화된 표현) |

### 5-3. Level Up + AI 하이브리드의 통합 비전

```
[Level Up 커리큘럼 + AI 하이브리드]

사전학습 (AI, 15분):
  - 한국어 설명 포함된 문법/단어 학습
  - 발음 드릴
  - 패턴 연습

수업 (튜터, 15분):
  - 바이링구얼 OR 비바이링구얼 모두 가능
  - 사전학습 기반 교정/피드백
  - 간단한 실전 회화

→ 비바이링구얼 튜터 풀 확대 → 수급 문제 근본 해결
```

## 7. 단계적 도입 계획

### Phase 0: 리서치 (1주)
- [ ] 기존 AI 인터랙티브 콘텐츠 인벤토리 작성
- [ ] 초급(A1-A2) 적합 콘텐츠 분류
- [ ] Level Up 팀(하림팀)과 통합 가능성 논의
- [ ] 100% AI 수업 데이터에서 환불 원인 상세 분석

### Phase 1: AI 사전학습 모듈 (3주)
- [ ] Level Up 사전학습 자료에 기존 AI 콘텐츠 통합
- [ ] AI → 튜터 리포트 생성 기능 개발
- [ ] 사전학습 완료 여부 트래킹
- [ ] 튜터 대시보드에 학생 사전학습 결과 표시

### Phase 2: 하이브리드 수업 파일럿 (2주)
- [ ] 바이링구얼 튜터 5명 + 비바이링구얼 튜터 5명으로 파일럿
- [ ] 학생 50명 대상 (사전학습 + 15분 수업)
- [ ] 수업 평점, 재예약률, 학생 피드백 수집

### Phase 3: 확대 (ongoing)
- [ ] 파일럿 결과 기반 확대 or 조정
- [ ] 비바이링구얼 튜터 비율 점진적 확대

## 9. 리스크 및 완화

| 리스크 | 확률 | 영향 | 완화 방안 |
|--------|------|------|----------|
| "AI 수업이잖아" 거부감 | 중 | 높 | "사전학습"으로 프레이밍. 수업은 튜터와 |
| AI 기술 품질 (음성 인식 등) | 중 | 중 | 기존 검증된 AI 모듈만 활용 |
| 사전학습 안 하고 수업 참여 | 높 | 중 | 사전학습 미완료 시 튜터에게 알림, 수업 중 보충 |
| 100% AI 실패 이미지 전이 | 중 | 중 | 완전히 다른 상품으로 포지셔닝 ("AI 수업"이 아닌 "스마트 사전학습") |
| Level Up 팀과 일정 충돌 | 중 | 높 | Phase 0에서 사전 협의, 통합 로드맵 수립 |
