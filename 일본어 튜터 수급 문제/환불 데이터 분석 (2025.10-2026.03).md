## 2. 분석 방법

### 데이터 조인 구조

```
ticket (환불 티켓)
  → user_id로 GT_PAYMENT_INFO.USER_UID 조인
  → 일본어 환불 티켓 중 booking 관련 키워드 매칭 유저의 실제 결제 환불 금액 추출
```

### Full Booking 판별 키워드

환불 티켓 content에서 아래 키워드 중 하나 이상 포함 OR 환불 사유가 "튜터 배정 시스템" / "예약 시스템"인 건:

| 카테고리 | 키워드 |
|---------|--------|
| 예약 관련 | 예약, 예약하기, 예약 가능, 예약이 안 |
| 수업 불가 | 수업이 없, 수업 못, 수업을 못 |
| 자리/마감 | 자리, 빈자리, 자리가 없, 마감, 다 찼 |
| 튜터 부족 | 튜터가 없, 튜터 없, 선생님이 없, 배정, 수급 |
| 시간 관련 | 원하는 시간, 시간이 안 맞, 스케줄 |
| 기타 | full, 풀부킹, 못 잡, 잡기 힘, 잡을 수 없 |

## 7. 인사이트

### 7-1. 일본어 환불이 압도적

- JP+ENJP가 전체 환불의 **70.2%** (금액 기준 69.6%)
- 일본어 튜터 수급 문제가 해결되면 전체 환불 규모에 직접적 영향

### 7-2. Full Booking 환불은 빙산의 일각

- 키워드 매칭 기준 약 **1,583만원 / 74명** (5개월)
- 그러나 이는 content에 명시적으로 불만을 작성한 건만 포착
- "학습시간 부족"으로 사유를 선택했지만 실제론 예약 불가가 원인인 **잠재 이탈자**가 훨씬 많을 것
- 보수적으로 2~3배, 낙관적으로 5배까지 실제 규모가 클 수 있음 (추정 3,000만~8,000만원)

### 7-3. 3월 경고 신호

- 3월은 아직 3주(W10~W13)인데 booking 관련 환불 498만원으로 **월 기준 최고치 페이스**
- 건당 환불 금액도 높아지는 추세 (고가 패키지 유저가 이탈하기 시작)
- W10~W11에 연속 4건/주 발생 → 수급 악화 시그널

### 7-4. 환불 금액 변동성이 큼

- Booking 관련 환불은 건수는 적지만 **건당 금액 편차가 큼** (10만~180만)
- 고가 패키지(12개월권 등) 유저 1명 이탈 시 환불 금액이 급등
- 이는 LTV가 높은 유저가 이탈한다는 뜻으로, **수급 문제의 실질적 비용이 단순 환불 금액보다 훨씬 큼**

### 7-5. 숨겨진 비용: 환불하지 않는 이탈

- 환불 데이터에 잡히지 않는 비용이 더 클 수 있음:
  - 구독 갱신을 하지 않는 조용한 이탈 (non-renewal)
  - 수업을 듣지 않고 레슨권 소진 → 낮은 NPS / 부정적 후기
  - 입소문 악화로 인한 신규 유입 감소

## 9. 데이터 한계 및 주의사항

1. **키워드 매칭 방식**: 자연어 표현을 모두 포착하지 못함 (False Negative 존재)
2. **사유 선택 편향**: 유저가 실제 사유와 다른 항목을 선택할 수 있음
3. **ENJP 분리 불가**: 영일패키지 유저의 환불이 일본어 수급 때문인지 영어 불만인지 구분 어려움
4. **위약금 상쇄**: 일부 주차에서 +금액이 나타나는 건 위약금이 환불금보다 큰 경우
5. **W13(3/23)은 진행 중**: 해당 주차 데이터는 아직 완결되지 않음
