# 방안 2: 주간 일관성 보너스

> 상위 문서: [[문제 분석 및 해결 방안]]
> 난이도: 비개발 (운영) | 소요: 즉시 | 기대 효과: 높음 | 우선순위: Tier 1 - 2순위

---

## 1. 핵심 가설

> "월 단위 선물 인센티브(1월 실적: +2.5시간/주, 230만원)"의 실패 원인은 주기가 너무 길어 습관 형성이 안 된 것이다. 주간 단위 + 개인 성장 기반으로 바꾸면 습관 형성률이 높아지고, 인센티브 종료 후에도 효과가 유지될 수 있다.

**근거:**
- 행동심리학: 보상 주기가 짧을수록 행동 강화 효과가 큼 (즉시 보상 > 지연 보상)
- 1월 인센티브: 230만원으로 평균 8.5시간 → 11시간 달성했으나, 종료 즉시 원복
- 핵심 문제: 월 1회 보상은 "이번 달은 어차피 놓쳤으니 다음 달에"라는 심리 유발

---

## 2. 인센티브 설계

### 2-1. 기본 구조: 주간 마일스톤 + 연속 보너스

```
[주간 마일스톤]
  이번 주 수업 N회 이상 → 기본 보너스

[연속 보너스]
  2주 연속 달성 → 추가 보너스
  4주 연속 달성 → 더 큰 추가 보너스

[개인 성장 보너스]
  지난 주 대비 수업 수 증가 → 별도 보상
```

### 2-2. 구체적 금액 설계 (월 230만원 예산 기준)

**현황:** JP 바이링구얼 튜터 ~290명, 평균 8.5시간/주

**목표:** 평균 11시간/주 (1월 인센티브 때 달성한 수치 재현)

#### 옵션 A: 절대치 마일스톤

| 주간 수업 수 | 보너스 | 대상 튜터 수 (추정) | 주간 비용 |
|------------|--------|-----------------|----------|
| 15회 이상 | 10,000원 | ~100명 | 100만원 |
| 20회 이상 | 추가 10,000원 | ~60명 | 60만원 |
| 25회 이상 | 추가 15,000원 | ~30명 | 45만원 |
| **주간 소계** | | | **~205만원** |
| **월간 소계** | | | **~820만원** |

> 문제: 월 230만원 예산을 크게 초과. 대상을 좁히거나 금액을 낮춰야 함.

#### 옵션 B: 개인 성장 기반 (권장)

| 조건 | 보너스 | 대상 (추정) | 주간 비용 |
|------|--------|-----------|----------|
| 지난 주 대비 +3회 이상 | 5,000원 | ~40명 | 20만원 |
| 지난 주 대비 +5회 이상 | 10,000원 | ~20명 | 20만원 |
| 2주 연속 성장 | 추가 5,000원 | ~15명 | 7.5만원 |
| 4주 연속 성장 | 추가 10,000원 | ~8명 | 8만원 |
| **주간 소계** | | | **~55만원** |
| **월간 소계** | | | **~220만원** |

> 장점: 예산 내 운영 가능. 소규모 튜터도 동기부여. 고정 비용 아닌 성과 비용.

#### 옵션 C: 하이브리드 (최종 권장)

| 조건 | 보너스 | 설명 |
|------|--------|------|
| 주간 기본 달성 (12회+) | 3,000원 | 최소 참여 유지 보상 |
| 지난 주 대비 +3회 | 5,000원 | 성장 인센티브 |
| 피크타임 수업 비율 50%+ | 3,000원 | 수요 높은 시간대 유도 |
| 4주 연속 12회+ 달성 | 15,000원 | 습관 형성 보너스 |

**추정 월 비용: ~200-250만원** (1월 인센티브와 동일 수준)

### 2-3. 피크타임 가중치

현재 문제는 단순히 수업 수가 적은 게 아니라, **피크타임(19-22시)에 공급이 부족**한 것.

- 모든 시간대 수업을 동일 가치로 치면 비피크 수업만 늘어날 위험
- **피크타임 수업에 1.5배 가중치** 부여
  - 예: 피크타임 1수업 = 1.5수업으로 카운트
  - 또는 피크타임 전용 추가 보너스 (위 옵션 C의 "피크타임 비율" 조건)

## 3. 습관 형성 메커니즘

### 3-1. 왜 주간이 월간보다 효과적인가

| 측면 | 월간 | 주간 |
|------|------|------|
| 피드백 주기 | 30일 후 | 7일 후 |
| "놓침" 회복 | 불가 (한 번 놓치면 포기) | 다음 주에 바로 재시작 |
| 진행 가시성 | 한 달 뒤에야 알 수 있음 | 매주 확인 가능 |
| 행동 변화 타이밍 | "다음 달에" | "이번 주에" |

### 3-2. 인센티브 종료 후 유지를 위한 설계

1월 인센티브의 핵심 실패: 인센티브 종료 → 즉시 원복

**습관 정착 전략:**

1. **8주 프로그램으로 설계** (습관 형성에 평균 66일 필요 - Phillippa Lally 연구)
   - 1-4주: 주간 보너스 + 연속 보너스
   - 5-8주: 주간 보너스 금액 점진적 감소 (weaning off)
   - 9주+: 월간 "Top 튜터" 인정 프로그램으로 전환 (금전 → 인정)

2. **"페이드아웃" 보너스 구조**
   - 1-2주: 100% 보너스
   - 3-4주: 100% 보너스
   - 5-6주: 75% 보너스
   - 7-8주: 50% 보너스
   - 9주+: 인정 기반 (프로필 뱃지, 우선 배정 등)

3. **비금전적 보상 병행**
   - 수업 리뷰 우선 표시
   - "인기 튜터" 뱃지
   - 학생 매칭 알고리즘에서 우선순위 상향 (더 많은 학생 유입)
   - 주간 보너스 달성 튜터에게 "이번 주 최고 튜터" 메시지 발송

## 4. 운영 플로우

### 4-1. 주간 사이클

```
[월요일 00:00] 새 주간 사이클 시작
    ↓
[월요일 09:00] 지난 주 결과 + 이번 주 목표 알림 발송
    "지난 주 14회 수업을 진행하셨어요! 이번 주 17회를 달성하면 5,000원 보너스!"
    ↓
[수요일 09:00] 중간 현황 알림
    "이번 주 현재 8회 완료. 목표까지 9회 남았어요!"
    ↓
[금요일 09:00] 마지막 푸시
    "주말에 5회만 더 열면 보너스 달성! 토요일 저녁이 가장 수요가 많아요."
    ↓
[일요일 23:59] 주간 마감, 정산
    ↓
[월요일] 보너스 지급 + 다음 주 시작
```

### 4-2. 커뮤니케이션 톤

**중요:** JP 튜터는 일본에서 부업으로 하는 사람이 대부분. 커뮤니케이션이 "압박"이 되면 역효과.

**DO:**
- "이번 주 목표를 달성하셨네요! 감사합니다 🎉"
- "학생들이 [튜터이름] 선생님과의 수업을 좋아해요"
- 개인 성장을 축하하는 톤

**DON'T:**
- "아직 목표에 도달하지 못했습니다"
- "더 많은 수업을 열어주세요"
- 경쟁이나 비교를 유발하는 메시지

## 5. ROI 분석

### 5-1. 투자 (월 기준)

| 항목 | 금액 |
|------|------|
| 보너스 지급 (옵션 C 기준) | ~220만원 |
| 운영 인건비 (주간 정산, 알림 관리) | ~30만원 |
| **월 총 투자** | **~250만원** |

### 5-2. 기대 수익

| 항목 | 보수적 | 낙관적 |
|------|--------|--------|
| 평균 근무시간 증가 | +1.5시간/주 | +2.5시간/주 |
| 추가 수업 수/주 | +600 | +1,000 |
| booking 관련 환불 감소 | 월 300만원 | 월 1,000만원 |
| 조용한 이탈 방지 (추정) | 월 200만원 | 월 500만원 |
| **월 총 기대 수익** | **월 500만원** | **월 1,500만원** |

### 5-3. ROI

- **보수적: 500만 / 250만 = ROI 2.0x**
- **낙관적: 1,500만 / 250만 = ROI 6.0x**
- 1월 인센티브 실적(+2.5시간) 기준이면 낙관적에 가까움

## 7. 리스크 및 완화

| 리스크 | 확률 | 영향 | 완화 방안 |
|--------|------|------|----------|
| 인센티브 종료 시 원복 (1월 재현) | 높 | 높 | 8주 페이드아웃 + 비금전 보상 병행 |
| 보너스 게이밍 (짧은 수업 남발) | 중 | 중 | 최소 수업 시간(15분) 기준 적용, 학생 평점 하한선 설정 |
| 수업 질 저하 (양에만 집중) | 중 | 높 | 학생 평점 4.0 이상 유지 조건 추가 |
| 예산 초과 | 낮 | 중 | 주간 정산으로 실시간 비용 모니터링, 캡 설정 |
| 일부 튜터에게만 집중 (이미 많이 하는 사람만 수혜) | 높 | 중 | 개인 성장 기반 설계로 완화 |

## 8. 실행 체크리스트

- [ ] 첫 정산 완료
