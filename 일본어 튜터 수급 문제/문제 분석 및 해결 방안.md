# 일본어 튜터 수급 부족 → 학생 이탈 문제 분석 및 해결 방안

> 원본 자료: [[Strategy & Plan_ Enhancing Lesson Experience]]
> 작성자: Andrew (원본), GA (분석 정리)
> 분석일: 2026-03-24

---

## 1. 문제 요약

일본어 수업이 항상 풀부킹되어 학생들이 수업을 예약할 수 없고, 이로 인해 이탈(환불 + 미리텐션)이 발생하고 있다.

### 문제의 구조

```
튜터 수급 한계 (바이링구얼 ~290명 상한)
    ↓
풀부킹 → 학생 수업 예약 불가
    ↓
학생 이탈 (환불 33% + 낮은 리텐션)
    ↓
성장 정체
```

### 근본 원인

- 일본어 초급 수업은 **바이링구얼(일-한) 튜터**에 의존
- 이런 튜터는 공급이 제한적 (~290명이 거의 상한)
- 학생 유입 속도 > 튜터 수급 속도
- JP 튜터 평균 근무시간도 하락 추세 (1월 11시간 → 3월 8.5시간/주)

## 2. 핵심 수치 (2026년 3월 기준)

| 지표 | 현황 |
|------|------|
| JP 바이링구얼 튜터 수 | ~290명 (정체) |
| 튜터 평균 근무시간 | 8.5시간/주 (하락 추세) |
| 풀부킹 비율 | 17~24% (피크타임 더 높음) |
| JP 로우레벨 환불률 | 33.06% |
| JP 전체 환불률 | 29.99% |
| EN 전체 환불률 | 33.93% |
| JP 튜터 비용 비중 | 매출의 ~33% |
| EN 튜터 비용 비중 | 매출의 ~19% |
| JP 수업당 비용 | 3,374원 (25분) / 2,309원 (15분) |

### 수급 레버 비교

| 레버 | 효과 |
|------|------|
| 튜터 50명 추가 (300→350) | +800 수업/주 |
| 튜터당 2시간 추가 (8→10시간) | +1,200 수업/주 |

> **시간 증가가 튜터 수 증가보다 효과가 크다**

### 1월 선물 인센티브 결과

- 총 비용: ~230만원
- 효과: 평균 8.5시간 → 11시간/주 (약 +2.5시간)
- 문제: 인센티브 종료 즉시 원래대로 하락 (습관 형성 실패)

## 3. 단기 해결 방안

### 방안 1: 웨이팅리스트 + 알림 시스템

> 구현 난이도: 낮음 | 소요: ~1-2주 | 기대 효과: 높음

학생이 원하는 시간에 예약 불가 시:
- "이 시간대에 빈자리가 나면 알림을 받으시겠습니까?" 옵션 제공
- 튜터 취소/추가 오픈 시 자동 알림
- 알림 테이블 하나 + 크론잡으로 구현 가능
- 학생 입장에서 "할 수 있는 게 없다" → "기다려볼 수는 있다"로 전환되어 즉시 이탈 방지

**보강 포인트:** 알림 받은 학생에게 **5분간 예약 우선권**을 주는 것이 필요. 안 그러면 알림만 받고 예약 실패 → 더 큰 불만.

### 방안 2: 주간 일관성 보너스

> 구현 난이도: 비개발 (운영) | 소요: 즉시 | 기대 효과: 높음

1월 인센티브의 문제점은 **월 단위 → 습관 형성 실패**.

**개선안:**
- 주간 기준으로 변경 (예: 이번 주 20수업 달성 시 보너스)
- 4주 연속 달성 시 추가 보너스
- 월 ~230만원 수준 예산으로 검증된 효과 재현
- 습관 형성에 초점

**보강 포인트:** 단순 절대치 마일스톤보다 **"이번 주 vs 지난 주" 개인 성장 기반 보너스**가 더 효과적일 수 있음. 상대적 증가를 보상하면 소규모 튜터도 동기부여가 됨.

### 방안 3: 예약 가능 시간대 사전 공지

> 구현 난이도: 낮음 | 소요: ~1주 | 기대 효과: 중-높

현재는 학생이 예약하려고 들어와서 "풀부킹"을 발견하는 구조. 이걸 뒤집는 것.

- 매일 아침 or 전날 저녁에 **"내일 예약 가능한 시간대"를 푸시 알림**으로 발송
- 학생이 예약 가능한 시간을 먼저 알고 들어옴 → **"예약 실패" 경험 자체를 줄임**
- 이른 시간대에 예약을 유도하는 부수 효과
- 개발 공수 낮음 (푸시 알림 + 가용 시간 조회 API)

### 방안 4: "개인화 이메일" 자동화

> 구현 난이도: 중 | 소요: ~2주 | 기대 효과: 중

수동 테스트에서 **50% 성공률** 검증된 방법.

**작동 방식:**
1. 풀부킹 시간대를 감지
2. 해당 시간 직전에 스케줄이 끝나는 튜터 2명 선정
3. 개인화된 이메일 발송:
   > "○○ 선생님, 오늘 21시에 수업을 받고 싶어하는 학생이 많은데 자리가 부족합니다. 혹시 1시간만 더 스케줄을 열어주실 수 있을까요?"

**핵심 제약 조건:**
- 튜터당 **주 1회 이하**로 제한 (자주 보내면 효과 소멸)
- 이메일은 **담당자가 직접 보낸 것처럼** 자연스럽게 작성
- 대량 발송이 아닌 **시간대별 2명에게만** 타겟팅

**실행 가능성 점검:**
- 50% 성공률은 수동일 때의 수치. 자동화하면 성공률이 다소 떨어질 것
- 튜터들이 패턴을 파악하면 효과 소멸 — 유효 기간 2~3개월 정도로 예상
- 17시 고정 감지보다 **풀부킹 발생 즉시 실시간 트리거**로 바꾸면 효과 상승 (개발 공수 증가)

### 방안 5: 수업 취소 시 "감정 마찰" 추가

> 구현 난이도: 매우 낮음 | 소요: ~1주 | 기대 효과: 낮음

**현재 (As-is):**
> "수업을 취소하시겠습니까?"

**변경 (To-be):**
> "[학생이름]님과의 수업입니다. 이 학생은 당신과 [N]번 수업했고, [피드백 내용]을 남겼습니다. 정말 취소하시겠습니까?"

**실행 가능성 점검:**
- 프론트엔드 텍스트 변경 + 간단한 API 호출 하나면 구현 가능
- 단, JP 튜터에게 Podo는 부업. 감정 마찰이 **일본 문화에서는 오히려 부담**으로 작용해 스케줄을 덜 여는 역효과 가능성 있음
- 취소를 줄이더라도 근본적인 오픈 시간 부족 문제는 해결 안 됨
- **일본 문화 맥락에서 소규모 테스트 후 결정** 필요

## 4. 중기 해결 방안: 비바이링구얼 "안심 옵션" 실험

### 개요

바이링구얼 필터를 전체 해제하는 게 아니라, 유저에게 선택권을 주는 방식으로 시작.

### 튜터팀 경고

#### 경고 1: 수업 품질 저하 → 환불 증가 (Andrew)

> 현재 커리큘럼이 한국어를 못하는 튜터가 가르치기 매우 어렵게 설계되어 있다. 비바이링구얼 튜터가 현재 교재로 초급 수업을 진행하면 수업 품질이 떨어져 환불률이 오히려 증가할 수 있다.

**핵심 질문:**
> 하루 늦게 예약하는 불편함 vs 나쁜 수업 경험, 어느 쪽이 더 나쁜가?

| | 예약 못함 (현재 문제) | 나쁜 수업 경험 (새 리스크) |
|---|---|---|
| 이탈 방식 | 조용히 떠남 | 적극적 환불 + 부정적 입소문 |
| 복구 가능성 | 높음 (자리 나면 돌아올 수 있음) | 낮음 (신뢰 깨짐) |
| 데이터 가시성 | 안 보임 | 보임 (환불률, 수업 평점) |

#### 경고 2: 현재 교재로는 수업 자체가 불가능 (튜터팀)

현재 일본어 초급 커리큘럼(마이라이트, JLPT)의 수업 진행 구조:

1. **단어 발음 교정** — 한국어 없이도 가능
2. **단어로 문장 만들기 + 간단한 회화** — 초급 학생은 불가능
3. **문법으로 문장 만들기 + 간단한 회화** — 초급 학생은 불가능
4. **프리토킹 질문으로 회화** — 초급 학생은 불가능

> 현재 교재에서 학생과 튜터 모두 스트레스 없이 할 수 있는 파트는 **1번(발음 교정)뿐**이며, 2~4번은 초급 학생이라면 **수업 자체가 불가능**하다.

- Andrew와 Scott이 학생 역할로 2차례 테스트 수업을 진행한 결과, **한국어 없이는 대화 자체가 성립하지 않음**을 확인
- 비바이링구얼 튜터가 수업을 하려면, 튜터와 학생 간 티키타카 없이도 일본어를 트레이닝할 수 있는 **새로운 구조의 교재가 필요**
- 현재 하림팀과 튜터팀에서 초급 교재 틀을 거의 다 잡은 상황

**튜터팀 결론:**
> 가능하다면 **초급 커리큘럼이 개선된 이후에** 비바이링구얼 튜터가 초급 수업을 진행하는 것이 바람직하다.

### 전략: 웨이팅리스트 통합형 안심 옵션

위 경고들을 감안하면, 비바이링구얼 opt-in의 목적을 **실제 수업 연결이 아닌 "선택지가 있다는 인식 제공"**으로 전환. 단, 빈 약속이 되지 않도록 **웨이팅리스트와 자연스럽게 통합**.

#### 구체적 UX 플로우

풀부킹 상황에서 학생에게 보여주는 화면:

> "현재 바이링구얼 튜터가 모두 예약되었습니다."
>
> - **[다른 시간대 보기]** ← 메인 CTA
> - **[빈자리 알림 받기]** ← 웨이팅리스트
> - **[일본어 전용 수업 오픈 시 알림 받기]** ← 수요 데이터 수집 + 미래 연결

세 번째 옵션은 "준비 중입니다"가 아니라 **알림 등록으로 연결** → 빈 약속 느낌을 피하면서 수요 데이터 수집.

#### 이 접근의 장점

1. **학생**: "대안이 아예 없는 건 아니구나" → 즉시 이탈 방지
2. **튜터**: 실제 비바이링구얼 초급 수업은 없으므로 리스크 0
3. **시간 벌기**: 새 커리큘럼(Level Up) 준비될 때까지 브릿지 역할
4. **수요 데이터 수집**: 세 번째 옵션 클릭률로 비바이링구얼 수요를 정량적으로 파악 가능

#### 새 커리큘럼 출시 후 전환

하림팀의 초급 교재가 완성되면 → 세 번째 옵션을 **실제 수업으로 연결**. 이미 UI/플로우가 만들어져 있으니 전환이 매끄럽다.

### 실행 순서

1. 웨이팅리스트 통합형 UI 구현 (풀부킹 시 3가지 선택지 표시) (~1주)
2. 세 번째 옵션 클릭률 모니터링으로 수요 데이터 수집
3. 새 커리큘럼(Level Up) 출시 시 → 실제 비바이링구얼 수업으로 연결
4. 수업 평점 + 환불률 모니터링 후 확대 or 조정

## 5. 추가 검토 방안

### 그룹 수업 도입 (1:1 → 1:2 또는 1:3)

> 내부에서 이미 논의된 바 있는 방안. 실행 가능성은 미확정.

**원리:**
- 현재 1:1 모델이면 튜터 1명 = 수업 1개
- 1:2로 바꾸면 **같은 튜터 수로 수업 가능 인원이 2배**
- 초급 학생끼리 같은 레벨이면 그룹 수업이 오히려 효과적일 수 있음 (서로 배우는 효과)
- 튜터 비용은 동일하지만 학생당 비용은 절반

**리스크:**
- Podo의 핵심 가치가 "1:1 수업"이라면 브랜드 정체성 훼손
- 스케줄 매칭이 훨씬 복잡해짐 (같은 시간 + 같은 레벨 학생 2~3명을 찾아야 함)

**실행 방법 (만약 진행 시):**
- 풀부킹 시에만 "다른 학생과 함께 수업하시겠습니까?" 옵션 제공
- 그룹 수업은 티켓 할인으로 학생에게도 인센티브 부여
- 1:1은 유지하되, 그룹은 **추가 옵션**으로 포지셔닝

### AI 보조 하이브리드 수업

> Podo에 이미 인터랙티브 AI 수업이 존재. 이를 활용한 리사이클링 가능성 검토.

**원리:**
- 기존 AI 인터랙티브 수업(스피킹 아닌 형태)을 **튜터 수업과 결합**
- 예: 15분 AI 인터랙티브 연습 + 10분 튜터 피드백
- 튜터 1명이 시간당 더 많은 학생을 커버 가능
- 비바이링구얼 튜터도 10분 피드백 정도는 가능

**기존 AI 수업의 문제:**
- 100% AI 대화 커리큘럼의 환불률이 17.9%로 높았음
- 하지만 "AI만"이 아니라 **"AI + 튜터 하이브리드"**라면 다른 결과가 나올 수 있음

**리사이클링 가능성:**
- 기존 AI 인터랙티브 콘텐츠를 **수업 전 예습** 또는 **수업 중 액티비티**로 재활용
- 새로운 AI 콘텐츠 개발 없이, 기존 자산을 튜터 수업 보조 도구로 재포지셔닝
- Level Up 커리큘럼의 "사전학습(Pre-study)" 자료로 통합하면 시너지

**검토 필요 사항:**
- 기존 AI 인터랙티브 콘텐츠 중 초급에 적합한 것이 얼마나 있는가
- 튜터 수업과 AI 콘텐츠를 어떤 순서/비율로 결합할 것인가
- 학생 입장에서 "AI + 튜터" 하이브리드에 대한 수용도

## 6. 장기 해결 방안 (문서 내 전략)

Andrew 문서에서 제시한 근본 해결책:

### Level Up 커리큘럼

## 7. 실행 우선순위 종합

### Tier 1: 즉시 실행 (1-2주 내)

| 순위 | 방안 | 난이도 | 기대 효과 |
|------|------|--------|----------|
| 1 | 웨이팅리스트 + 알림 (예약 우선권 포함) | 낮음 | 높음 |
| 2 | 주간 일관성 보너스 (개인 성장 기반) | 비개발 | 높음 |
| 3 | 예약 가능 시간대 사전 공지 | 낮음 | 중-높 |

### Tier 2: 2-3주 내 실행

| 순위 | 방안 | 난이도 | 기대 효과 |
|------|------|--------|----------|
| 4 | 개인화 이메일 자동화 | 중 | 중 |
| 5 | 안심 옵션 (웨이팅리스트 통합) | 낮음 | 중 |

### Tier 3: 검토 및 테스트 필요

| 방안 | 난이도 | 비고 |
|------|--------|------|
| 취소 시 감정 마찰 | 매우 낮음 | 일본 문화 맥락 소규모 테스트 후 결정 |
| 그룹 수업 옵션 | 중-높 | 브랜드 리스크 검토 필요, 효과는 큼 |
| AI 하이브리드 수업 | 중 | 기존 AI 콘텐츠 리사이클링 가능성 확인 필요 |

### 제외

| 방안 | 제외 사유 |
|------|----------|
| 비피크타임 수요 분산 | 학생(낮에 일함)과 튜터(부업이라 낮에 불가) 모두 피크에 집중. 인센티브 줘도 분산 효과 미미 |
| 수업 요청 기능 (독립) | 웨이팅리스트 + 이메일 자동화에 통합하는 것이 더 효율적 |

### 추천 실행 계획

**1, 2, 3번을 동시 시작**하면 2주 안에 체감 가능한 변화.
- 방안 1(웨이팅리스트) = 학생 쪽 이탈 방지
- 방안 2(주간 보너스) = 튜터 쪽 공급 증가
- 방안 3(사전 공지) = "예약 실패" 경험 자체를 줄임

이 세 가지가 **양면(학생+튜터)에서 동시에** 작동하는 구조.

5번(안심 옵션)은 1번에 자연스럽게 통합. 실제 비바이링구얼 수업 연결은 새 커리큘럼 출시 후 진행.
