# JP 풀부킹 해결책 — 예약 변경 제한 정책

> 선행 분석: [[JP 수업 예약자 세그먼트 분석 및 가격 정책 제안]]
> 데이터 소스: ClickHouse (podo_mysql.le_class_status_history, podo_events.events)
> 분석일: 2026-03-25

---

## 1. 핵심 발견: 예약 "변경"이 슬롯을 잡아먹고 있다

가격 인상 없이 독점을 막을 수 있는 근거가 데이터에서 발견되었다.

| 지표 | 수치 |
|------|------|
| 3개월간 총 예약 변경 (RESERVED→RESERVED) | **131,553회** |
| 3개월간 신규 예약 (CREATED→RESERVED) | 119,208회 |
| **변경이 신규 예약보다 많음** | 변경/신규 = **1.1배** |

> 예약 변경이 전체 상태 전환의 **37%**로, 수업 완료(30%)보다 더 큰 비중을 차지한다.

---

## 2. 빈도 그룹별 예약 변경 행동

| 그룹 | 총 전환 수 | 변경 비율 | 취소 비율 |
|------|-----------|----------|----------|
| **고빈도 (13회+)** | 188,263 | **55.3%** | 2.7% |
| 중빈도 (5-12회) | 16,140 | 39.9% | 4.2% |
| 저빈도 (1-4회) | 11,386 | 34.5% | **8.6%** |

### 해석

- **고빈도 유저 행동의 55%가 예약 변경**. 수업을 듣는 것보다 예약을 바꾸는 횟수가 더 많다.
- 저빈도 유저의 **취소율 8.6%**는 고빈도(2.7%)의 3배 → 예약을 못 잡아서 포기하는 패턴.

## 3. "슈퍼 체인저" 분석

### 변경 횟수별 유저 분포

| 변경 횟수 | 유저 수 | 총 변경 건수 | 평균 변경 | 최대 |
|----------|---------|------------|----------|------|
| 1-5회 | 3,227 | 7,040 | 2.2 | 5 |
| 6-20회 | 1,766 | 19,489 | 11.0 | 20 |
| 21-50회 | 781 | 24,290 | 31.1 | 50 |
| **51회+** | **355** | **80,734** | **227.4** | **3,915** |

> **355명이 전체 변경의 61%(80,734건)를 차지.**

### 슈퍼 체인저의 수강권 분포

| 수강권 | 유저 수 | 평균 변경 | 총 변경 수 |
|--------|---------|----------|-----------|
| **UNLIMIT** | **313명** | **230회** | **71,987건** |
| NOTHING (만료) | 39명 | 217회 | 8,465건 |
| COUNT | 3명 | 94회 | 282건 |

> 313명의 UNLIMIT 유저가 72,000번 예약을 변경했다. 1인당 3개월간 평균 230번.

## 4. 일평균 변경 규모

| 지표 | 수치 |
|------|------|
| 3월 일평균 예약 변경 | **~1,450건** |
| 일평균 변경하는 유저 수 | **~600명** |
| 일 전체 슬롯 수 | ~1,053개 |
| **변경/슬롯 비율** | **1.4배** |

> 매일 슬롯 수보다 더 많은 예약 변경이 발생하고 있다.

### 왜 문제인가

```
예약 변경 프로세스:
1. 유저가 "변경" 클릭
2. 새 시간/튜터 탐색 (이 동안 기존 슬롯 점유 유지)
3. 새 슬롯 선택 → 기존 슬롯 해제 + 새 슬롯 점유

→ 2단계에서 "탐색 중 슬롯 이중 점유" 발생 가능
→ 매일 1,450건의 변경이 슬롯 가용성을 일시적으로 축소
→ 다른 유저가 보는 "풀 부킹"의 숨겨진 원인
```

|------|-----------------|----------------|
| 고빈도 | 28.1명 | 12.9% |

## 6. 해결 방안 (가격 인상 없음)

### 방안 1: 예약 변경 횟수 제한 ⭐ 최우선

```
현행:  변경 무제한
변경:  1예약당 변경 최대 2회 / 이후 취소→재예약만 가능
```

| 항목 | 내용 |
|------|------|
| 타겟 | 슈퍼 체인저 355명 (변경 80,734건의 주범) |
| 기대 효과 | 일 1,450건 변경 → 500건 이하 감소, 슬롯 회전율 대폭 개선 |
| 구현 난이도 | 중 (백엔드 변경 카운터 추가) |
| 리스크 | 고빈도 유저 불만 → 단, "튜터 사유 변경"은 제한 없음으로 완화 |

### 방안 2: 변경 쿨다운 (1시간)

```
예약 변경 완료 후 → 1시간 대기 → 다시 변경 가능
```

| 항목 | 내용 |
|------|------|
| 타겟 | 연속 변경으로 슬롯을 "점유"하는 패턴 차단 |
| 기대 효과 | 충동적/탐색적 변경 감소 |
| 구현 난이도 | 하 (타임스탬프 체크만 추가) |
| 리스크 | 거의 없음 |

> 3,915회 변경 유저 = 3개월간 매 시간마다 변경한 수준. 쿨다운만으로도 효과적.

### 방안 3: 저빈도 유저 보호 슬롯

```
각 시간대 슬롯의 10~20%를 "주 4회 미만 유저 전용"으로 예약
```

| 항목 | 내용 |
|------|------|
| 타겟 | 저빈도 유저 2,668명 (취소율 8.6%, 이탈 위험) |
| 기대 효과 | 일 1,053슬롯 중 100~200개 보호 → 저빈도 이탈 방지 |
| 구현 난이도 | 중 (슬롯 할당 로직 변경) |
| 리스크 | 고빈도 유저 선택지 약간 줄어듦 |

> 저빈도 유저가 이탈하면 구독 자체가 사라진다. 매출 보호 관점에서도 중요.

### 방안 4: 피크타임 예약 오픈 시차

```
밤 20-24시 슬롯 → 48시간 전 오픈
그 외 시간대 → 24시간 전 오픈 (현행 유지)
```

| 항목 | 내용 |
|------|------|
| 타겟 | 밤 시간대 44% 수요 집중 분산 |
| 기대 효과 | 피크 경쟁 완화, 오전/오후 수요 유도 |
| 구현 난이도 | 중 |
| 리스크 | UX 복잡도 증가 |

---

## 7. 우선순위 및 실행 계획

| 순위 | 방안 | 구현 난이도 | 기대 효과 | 부작용 |
|------|------|-----------|----------|--------|
| **1** | **예약 변경 횟수 제한** | 중 | ★★★★★ | 고빈도 유저 불만 (완화 가능) |
| **2** | **보호 슬롯** | 중 | ★★★★ | 고빈도 선택지 약간 축소 |
| 3 | 변경 쿨다운 | 하 | ★★★ | 거의 없음 |
| 4 | 피크타임 시차 | 중 | ★★ | 복잡도 증가 |

### 추천 실행 순서

1. **즉시**: 방안 3(쿨다운) — 구현 쉽고 부작용 없음
2. **1주 내**: 방안 1(변경 제한) — 가장 큰 효과, A/B 테스트 추천
3. **2주 내**: 방안 2(보호 슬롯) — 저빈도 유저 이탈 방지
4. **검토 후**: 방안 4(시차 오픈) — 효과 측정 후 판단
