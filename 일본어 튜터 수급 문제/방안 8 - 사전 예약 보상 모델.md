# 방안 8: 사전 예약 보상 모델

> 상위 문서: [[문제 분석 및 해결 방안]]
> 난이도: 중 | 소요: ~2-3주 | 기대 효과: 높음

---

## 1. 문제 재정의: 왜 Last Minute Booking이 문제인가

### 1-1. 예약 맵에서 확인된 사실 (2026-03-23 주간)

| 구분 | 가용 슬롯 | 상태 |
|------|----------|------|
| 월/화 (당일~내일) | 0~10개 | 빨간색/분홍색 (거의 마감) |
| 수~일 (2일+ 후) | 12~48개 | 초록색 (넉넉) |
| 0 Tutor 시간대 | 월 14개, 화 23개 | 튜터 자체가 없는 슬롯 |

**핵심 패턴:** 2일 후까지는 충분한 자리가 있지만, 당일/내일에 몰리면서 풀부킹 발생.

### 1-2. Last Minute Booking의 원인

```
현재 정책: 수업 1시간 전까지 예약 가능

한국 직장인의 현실:
  09:00 출근
  ... 미팅, 야근 여부 불확실 ...
  18:00 "오늘 7시에 끝날 것 같다" → 7:30 수업 예약 시도
  → 이미 풀부킹 → 좌절/이탈
```

- 학생의 "게으름"이 아니라 **불확실한 퇴근 시간에 대한 합리적 대응**
- 이전 12시간 전 예약 정책 시절에는 쏠림이 없었지만, **유저 편의를 위해 1시간으로 완화**한 이력 있음

### 1-3. 왜 "강제"가 아닌 "보상"인가

| 접근 | 효과 | 리스크 |
|------|------|--------|
| 6-12시간 전 예약 **강제** | 쏠림 감소 | 이탈 가속 (수업 못 듣는 학생 증가) |
| 사전 예약 **보상** | 점진적 분산 | 이탈 없음 (기존 유저 경험 유지) |

> **원칙: 기존 유저 경험을 절대 악화시키지 않으면서, 사전 예약에 확실한 이점을 부여하여 행동 변화를 유도한다.**

---

## 2. 핵심 가설

> "사전 예약 시 확실한 보상(튜터 선택권, 인기 시간 우선 배정)을 제공하면, 학생의 예약 시점이 점진적으로 앞당겨지고, 피크타임 풀부킹 빈도가 줄어든다."

**측정 지표:**
- 사전 예약(6시간+ 전) 비율 변화
- 피크타임(19-23시) 풀부킹 빈도 변화
- 전체 예약 성공률 변화
- 환불률 변화 (booking 관련)
- 리텐션 변화 (사전 예약자 vs last-minute 예약자)

## 3. 보상 체계 설계

### 3-1. 사전 예약 시간대별 보상 등급

```
┌──────────────────────────────────────────────────────────┐
│                 사전 예약 보상 체계                        │
│                                                          │
│  🥉 1시간 전     → 기존과 동일 (제한 없음)                │
│  🥈 6시간+ 전    → "튜터 우선 선택권"                     │
│  🥇 12시간+ 전   → "인기 시간대 우선 배정" + 튜터 선택     │
│  💎 24시간+ 전   → "원하는 튜터 지정 예약" + 우선 배정     │
│                                                          │
│  ⚡ 핵심: 기존 1시간 전 예약은 그대로 유지!               │
│     아무것도 빼앗지 않고, 더 일찍 예약하면 더 줌           │
└──────────────────────────────────────────────────────────┘
```

### 3-2. 보상 항목 상세

#### 보상 1: 튜터 우선 선택권 (6시간+ 전)

**현재:** 예약 시 가용 튜터 중 랜덤/알고리즘 배정
**변경:** 6시간+ 전 예약 시 튜터 프로필을 보고 직접 선택 가능

```
[6시간+ 전 예약 화면]

"사전 예약 보상: 튜터를 직접 선택할 수 있어요! ⭐"

┌─────────────────────────┐  ┌─────────────────────────┐
│ 🧑‍🏫 田中 선생님          │  │ 🧑‍🏫 佐藤 선생님          │
│ ⭐ 4.8 (324회)          │  │ ⭐ 4.9 (215회)          │
│ 강점: 발음 교정          │  │ 강점: 문법 설명          │
│ [이 선생님으로 예약]      │  │ [이 선생님으로 예약]      │
└─────────────────────────┘  └─────────────────────────┘

vs

[1시간 전 예약 화면]

"3/25(화) 20:00 예약"
[예약하기]  ← 튜터 자동 배정
```

**왜 효과적인가:**
- 학생들은 "좋은 튜터"와 수업하고 싶어함
- 선택권 = 높은 체감 가치 (실제 비용 0)
- "일찍 예약하면 내가 원하는 선생님과 수업할 수 있다" → 강력한 동기

#### 보상 2: 인기 시간대 우선 배정 (12시간+ 전)

**개념:** 피크타임(19-23시) 슬롯의 일부를 사전 예약자에게 우선 배정

```
[튜터 A의 20:00 슬롯]

12시간+ 전 예약 가능 시점: 아침 8시부터 오픈
6시간+ 전 예약 가능 시점: 오후 2시부터 오픈
1시간 전 예약 가능 시점: 오후 7시부터 오픈 (남은 자리만)
```

**구현 방식: "시차 오픈" (Staggered Release)**

```
피크타임 슬롯 = 총 30개라고 가정

Phase 1 (수업 24시간+ 전): 30개 중 10개(33%) 오픈
  → 24시간+ 전 예약자가 가장 먼저 선점

Phase 2 (수업 12시간+ 전): 추가 10개(33%) 오픈
  → 12시간+ 전 예약자가 선점

Phase 3 (수업 6시간+ 전): 추가 5개(17%) 오픈

Phase 4 (수업 1시간+ 전): 나머지 5개(17%) 오픈
  → Last minute 예약자도 가능 (단, 자리 적음)
```

**효과:**
- "일찍 예약하면 확실히 수업할 수 있다" 경험 → 다음번에도 일찍 예약
- Last minute 유저도 수업 가능 (17% 잔여분) → 이탈 방지
- 풀부킹 체감이 줄어듦 (시간대별로 점진적 마감)

#### 보상 3: 원하는 튜터 지정 예약 (24시간+ 전)

**개념:** 하루 전 예약 시 특정 튜터를 "지명"하여 예약 가능

```
"내일 저녁 8시에 田中 선생님과 수업하고 싶어요"
→ 田中 선생님의 8시 슬롯이 열려 있으면 즉시 확정
→ 아직 안 열려 있으면 "田中 선생님이 스케줄을 열면 자동 예약" 설정
```

**시너지:** 방안 4(개인화 이메일)와 연결
- "田中 선생님, 내일 20시에 [학생이름]님이 선생님과 수업하고 싶어합니다"
- 학생 지명 = 튜터에게 강력한 동기부여 (나를 원하는 학생이 있다!)

## 4. 시차 오픈 (Staggered Release) 상세 설계

### 4-1. 왜 시차 오픈인가

**현재 문제:**
```
모든 슬롯이 수업 1시간 전부터 동시에 예약 가능
→ 18시에 퇴근한 학생들이 19-23시 슬롯에 일제히 몰림
→ 10분 만에 풀부킹
→ 18:15에 접속한 학생은 이미 마감
```

**시차 오픈 후:**
```
아침 8시: 피크 슬롯의 33% 오픈 → 출근 전에 예약하는 학생 확보
오후 2시: 추가 33% 오픈 → 점심시간에 예약하는 학생 확보
오후 6시: 추가 17% 오픈 → 퇴근 직후 예약 가능
오후 7시: 잔여 17% 오픈 → Last minute도 가능

→ 풀부킹 시점이 "18시 한꺼번에"에서 "하루에 걸쳐 점진적"으로 변화
→ "지금 안 되면 끝"이 아니라 "더 일찍 했으면 됐다"로 프레이밍 전환
```

### 4-2. 슬롯 배분 비율

| Phase | 오픈 시점 | 배분 비율 | 타겟 학생 | 보상 |
|-------|----------|----------|----------|------|
| Early Bird | 수업 24시간+ 전 | 30% | 계획형 학습자 | 튜터 지정 + 우선 선택 |
| Planner | 수업 12시간+ 전 | 30% | 아침 예약자 | 튜터 우선 선택 |
| Same-day | 수업 6시간+ 전 | 20% | 오후 예약자 | 기본 예약 |
| Last Minute | 수업 1시간+ 전 | 20% | 퇴근 후 예약자 | 기본 예약 (잔여분) |

**주의:** 비율은 초기값. 실제 예약 패턴 데이터를 보고 2주 단위로 조정.

### 4-3. 비피크 시간대는?

시차 오픈은 **피크타임(19-23시)에만 적용**.

비피크(10-18시)는 이미 공급이 충분하므로 기존대로 1시간 전 예약 유지.

```
피크타임 (19:00-23:30): 시차 오픈 적용
비피크 (10:00-18:30):    기존 1시간 전 예약 유지
```

## 5. UX 설계

### 5-1. 예약 화면 변경

**현재 예약 화면:**
```
3/25(화)
19:00  [예약하기]
19:30  [예약하기]
20:00  [마감]
20:30  [마감]
21:00  [예약하기]
```

**변경 후 예약 화면:**
```
3/25(화)

19:00  [예약하기]
19:30  [예약하기]
20:00  [마감] → 🔔 빈자리 알림 (방안 1 연계)
20:30  ⭐ 사전예약 전용 (12시간 전 오픈)
       "내일 아침 8:30에 예약 가능해요!"
21:00  [예약하기]
21:30  ⭐ 사전예약 전용 (내일 오후 2시 오픈)
```

### 5-2. "사전 예약하면 좋은 이유" 넛지

첫 사전 예약 유도를 위한 온보딩 메시지:

```
┌─────────────────────────────────────────────┐
│  💡 알고 계셨나요?                           │
│                                              │
│  미리 예약하면 더 좋은 수업을 받을 수 있어요!  │
│                                              │
│  ⭐ 6시간 전 → 원하는 선생님 선택 가능         │
│  ⭐ 12시간 전 → 인기 시간대 우선 예약          │
│  ⭐ 하루 전 → 선생님 지정 예약까지!            │
│                                              │
│  [내일 수업 미리 예약하기]                     │
└─────────────────────────────────────────────┘
```

### 5-3. 예약 성공 시 강화 메시지

```
"⭐ 사전 예약 완료! 田中 선생님과의 수업이 확정되었습니다."
"미리 예약해주셔서 원하시는 선생님과 수업할 수 있었어요!"
```

→ "미리 예약한 덕분에" 프레이밍 = 다음번에도 미리 예약하도록 강화

### 5-4. Last Minute 예약자에게 보여줄 넛지

```
[20:00 슬롯 마감 시]

"이 시간대는 사전 예약으로 마감되었어요."
"다음에는 미리 예약하면 인기 시간대를 확보할 수 있어요!"

[내일 수업 미리 예약하기]  [다른 시간대 보기]  [빈자리 알림 (방안 1)]
```

→ "풀부킹"이 아니라 "사전 예약으로 마감" = **다음 행동(미리 예약)을 학습**

## 6. 행동경제학 기반 설계 원칙

### 6-1. 손실 회피 (Loss Aversion)

```
❌ "미리 예약하면 보너스를 드려요" (획득 프레이밍)
✅ "이 시간대는 사전 예약 학생에게 먼저 배정됩니다" (손실 프레이밍)
```

"내가 안 하면 다른 사람이 먼저 가져간다" → 더 강력한 동기

### 6-2. 기본값 효과 (Default Effect)

```
수업 완료 후 자동 팝업:
"다음 수업도 미리 예약해두시겠어요?"
[내일 같은 시간 예약] [이번 주 금요일 예약] [나중에]
```

→ 수업 직후가 "다음 수업 예약" 동기가 가장 높은 시점

### 6-3. 사회적 증거 (Social Proof)

```
"이 시간대를 예약한 학생의 78%가 사전 예약을 이용했어요"
"田中 선생님의 내일 수업은 이미 2명이 사전 예약했어요"
```

### 6-4. 진행 효과 (Endowed Progress)

```
[사전 예약 습관 트래커]

"이번 달 사전 예약: 🟢🟢🟢⚪⚪⚪⚪⚪ (3/8)"
"5회 달성 시 다음 달 프리미엄 시간대 우선권!"
```

## 7. 튜터 측 영향

### 7-1. 튜터에게 변하는 것

**거의 없음.** 이게 이 방안의 최대 장점.

| 항목 | 변화 여부 |
|------|----------|
| 튜터 보수 | 변화 없음 |
| 튜터 스케줄 관리 | 변화 없음 |
| 수업 방식 | 변화 없음 |
| 학생 배정 로직 | 약간 변경 (사전 예약자 우선 매칭) |

### 7-2. 튜터에게 긍정적 영향

- **수업 취소 감소**: 미리 예약한 학생이 취소할 확률이 last-minute 예약자보다 낮음 (심리적 commitment)
- **스케줄 예측 가능성 증가**: 하루 전에 이미 30-60%가 차 있으면 튜터도 계획 세우기 편함
- **지명 예약 = 동기부여**: "나를 지명한 학생이 있다" → 수업 만족도/퀄리티 향상

### 7-3. 튜터 스케줄 오픈 시점과의 시너지

현재 문제: 튜터가 스케줄을 늦게 열면 사전 예약 자체가 불가능

**보완:**
- 튜터에게 "내일 스케줄을 오늘 저녁까지 열어주시면, 사전 예약 학생이 바로 배정됩니다" 안내
- [[방안 2 - 주간 일관성 보너스]]의 조건에 "스케줄 24시간 전 오픈" 보너스 추가
- 사전 예약 학생이 지명한 튜터에게 "○○님이 내일 수업을 원합니다" 알림 → 스케줄 오픈 유도

## 8. 수요 분산 시뮬레이션

### 8-1. 현재 예약 시점 분포 (추정)

```
예약 시점별 분포 (피크타임 19-23시):
  24시간+ 전:  10%
  12시간+ 전:  10%
  6시간+ 전:   15%
  1-6시간 전:  25%
  수업 1시간 전: 40%  ← 여기서 풀부킹 발생
```

### 8-2. 사전 예약 보상 도입 후 기대 분포

```
목표 분포 (3개월 후):
  24시간+ 전:  25%  (+15pp)  ← 튜터 지정 인센티브
  12시간+ 전:  20%  (+10pp)  ← 인기 시간대 우선
  6시간+ 전:   20%  (+5pp)   ← 튜터 선택권
  1-6시간 전:  20%  (-5pp)
  수업 1시간 전: 15%  (-25pp) ← 대폭 감소
```

### 8-3. 풀부킹 빈도 변화 예측

```
현재: 피크타임 30개 슬롯이 18시~19시 사이 1시간 안에 전부 소진
  → 18:15에 오면 이미 마감 (체감 풀부킹률 ~80%)

변경 후: 30개 슬롯이 하루에 걸쳐 점진적 소진
  - 아침까지 9개 소진 (30%)
  - 오후까지 18개 소진 (60%)
  - 저녁 6시까지 24개 소진 (80%)
  - 저녁 7시: 6개 잔여 (20%)
  → 18:15에 와도 6개 남아있음 (체감 풀부킹률 ~30%)
```

**같은 공급(30개)인데 체감 풀부킹률이 80% → 30%로 개선.**

## 9. 점진적 도입 전략

### 9-1. Phase 0: 데이터 수집 (1주)

현재 예약 패턴의 정확한 기준선 확보

- [ ] 예약 시점 분포 데이터 수집 (수업 시작 N시간 전에 예약했는가)
- [ ] 시간대별/요일별 예약-마감 시점 분석
- [ ] 사전 예약자 vs last-minute 예약자의 리텐션/취소율 비교
- [ ] 학생 서베이: "수업을 보통 언제 예약하시나요? 미리 예약하면 좋은 점이 있다면?"

### 9-2. Phase 1: 소프트 런칭 — 보상만 (2주)

**시차 오픈 없이**, 보상만 먼저 도입하여 행동 변화 유도

- [ ] 6시간+ 전 예약 시 "튜터 우선 선택권" 활성화
- [ ] 예약 완료 후 "다음 수업 미리 예약하기" 넛지 추가
- [ ] 사전 예약 비율 변화 모니터링
- [ ] 예약 시점 분포 변화 측정

**Phase 1에서 이미 사전 예약 비율이 유의미하게 증가하면 → Phase 2 불필요할 수도 있음.**

### 9-3. Phase 2: 시차 오픈 도입 (2주)

Phase 1의 보상만으로 분산이 부족할 경우

- [ ] 피크타임 슬롯에 시차 오픈 적용 (30/30/20/20 비율)
- [ ] "사전 예약 전용" 슬롯 UI 표시
- [ ] Last minute 예약 화면에 "사전 예약으로 마감" 메시지
- [ ] 풀부킹 빈도 변화 측정

### 9-4. Phase 3: 최적화 (ongoing)

- [ ] 시차 오픈 비율 데이터 기반 최적화
- [ ] 튜터 지정 예약 기능 추가 (24시간+ 전)
- [ ] 사전 예약 습관 트래커 / 게이미피케이션
- [ ] 월간 "사전 예약 챔피언" 뱃지

## 11. 엣지 케이스

### 11-1. 사전 예약 후 취소하면?

- 사전 예약 혜택을 남용할 수 있음 (좋은 튜터 선점 → 취소 → 다시 선점)
- **완화:** 사전 예약 취소 시 해당 슬롯은 일반 풀로 즉시 반환, 당일 재예약 시 사전 예약 혜택 미적용
- **패턴 감지:** 사전 예약 후 취소 비율이 30%+ 넘으면 경고

### 11-2. 사전 예약 전용 슬롯이 안 채워지면?

- 시차 오픈에서 Phase 1(24시간 전) 슬롯이 비어있으면 → Phase 2 시점에 일반 풀로 전환
- **절대 빈 슬롯으로 남겨두지 않음** → 공급 낭비 방지

```
24시간 전 오픈된 10개 슬롯 중 3개만 예약됨
→ 12시간 전 시점에 남은 7개는 일반 풀에 합산
→ 결과적으로 12시간 전 오픈 = 7 + 10 = 17개
```

### 11-3. 튜터가 스케줄을 늦게 열면?

- 24시간 전에 스케줄을 안 열어놓은 튜터의 슬롯은 시차 오픈 대상이 아님
- 해당 슬롯은 오픈 시점부터 일반 풀로 진입
- **방안 2(주간 보너스)에 "24시간 전 스케줄 오픈" 보너스 추가로 유도**

### 11-4. 비피크 시간대도 요청하면?

- 비피크에서도 사전 예약 시 튜터 선택권은 제공 (인센티브 일관성)
- 시차 오픈은 피크타임에만 적용 (비피크는 공급 충분)

## 15. 이 방안이 특별한 이유

기존 방안 1-7은 대부분 **공급 측(튜터)에 개입**하는 방안. 이 방안은:

1. **수요 측(학생)의 행동 패턴을 바꿈** → 같은 공급으로 체감 풀부킹 대폭 감소
2. **튜터 정책 변경 없음** → 튜터 이탈 리스크 0
3. **비용 없음** → 인센티브가 아닌 "기능 잠금해제" 방식
4. **되돌리기 쉬움** → Phase 1이 안 되면 끄면 됨
5. **데이터 축적** → 예약 시점 분포를 정밀하게 파악 가능

> **핵심 프레이밍: "일찍 예약하면 더 좋은 수업을 받는다" — 빼앗는 게 아니라 주는 것.**
