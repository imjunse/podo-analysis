# 방안 8: 사전 예약 보상 모델

> 상위 문서: [[문제 분석 및 해결 방안]]
> 난이도: 중 | 소요: ~2-3주 | 기대 효과: 높음

---

## 1. 문제 재정의: 왜 Last Minute Booking이 문제인가

### 1-1. 예약 맵에서 확인된 사실 (2026-03-23 주간)

| 구분 | 가용 슬롯 | 상태 |
|------|----------|------|
| 월/화 (당일~내일) | 0~10개 | 빨간색/분홍색 (거의 마감) |
| 수~일 (2일+ 후) | 12~48개 | 초록색 (넉넉) |
| 0 Tutor 시간대 | 월 14개, 화 23개 | 튜터 자체가 없는 슬롯 |

**핵심 패턴:** 2일 후까지는 충분한 자리가 있지만, 당일/내일에 몰리면서 풀부킹 발생.

### 1-2. Last Minute Booking의 원인

```
현재 정책: 수업 1시간 전까지 예약 가능

한국 직장인의 현실:
  09:00 출근
  ... 미팅, 야근 여부 불확실 ...
  18:00 "오늘 7시에 끝날 것 같다" → 7:30 수업 예약 시도
  → 이미 풀부킹 → 좌절/이탈
```

- 학생의 "게으름"이 아니라 **불확실한 퇴근 시간에 대한 합리적 대응**
- 이전 12시간 전 예약 정책 시절에는 쏠림이 없었지만, **유저 편의를 위해 1시간으로 완화**한 이력 있음

### 1-3. 왜 "강제"가 아닌 "보상"인가

| 접근 | 효과 | 리스크 |
|------|------|--------|
| 6-12시간 전 예약 **강제** | 쏠림 감소 | 이탈 가속 (수업 못 듣는 학생 증가) |
| 사전 예약 **보상** | 점진적 분산 | 이탈 없음 (기존 유저 경험 유지) |

> **원칙: 기존 유저 경험을 절대 악화시키지 않으면서, 사전 예약에 확실한 이점을 부여하여 행동 변화를 유도한다.**

---

## 2. 핵심 가설

> "사전 예약 시 확실한 보상(인기 시간대 우선 배정, 취소 유연성)을 제공하면, 학생의 예약 시점이 점진적으로 앞당겨지고, 피크타임 풀부킹 빈도가 줄어든다."

**측정 지표:**
- 사전 예약(12시간+ 전) 비율 변화
- 피크타임(19-23시) 풀부킹 빈도 변화
- 전체 예약 성공률 변화
- 환불률 변화 (booking 관련)
- 리텐션 변화 (사전 예약자 vs last-minute 예약자)

## 3. 시스템 제약 조건

### 3-1. 랜덤 매칭 시스템

현재 앱은 **튜터 랜덤/알고리즘 배정** 방식으로, 학생이 특정 튜터를 골라서 예약할 수 없다.

→ 기존에 검토했던 "튜터 선택권", "튜터 지정 예약" 보상은 **시스템 구조상 불가**.

### 3-2. 현재 취소 정책 (코드베이스 기준)

| 시간 구간 | 상태 | 결과 |
|-----------|------|------|
| **2시간 이상** 전 | `CANCEL` | 무료 취소 (수업권 복구 + 재예약) |
| **1~2시간** 전 | `CANCEL` | 수업권 차감 / 무제한은 72시간 예약 금지 |
| **1시간 이내** | `CANCEL_PAID` | 수업권 차감 / 무제한은 72시간 예약 금지 |
| **노쇼** | `NOSHOW_S` | 24시간 예약 금지 (기존 72시간에서 완화) |

> 참조: [[정규 수업 예약 및 취소 정책]]

## 4. 보상 체계 설계 (v2)

### 4-1. 사전 예약 시간대별 보상 등급

```
┌──────────────────────────────────────────────────────────┐
│                 사전 예약 보상 체계 (v2)                    │
│                                                          │
│  🥉 1시간 전     → 기존과 동일 (제한 없음)                │
│  🥈 6시간+ 전    → 피크 시간대 우선 배정                   │
│  🥇 12시간+ 전   → 피크 우선 배정 + 무료 취소 1시간 전까지  │
│  💎 24시간+ 전   → 모든 혜택 + 최우선 배정                 │
│                                                          │
│  ⚡ 핵심: 기존 1시간 전 예약은 그대로 유지!               │
│     아무것도 빼앗지 않고, 더 일찍 예약하면 더 줌           │
└──────────────────────────────────────────────────────────┘
```

### 4-2. 보상 항목 상세

#### 보상 1: 인기 시간대 우선 배정 (6시간+ 전)

**개념:** 피크타임(19-23시) 슬롯의 일부를 사전 예약자에게 우선 배정

```
[튜터 A의 20:00 슬롯]

12시간+ 전 예약 가능 시점: 아침 8시부터 오픈
6시간+ 전 예약 가능 시점: 오후 2시부터 오픈
1시간 전 예약 가능 시점: 오후 7시부터 오픈 (남은 자리만)
```

**구현 방식: "시차 오픈" (Staggered Release)**

```
피크타임 슬롯 = 총 30개라고 가정

Phase 1 (수업 24시간+ 전): 30개 중 10개(33%) 오픈
  → 24시간+ 전 예약자가 가장 먼저 선점

Phase 2 (수업 12시간+ 전): 추가 10개(33%) 오픈
  → 12시간+ 전 예약자가 선점

Phase 3 (수업 6시간+ 전): 추가 5개(17%) 오픈

Phase 4 (수업 1시간+ 전): 나머지 5개(17%) 오픈
  → Last minute 예약자도 가능 (단, 자리 적음)
```

**효과:**
- "일찍 예약하면 확실히 수업할 수 있다" 경험 → 다음번에도 일찍 예약
- Last minute 유저도 수업 가능 (17% 잔여분) → 이탈 방지
- 풀부킹 체감이 줄어듦 (시간대별로 점진적 마감)

**왜 효과적인가:**
- 학생이 가장 원하는 건 "원하는 시간에 수업하는 것"
- 시간대 확보 = 높은 체감 가치 (실제 비용 0)
- "일찍 예약하면 좋은 시간을 확보할 수 있다" → 강력한 동기
- 별도의 시스템 개발 없이 슬롯 오픈 로직만 변경하면 됨

#### 보상 2: 취소 유연성 확대 (12시간+ 전 예약자)

**현재 정책:** 모든 예약자 동일하게 수업 2시간 전까지 무료 취소
**변경:** 12시간+ 전 사전 예약자는 수업 1시간 전까지 무료 취소

```
┌─────────────────────────────────────────────────────┐
│              취소 정책 비교                            │
│                                                      │
│  일반 예약:           수업 2시간 전까지 무료 취소 (현행) │
│  사전 예약 (12시간+):  수업 1시간 전까지 무료 취소      │
│                                                      │
│  * 튜터 확인: 1시간 전 무료 취소까지는 수용 가능       │
│  * 노쇼 패널티(24시간 금지)는 그대로 유지              │
└─────────────────────────────────────────────────────┘
```

**왜 이것이 핵심 보상인가 — 직장인 시나리오:**

```
[현행] 미리 예약하면 리스크가 큼:
  월요일 밤: 화요일 20시 수업 예약
  화요일 18시: "오늘 야근 확정" → 2시간 전 넘김 → 취소 불가
  → 수업권 차감 or 72시간 예약 금지 💀
  → 결론: "미리 예약하면 손해" → 당일에만 예약

[변경 후] 미리 예약해도 안전:
  월요일 밤: 화요일 20시 수업 사전 예약 (12시간+ 전)
  화요일 19시: "야근 확정" → 1시간 전이지만 무료 취소 가능 ✅
  → 슬롯이 19시에 풀림 → 다른 학생이 잡을 수 있음
  → 튜터도 1시간 여유면 수용 가능
  → 결론: "미리 예약해도 안전하다" → 사전 예약 장벽 제거
```

**이 보상의 강점:**
- 사전 예약의 **가장 큰 심리적 장벽**(야근 불확실성)을 정확히 제거
- 비용 0원 (무료 취소 시간만 1시간 확대)
- 튜터 입장에서도 1시간 전 취소는 수용 가능 (확인 완료)
- 취소된 슬롯은 즉시 재오픈 → 다른 last-minute 학생에게 기회

## 5. 시차 오픈 (Staggered Release) 상세 설계

### 5-1. 왜 시차 오픈인가

**현재 문제:**
```
모든 슬롯이 수업 1시간 전부터 동시에 예약 가능
→ 18시에 퇴근한 학생들이 19-23시 슬롯에 일제히 몰림
→ 10분 만에 풀부킹
→ 18:15에 접속한 학생은 이미 마감
```

**시차 오픈 후:**
```
아침 8시: 피크 슬롯의 33% 오픈 → 출근 전에 예약하는 학생 확보
오후 2시: 추가 33% 오픈 → 점심시간에 예약하는 학생 확보
오후 6시: 추가 17% 오픈 → 퇴근 직후 예약 가능
오후 7시: 잔여 17% 오픈 → Last minute도 가능

→ 풀부킹 시점이 "18시 한꺼번에"에서 "하루에 걸쳐 점진적"으로 변화
→ "지금 안 되면 끝"이 아니라 "더 일찍 했으면 됐다"로 프레이밍 전환
```

### 5-2. 슬롯 배분 비율

| Phase | 오픈 시점 | 배분 비율 | 타겟 학생 | 보상 |
|-------|----------|----------|----------|------|
| Early Bird | 수업 24시간+ 전 | 30% | 계획형 학습자 | 시간대 우선 배정 + 취소 유연성 |
| Planner | 수업 12시간+ 전 | 30% | 아침 예약자 | 시간대 우선 배정 + 취소 유연성 |
| Same-day | 수업 6시간+ 전 | 20% | 오후 예약자 | 시간대 우선 배정 |
| Last Minute | 수업 1시간+ 전 | 20% | 퇴근 후 예약자 | 기본 예약 (잔여분) |

**주의:** 비율은 초기값. 실제 예약 패턴 데이터를 보고 2주 단위로 조정.

### 5-3. 비피크 시간대는?

시차 오픈은 **피크타임(19-23시)에만 적용**.

비피크(10-18시)는 이미 공급이 충분하므로 기존대로 1시간 전 예약 유지.

```
피크타임 (19:00-23:30): 시차 오픈 적용
비피크 (10:00-18:30):    기존 1시간 전 예약 유지
```

## 6. UX 설계

### 6-1. 예약 화면 변경

**현재 예약 화면:**
```
3/25(화)
19:00  [예약하기]
19:30  [예약하기]
20:00  [마감]
20:30  [마감]
21:00  [예약하기]
```

**변경 후 예약 화면:**
```
3/25(화)

19:00  [예약하기]
19:30  [예약하기]
20:00  [마감] → 🔔 빈자리 알림 (방안 1 연계)
20:30  ⭐ 사전예약 전용 (12시간 전 오픈)
       "내일 아침 8:30에 예약 가능해요!"
21:00  [예약하기]
21:30  ⭐ 사전예약 전용 (내일 오후 2시 오픈)
```

### 6-2. "사전 예약하면 좋은 이유" 넛지

첫 사전 예약 유도를 위한 온보딩 메시지:

```
┌─────────────────────────────────────────────┐
│  💡 알고 계셨나요?                           │
│                                              │
│  미리 예약하면 더 좋은 혜택이 있어요!         │
│                                              │
│  ⭐ 6시간 전 → 인기 시간대 우선 배정          │
│  ⭐ 12시간 전 → 취소 유연성까지! (1시간 전    │
│     무료 취소 가능)                           │
│                                              │
│  [내일 수업 미리 예약하기]                     │
└─────────────────────────────────────────────┘
```

### 6-3. 예약 성공 시 강화 메시지

```
"⭐ 사전 예약 완료! 인기 시간대를 확보했어요."
"12시간+ 전 예약이라 수업 1시간 전까지 무료 취소도 가능합니다!"
```

→ "미리 예약한 덕분에" 프레이밍 = 다음번에도 미리 예약하도록 강화

### 6-4. Last Minute 예약자에게 보여줄 넛지

```
[20:00 슬롯 마감 시]

"이 시간대는 사전 예약으로 마감되었어요."
"다음에는 미리 예약하면 인기 시간대를 확보할 수 있어요!"
"미리 예약하면 취소도 더 자유롭습니다 (1시간 전까지 무료 취소)"

[내일 수업 미리 예약하기]  [다른 시간대 보기]  [빈자리 알림 (방안 1)]
```

→ "풀부킹"이 아니라 "사전 예약으로 마감" = **다음 행동(미리 예약)을 학습**

## 7. 행동경제학 기반 설계 원칙

### 7-1. 손실 회피 (Loss Aversion)

```
❌ "미리 예약하면 보너스를 드려요" (획득 프레이밍)
✅ "이 시간대는 사전 예약 학생에게 먼저 배정됩니다" (손실 프레이밍)
```

"내가 안 하면 다른 사람이 먼저 가져간다" → 더 강력한 동기

### 7-2. 기본값 효과 (Default Effect)

```
수업 완료 후 자동 팝업:
"다음 수업도 미리 예약해두시겠어요?"
[내일 같은 시간 예약] [이번 주 금요일 예약] [나중에]
```

→ 수업 직후가 "다음 수업 예약" 동기가 가장 높은 시점

### 7-3. 리스크 제거 (Risk Reversal)

```
❌ "미리 예약하세요" (야근하면 어쩌지? → 불안)
✅ "미리 예약하면 1시간 전까지 무료 취소도 돼요" (리스크 제거)
```

→ 사전 예약의 최대 장벽(야근 불확실성)을 정면 돌파

### 7-4. 사회적 증거 (Social Proof)

```
"이 시간대를 예약한 학생의 78%가 사전 예약을 이용했어요"
"내일 20시 수업은 이미 3명이 사전 예약했어요"
```

### 7-5. 진행 효과 (Endowed Progress)

```
[사전 예약 습관 트래커]

"이번 달 사전 예약: 🟢🟢🟢⚪⚪⚪⚪⚪ (3/8)"
"5회 달성 시 다음 달 프리미엄 시간대 우선권!"
```

## 8. 튜터 측 영향

### 8-1. 튜터에게 변하는 것

**거의 없음.** 이게 이 방안의 최대 장점.

| 항목 | 변화 여부 |
|------|----------|
| 튜터 보수 | 변화 없음 |
| 튜터 스케줄 관리 | 변화 없음 |
| 수업 방식 | 변화 없음 |
| 학생 배정 로직 | 변화 없음 (랜덤 매칭 유지) |
| 수업 1시간 전 취소 | 수용 가능 (튜터 확인 완료) |

### 8-2. 튜터에게 긍정적 영향

- **수업 취소 감소**: 미리 예약한 학생이 취소할 확률이 last-minute 예약자보다 낮음 (심리적 commitment)
- **스케줄 예측 가능성 증가**: 하루 전에 이미 30-60%가 차 있으면 튜터도 계획 세우기 편함
- **1시간 전 취소 시 슬롯 재오픈**: 취소되더라도 다른 학생이 바로 잡을 수 있어 빈 슬롯 최소화

### 8-3. 튜터 스케줄 오픈 시점과의 시너지

현재 문제: 튜터가 스케줄을 늦게 열면 사전 예약 자체가 불가능

**보완:**
- 튜터에게 "내일 스케줄을 오늘 저녁까지 열어주시면, 사전 예약 학생이 바로 배정됩니다" 안내
- [[방안 2 - 주간 일관성 보너스]]의 조건에 "스케줄 24시간 전 오픈" 보너스 추가

## 9. 수요 분산 시뮬레이션

### 9-1. 현재 예약 시점 분포 (추정)

```
예약 시점별 분포 (피크타임 19-23시):
  24시간+ 전:  10%
  12시간+ 전:  10%
  6시간+ 전:   15%
  1-6시간 전:  25%
  수업 1시간 전: 40%  ← 여기서 풀부킹 발생
```

### 9-2. 사전 예약 보상 도입 후 기대 분포

```
목표 분포 (3개월 후):
  24시간+ 전:  25%  (+15pp)  ← 취소 유연성 + 시간대 우선 인센티브
  12시간+ 전:  20%  (+10pp)  ← 취소 유연성 인센티브
  6시간+ 전:   20%  (+5pp)   ← 시간대 우선 배정
  1-6시간 전:  20%  (-5pp)
  수업 1시간 전: 15%  (-25pp) ← 대폭 감소
```

### 9-3. 풀부킹 빈도 변화 예측

```
현재: 피크타임 30개 슬롯이 18시~19시 사이 1시간 안에 전부 소진
  → 18:15에 오면 이미 마감 (체감 풀부킹률 ~80%)

변경 후: 30개 슬롯이 하루에 걸쳐 점진적 소진
  - 아침까지 9개 소진 (30%)
  - 오후까지 18개 소진 (60%)
  - 저녁 6시까지 24개 소진 (80%)
  - 저녁 7시: 6개 잔여 (20%)
  → 18:15에 와도 6개 남아있음 (체감 풀부킹률 ~30%)
```

**같은 공급(30개)인데 체감 풀부킹률이 80% → 30%로 개선.**

### 9-4. 이 방안의 한계 (솔직한 평가)

```
⚠️ 방안 8은 "쏠림 시점"을 분산시키지, "절대적 공급 부족"을 해결하지는 않는다.

수요 50명 vs 공급 30슬롯이면:
  현재:   18시에 30개 소진 → 20명 못 잡음
  방안 8: 하루에 걸쳐 30개 소진 → 여전히 20명 못 잡음

→ 수요 ≈ 공급 (약간 부족): 방안 8만으로 체감 대폭 개선
→ 수요 >> 공급 (심각한 부족): 공급 확대 방안과 병행 필수
```

### 10-1. Phase 0: 데이터 수집 (1주)

현재 예약 패턴의 정확한 기준선 확보

- [ ] 예약 시점 분포 데이터 수집 (수업 시작 N시간 전에 예약했는가)
- [ ] 시간대별/요일별 예약-마감 시점 분석
- [ ] 사전 예약자 vs last-minute 예약자의 리텐션/취소율 비교
- [ ] 학생 서베이: "수업을 보통 언제 예약하시나요? 미리 예약하면 좋은 점이 있다면?"

### 10-2. Phase 1: 소프트 런칭 — 취소 유연성만 (2주)

**시차 오픈 없이**, 취소 유연성만 먼저 도입하여 행동 변화 유도

- [ ] 12시간+ 전 예약 시 무료 취소 기준을 2시간 → 1시간 전으로 확대
- [ ] 예약 완료 후 "다음 수업 미리 예약하기" 넛지 추가
- [ ] 사전 예약 시 "1시간 전까지 무료 취소 가능" 안내 표시
- [ ] 사전 예약 비율 변화 모니터링

**Phase 1에서 이미 사전 예약 비율이 유의미하게 증가하면 → Phase 2 불필요할 수도 있음.**

### 10-3. Phase 2: 시차 오픈 도입 (2주)

Phase 1의 보상만으로 분산이 부족할 경우

- [ ] 피크타임 슬롯에 시차 오픈 적용 (30/30/20/20 비율)
- [ ] "사전 예약 전용" 슬롯 UI 표시
- [ ] Last minute 예약 화면에 "사전 예약으로 마감" 메시지
- [ ] 풀부킹 빈도 변화 측정

### 10-4. Phase 3: 최적화 (ongoing)

- [ ] 시차 오픈 비율 데이터 기반 최적화
- [ ] 사전 예약 습관 트래커 / 게이미피케이션
- [ ] 월간 "사전 예약 챔피언" 뱃지

## 12. 엣지 케이스

### 12-1. 사전 예약 후 취소 남용 (슬롯 선점 → 취소 반복)

- 취소 유연성이 있으므로 "일단 잡고 취소" 패턴 가능
- **완화:** 사전 예약 취소 시 해당 슬롯은 일반 풀로 즉시 반환
- **패턴 감지:** 사전 예약 후 취소 비율이 30%+ 넘는 학생에게는 취소 유연성 혜택 제한
- **핵심:** 노쇼 패널티(24시간 금지)는 그대로 유지 → 무단 노쇼 억제력 건재

### 12-2. 사전 예약 전용 슬롯이 안 채워지면?

- 시차 오픈에서 Phase 1(24시간 전) 슬롯이 비어있으면 → Phase 2 시점에 일반 풀로 전환
- **절대 빈 슬롯으로 남겨두지 않음** → 공급 낭비 방지

```
24시간 전 오픈된 10개 슬롯 중 3개만 예약됨
→ 12시간 전 시점에 남은 7개는 일반 풀에 합산
→ 결과적으로 12시간 전 오픈 = 7 + 10 = 17개
```

### 12-3. 튜터가 스케줄을 늦게 열면?

- 24시간 전에 스케줄을 안 열어놓은 튜터의 슬롯은 시차 오픈 대상이 아님
- 해당 슬롯은 오픈 시점부터 일반 풀로 진입
- **방안 2(주간 보너스)에 "24시간 전 스케줄 오픈" 보너스 추가로 유도**

### 12-4. 비피크 시간대도 요청하면?

- 비피크에서도 사전 예약 시 취소 유연성은 제공 (인센티브 일관성)
- 시차 오픈은 피크타임에만 적용 (비피크는 공급 충분)

### 12-5. 취소 유연성으로 1시간 전 취소 → 슬롯이 너무 늦게 풀리면?

- 1시간 전 취소된 슬롯은 즉시 재오픈 → last-minute 예약자에게 기회
- 오히려 **현행보다 나음**: 현행은 패널티 무서워서 노쇼 → 슬롯 완전 낭비
- 변경 후: 무료 취소 → 슬롯 재활용 가능

## 16. 이 방안이 특별한 이유

기존 방안 1-7은 대부분 **공급 측(튜터)에 개입**하는 방안. 이 방안은:

1. **수요 측(학생)의 행동 패턴을 바꿈** → 같은 공급으로 체감 풀부킹 대폭 감소
2. **튜터 정책 변경 없음** → 튜터 이탈 리스크 0
3. **비용 없음** → 인센티브가 아닌 "정책 차등" 방식
4. **되돌리기 쉬움** → Phase 1이 안 되면 끄면 됨
5. **데이터 축적** → 예약 시점 분포를 정밀하게 파악 가능
6. **랜덤 매칭 시스템 그대로** → 앱 구조 변경 불필요

> **핵심 프레이밍: "일찍 예약하면 좋은 시간도 확보하고, 취소도 자유롭다" — 빼앗는 게 아니라 주는 것.**
