# 방안 6: 그룹 수업 도입 (1:1 → 1:2/1:3)

> 상위 문서: [[문제 분석 및 해결 방안]]
> 난이도: 중-높 | 소요: 4-6주 | 기대 효과: 높음 (공급 2-3배 확대) | 우선순위: Tier 3 - 검토 필요

---

## 1. 핵심 가설

> "풀부킹 상황에서 1:1 대신 1:2 그룹 수업을 '할인 옵션'으로 제공하면, 수업을 못 받는 학생 수가 줄면서도 튜터 비용 효율이 개선된다."

**수학적 근거:**
- 현재: 튜터 1명 = 학생 1명 (25분)
- 1:2: 튜터 1명 = 학생 2명 (25분) → **같은 공급으로 수용 인원 2배**
- 1:3: 튜터 1명 = 학생 3명 (25분) → **같은 공급으로 수용 인원 3배**

---

## 2. 브랜드 리스크 분석

### 2-1. Podo의 핵심 가치 = "1:1 수업"

Podo의 마케팅/포지셔닝이 1:1 수업에 기반한다면, 그룹 수업 도입은 브랜드 정체성을 훼손할 수 있음.

**브랜드 영향 시나리오:**

| 접근 | 브랜드 영향 | 리스크 |
|------|-----------|--------|
| 그룹 수업을 메인으로 전환 | 높음 (정체성 훼손) | 기존 유저 이탈 |
| 풀부킹 시에만 대안으로 제공 | 낮음 (보완재) | 제한적 |
| 별도 상품으로 분리 (Podo Group) | 없음 (별개 브랜딩) | 운영 복잡도 증가 |

**권장: "풀부킹 시에만 대안으로 제공" 또는 "별도 상품으로 분리"**

### 2-2. 학생 관점: 1:1 vs 그룹

| 항목 | 1:1 | 1:2 그룹 |
|------|-----|---------|
| 발화 시간 | 25분 중 ~12분 | 25분 중 ~6분 |
| 맞춤 피드백 | 100% 개인화 | 50% 개인화 |
| 긴장감 | 높음 (초급에겐 부담) | 낮음 (동료 효과) |
| 상호 학습 | 없음 | 있음 (서로 듣고 배움) |
| 가격 기대 | 프리미엄 | 할인 기대 |

**초급 학생에게는 그룹 수업이 오히려 장점이 될 수 있음:**
- 혼자 25분간 일본어로 말하는 부담 감소
- 다른 학생의 실수/성공에서 학습 (관찰 학습 효과)
- "나만 못하는 게 아니구나" → 학습 동기 유지

## 3. 수업 설계

### 3-1. 그룹 수업 포맷

```
[1:2 그룹 수업 - 25분 구성]

0-3분: 인사 + 오늘의 주제 소개
3-10분: 학생 A 발화 + 튜터 피드백
10-17분: 학생 B 발화 + 튜터 피드백
17-23분: A&B 대화 연습 (튜터 모니터링)
23-25분: 마무리 + 피드백 요약
```

**핵심:** 학생 간 대화 연습이 들어가면 튜터의 부담이 줄면서도 학생의 발화량은 유지 가능.

### 3-2. 레벨 매칭 조건

그룹 수업의 최대 리스크 = 레벨 불일치

| 허용 범위 | 예시 | 품질 영향 |
|----------|------|----------|
| 같은 서브레벨 | A1.1 + A1.1 | 최적 |
| ±1 서브레벨 | A1.1 + A1.2 | 허용 |
| ±2 이상 | A1.1 + A2.1 | 불허 (양쪽 불만) |

### 3-3. 스케줄 매칭 난이도

**이게 가장 큰 기술적 도전:**

```
1:1 매칭: 시간 + 튜터 타입 → 매칭 완료
1:2 매칭: 시간 + 튜터 타입 + 레벨 ±1 + 학생 2명 동시 → 매칭 매우 어려움
```

**해결 접근:**
1. **학생이 그룹 수업을 "신청"** → 같은 시간대 + 같은 레벨 학생이 2명 모이면 수업 확정
2. **최소 매칭 시간 설정**: 수업 24시간 전까지 2명 안 모이면 → 1:1로 전환 또는 취소
3. **인기 시간대(19-22시) 우선**: 학생 풀이 큰 시간대에서만 제공

## 4. 가격 설계

### 4-1. 가격 옵션

| 옵션 | 1:1 가격 대비 | 튜터 비용 | 마진 | 학생 인센티브 |
|------|-------------|----------|------|-------------|
| A. 동일 가격 | 100% | 50% (학생당) | ↑↑ | 없음 (수용 어려움) |
| B. 30% 할인 | 70% | 50% | ↑ | 중간 |
| C. 50% 할인 | 50% | 50% | 동일 | 높음 |
| D. 레슨권 0.5개 차감 | 50% | 50% | ↑ | 높음 |

**권장: 옵션 D (레슨권 0.5개 차감)**
- 학생 입장: 1:1 수업 1회 대신 1:2 수업 2회 가능 → "더 많이 수업할 수 있다"
- 회사 입장: 튜터 비용은 동일하지만 레슨권 소진 속도 절반 → 리텐션 향상
- 매출 임팩트: 단기적으로 레슨권당 매출 감소, 장기적으로 리텐션 향상으로 보상

### 4-2. 튜터 보상

| 옵션 | 수업당 보수 | 튜터 반응 (예상) |
|------|-----------|----------------|
| A. 1:1과 동일 | 3,374원 (25분) | 좋음 (추가 노력 대비 동일 보상) |
| B. 1.3배 | ~4,386원 | 매우 좋음 (인센티브) |
| C. 0.8배 | ~2,699원 | 나쁨 (학생 2명인데 보수 감소) |

**권장: 옵션 A (동일 보수)**
- 튜터에게 그룹 수업은 추가 부담이 약간 있지만, 보수가 동일하면 수용 가능
- 초기에 1.3배로 시작 → 안정화 후 1.0배로 조정하는 것도 고려

## 6. 기술적 도전

### 6-1. 매칭 알고리즘

```
function matchGroupLesson(request):
    // 같은 시간대 + 같은 언어 + 레벨 ±1인 대기자 조회
    candidates = getPendingRequests(
        date = request.date,
        hour = request.hour,
        language = request.language,
        levelRange = [request.level - 1, request.level + 1]
    )

    if candidates.count >= 1:  // 이미 1명 대기 중이면 2명 완성
        partner = candidates.sortBy(levelProximity).first()
        tutor = findAvailableTutor(request.date, request.hour, type='group')

        if tutor:
            createGroupLesson(tutor, [request.user, partner.user])
            notify([request.user, partner.user], "매칭 완료!")
        else:
            // 그룹 수업 가능한 튜터 없음
            addToWaitingPool(request)
    else:
        addToWaitingPool(request)

    // 24시간 후 매칭 안 되면 자동 취소
    scheduleJob(after: 24.hours, action: expireIfUnmatched(request.id))
```

### 6-2. 화상 수업 인프라

- 현재 1:1 수업 인프라가 1:2를 지원하는가?
- WebRTC/Agora/Twilio 등 3자 화상 지원 여부 확인 필요
- 화면 구성: 튜터 1명 + 학생 2명의 레이아웃

### 6-3. 교재/커리큘럼

- 현재 교재가 1:1 기준으로 설계됨
- 그룹용 교재 별도 필요: 학생 간 대화 프롬프트, 페어 액티비티 등
- **Level Up 커리큘럼에 그룹 모드를 같이 설계**하면 효율적

## 7. 단계적 도입 계획

### Phase 0: 검증 (2주)
- [ ] 학생 서베이: "1:1 대신 그룹 수업을 할인된 가격에 받을 의향이 있습니까?"
- [ ] 튜터 서베이: "그룹 수업(1:2)을 진행할 의향이 있습니까?"
- [ ] 화상 인프라 1:2 지원 여부 기술 검토
- [ ] 브랜드 영향 내부 토론

### Phase 1: 파일럿 (4주)
- [ ] 피크타임(20-21시)에만, 풀부킹 시에만 그룹 옵션 노출
- [ ] 그룹 수업 의향 있는 튜터 10명 모집
- [ ] 레슨권 0.5개 차감 모델 적용
- [ ] 주간 NPS + 수업 평점 모니터링

### Phase 2: 확대 (4주)
- [ ] 피크타임 전체로 확대
- [ ] 그룹 수업 전용 교재 개발
- [ ] 매칭 알고리즘 고도화

## 9. 리스크 및 완화

| 리스크 | 확률 | 영향 | 완화 방안 |
|--------|------|------|----------|
| 브랜드 정체성 훼손 | 중 | 높 | 풀부킹 시에만 + "대안 옵션" 명확히 포지셔닝 |
| 매칭 실패 (같은 레벨 학생 없음) | 높 | 중 | 피크타임 한정, 24시간 후 자동 취소 |
| 수업 품질 저하 | 중 | 높 | 전용 교재, 튜터 교육, 평점 모니터링 |
| 1:1 → 그룹 카니발리제이션 | 중 | 높 | 그룹은 풀부킹 시에만 노출, 1:1 빈자리 있으면 표시 안 함 |
| 화상 인프라 문제 | 낮 | 높 | Phase 0에서 기술 검토 선행 |
